1. Constraint Satisfaction Problems (CSP)

A CSP consists of variables, domains, and constraints. The goal is to assign values to all variables such that every constraint is satisfied.

1.1 CSP Components

X = {X1, X2, ..., Xn}   -- set of variables
D = {D1, D2, ..., Dn}   -- domains for each variable
C = {C1, C2, ..., Cm}   -- constraints

Example: Map Coloring (Australia)
  Variables: WA, NT, SA, Q, NSW, V, T
  Domains: {Red, Green, Blue} for each
  Constraints: Adjacent regions must have different colors
               WA != NT, WA != SA, NT != SA, NT != Q, ...

1.2 Backtracking Search

function BACKTRACK(assignment, csp):
    if assignment is complete: return assignment
    var = SELECT-UNASSIGNED-VARIABLE(csp)
    for each value in ORDER-DOMAIN-VALUES(var, assignment, csp):
        if value is consistent with assignment:
            add {var = value} to assignment
            result = BACKTRACK(assignment, csp)
            if result != failure: return result
            remove {var = value} from assignment
    return failure

1.3 CSP Heuristics

1.4 Arc Consistency (AC-3)

An arc (Xi, Xj) is consistent if for every value in Di,
there exists some value in Dj that satisfies the constraint.

AC-3 Algorithm:
1. Initialize queue with all arcs
2. Remove arc (Xi, Xj) from queue
3. If REVISE(Xi, Xj) removes any value from Di:
   - If Di is empty: return failure
   - Add all arcs (Xk, Xi) to queue (k != j)
4. Repeat until queue is empty

Time: O(n^2 * d^3) where n = variables, d = max domain size

2. Knowledge Representation

2.1 Propositional Logic

Symbols: P, Q, R (propositions that are true or false)
Connectives:
  NOT    (negation):    ~P
  AND    (conjunction): P ^ Q
  OR     (disjunction): P v Q
  IMPLIES (implication): P -> Q  (equivalent to ~P v Q)
  IFF    (biconditional): P <-> Q

Key equivalences:
  P -> Q  ≡  ~P v Q
  ~(P ^ Q) ≡ ~P v ~Q   (De Morgan's)
  ~(P v Q) ≡ ~P ^ ~Q   (De Morgan's)
  P -> Q  ≡  ~Q -> ~P  (Contrapositive)

2.2 First-Order Logic (FOL)

Additional elements beyond propositional logic:
  Constants:  John, 5, MIT
  Variables:  x, y, z
  Predicates: Loves(x, y), Greater(x, 5)
  Functions:  father(x), age(John)
  Quantifiers:
    Universal:   ∀x P(x) -- "for all x, P(x) holds"
    Existential: ∃x P(x) -- "there exists x such that P(x)"

Example: "Every student loves some course"
  ∀x (Student(x) -> ∃y (Course(y) ^ Loves(x, y)))

Negation rules:
  ~(∀x P(x)) ≡ ∃x ~P(x)
  ~(∃x P(x)) ≡ ∀x ~P(x)

3.2 Conditional Probability Tables

P(B) = 0.001,  P(E) = 0.002

P(A | B, E):
  B=T, E=T: 0.95    B=T, E=F: 0.94
  B=F, E=T: 0.29    B=F, E=F: 0.001

P(J | A):              P(M | A):
  A=T: 0.90             A=T: 0.70
  A=F: 0.05             A=F: 0.01

3.3 Joint Probability Computation

Full joint distribution:
P(B, E, A, J, M) = P(B) * P(E) * P(A|B,E) * P(J|A) * P(M|A)

Example: P(B=T, E=F, A=T, J=T, M=T)
  = P(B=T) * P(E=F) * P(A=T|B=T,E=F) * P(J=T|A=T) * P(M=T|A=T)
  = 0.001 * 0.998 * 0.94 * 0.90 * 0.70
  = 0.000591

Key property: Each node is conditionally independent of its
non-descendants given its parents.

3.4 Inference: Variable Elimination

To compute P(B | J=T, M=T):

P(B|J=T,M=T) = P(B,J=T,M=T) / P(J=T,M=T)

P(B,J=T,M=T) = SUM_E SUM_A P(B)*P(E)*P(A|B,E)*P(J=T|A)*P(M=T|A)

Eliminate variables one at a time, creating intermediate factors.

4. Markov Decision Processes (MDP)

An MDP models sequential decision making under uncertainty.

4.1 MDP Components

S:  Set of states
A:  Set of actions
T:  Transition function T(s, a, s') = P(s' | s, a)
R:  Reward function R(s, a, s')
gamma: Discount factor (0 ≤ gamma ≤ 1)

4.2 Bellman Equation

Optimal value function:
V*(s) = max_a SUM_{s'} T(s, a, s') * [R(s, a, s') + gamma * V*(s')]

Optimal policy:
pi*(s) = argmax_a SUM_{s'} T(s, a, s') * [R(s, a, s') + gamma * V*(s')]

4.3 Value Iteration

Initialize V(s) = 0 for all states
Repeat until convergence:
    For each state s:
        V(s) = max_a SUM_{s'} T(s,a,s') * [R(s,a,s') + gamma * V(s')]

Converges when max change in V(s) < epsilon

Example: Grid World (gamma = 0.9)
  States: 4 cells, Actions: {Up, Down, Left, Right}
  Terminal state at (2,2) with reward +1

  After iteration 1: V(goal neighbors) = 0.9
  After iteration 2: V(2-step away) = 0.81

4.4 Policy Iteration

1. Initialize random policy pi
2. Policy Evaluation: Solve V^pi(s) = SUM T(s,pi(s),s')[R + gamma*V^pi(s')]
3. Policy Improvement: pi'(s) = argmax_a SUM T(s,a,s')[R + gamma*V^pi(s')]
4. If pi' = pi, return pi; else pi = pi', go to step 2

5. Resolution in Propositional Logic

Resolution rule:
  From (A v B) and (~B v C), derive (A v C)

To prove KB |= alpha:
1. Convert KB ^ ~alpha to CNF
2. Apply resolution repeatedly
3. If empty clause {} is derived -> alpha is entailed
4. If no new clauses can be derived -> alpha is not entailed

Example: Prove P from {P v Q, ~Q}
  Add ~P (negate what we want to prove)
  Clauses: {P v Q}, {~Q}, {~P}
  Resolve {P v Q} and {~Q} -> {P}
  Resolve {P} and {~P} -> {} (empty clause!)
  Therefore P is proved.

6. Worked Example: Map Coloring CSP

Problem: Color 3 regions (A, B, C) with {R, G, B}
Constraints: A != B, B != C, A != C

Using backtracking with MRV:
  Step 1: Assign A = R (all variables have 3 values, pick any)
  Step 2: B has domain {G, B} (R eliminated by A!=B constraint)
          Assign B = G
  Step 3: C has domain {B} (R eliminated by A!=C, G by B!=C)
          Assign C = B
  Solution: A=R, B=G, C=B  (found with no backtracking!)
GATE Tip: In CSP questions, always check if arc consistency can reduce domains before starting backtracking. This often simplifies the problem significantly.

1. Domain Name System (DNS)

DNS is a hierarchical, distributed naming system that translates human-readable domain names (e.g., www.example.com) into IP addresses. It uses UDP port 53 (TCP for zone transfers and large responses).

1.1 DNS Hierarchy

LevelDescriptionExample
Root13 root server clusters worldwide (named a.root-servers.net through m.root-servers.net). (dot)
Top-Level Domain (TLD)Generic (.com, .org, .net) and country-code (.in, .uk, .jp) TLDs.com, .edu
AuthoritativeOrganization's own DNS servers that hold actual recordsns1.example.com
Local (Recursive Resolver)ISP or organization DNS that caches and resolves queries for clients8.8.8.8 (Google DNS)

1.2 DNS Resolution

Resolution TypeDescription
RecursiveThe resolver takes full responsibility to return the final answer (does all the work). Client to local DNS is typically recursive.
IterativeThe server returns the best answer it has (often a referral). Local DNS to root/TLD/authoritative is typically iterative.

2. HTTP (Hypertext Transfer Protocol)

HTTP is a stateless, request-response protocol for transferring web resources. It runs on TCP port 80 (HTTPS on port 443).

2.3 Persistent vs Non-Persistent HTTP

FeatureNon-Persistent (HTTP/1.0)Persistent (HTTP/1.1)
ConnectionNew TCP connection per requestMultiple requests on one connection
RTTs per object2 RTT + transmission time (1 for TCP setup, 1 for request/response)1 RTT per object (connection reused)
OverheadHigh (TCP handshake for each object)Low (amortized setup cost)
PipeliningNot possiblePossible (send multiple requests without waiting for responses)
Total time for non-persistent HTTP: To download a page with 1 base HTML and N referenced objects:
Total = (N + 1) × (2 RTT + file transmission time)
With parallel connections (e.g., 6 connections): Total = (1 + ceil(N/6)) × 2 RTT approximately.

3. HTTPS and TLS

HTTPS is HTTP over TLS (Transport Layer Security). TLS provides encryption, authentication, and integrity for HTTP communication.

TLS FeatureDescription
EncryptionSymmetric encryption (AES) for data; key exchanged using asymmetric crypto (RSA/ECDHE)
AuthenticationServer certificate verified using Certificate Authorities (CA) chain of trust
IntegrityHMAC (Hash-based Message Authentication Code) ensures data is not tampered
TLS HandshakeClientHello → ServerHello + Certificate → Key Exchange → Finished. Adds 1-2 RTTs before data transfer.

4. FTP (File Transfer Protocol)

FTP is used for transferring files between a client and server. It uses two TCP connections: a control connection (port 21) and a data connection (port 20 or dynamic).

FeatureActive ModePassive Mode
Data Connection Initiated ByServer (from port 20 to client)Client (to server's specified port)
Firewall Friendly?No (server initiates connection to client)Yes (client initiates both connections)
Client ActionSends PORT command with its IP and portSends PASV command; server responds with port
FTP is stateful (unlike HTTP). The control connection persists across multiple file transfers. FTP transmits credentials in plaintext; SFTP (SSH-based) or FTPS (FTP over TLS) should be used for security.

5. SMTP and Email Architecture

5.1 SMTP (Simple Mail Transfer Protocol)

SMTP is used for sending email. It runs on TCP port 25 (or 587 for submission). SMTP is a push protocol; it transfers mail from sender's MTA to receiver's MTA.

5.2 Email Delivery Flow

6. Summary and GATE Key Points

Key Points for GATE:
  • DNS uses UDP port 53 (TCP for zone transfers). Resolution is typically recursive from client to local DNS, iterative from local DNS to root/TLD/authoritative.
  • DNS caching at each level reduces query load. TTL (Time To Live) controls cache duration.
  • HTTP is stateless. Non-persistent HTTP needs 2 RTT per object; persistent HTTP reuses the TCP connection.
  • HTTP/1.1 default is persistent connections with pipelining support.
  • HTTPS = HTTP + TLS. TLS handshake adds 1-2 RTTs of latency.
  • FTP uses 2 connections: control (port 21, persistent) and data (port 20 or dynamic, per-transfer). Passive mode is firewall-friendly.
  • SMTP is a push protocol (port 25); POP3 (port 110) and IMAP (port 143) are pull protocols for mail retrieval.
  • IMAP keeps mail on server (better for multiple devices); POP3 downloads to client.
  • MX record in DNS determines which mail server handles email for a domain.
  • Key ports: HTTP=80, HTTPS=443, FTP=21/20, SSH=22, DNS=53, SMTP=25, POP3=110, IMAP=143, DHCP=67/68.

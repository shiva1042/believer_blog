1. Arrays

1.1 One-Dimensional Arrays

An array is a contiguous block of memory storing elements of the same type. Access is O(1) via index.

int arr[5] = {10, 20, 30, 40, 50};
// Address of arr[i] = Base + i * sizeof(element)
// arr[3] address = 1000 + 3*4 = 1012 (if base=1000, sizeof(int)=4)

1.2 Two-Dimensional Arrays

A 2D array A[M][N] can be stored in two ways:

Row-Major Order (C/C++)

Elements of each row are stored contiguously.

Address(A[i][j]) = Base + (i * N + j) * sizeof(element)

Column-Major Order (Fortran)

Elements of each column are stored contiguously.

Address(A[i][j]) = Base + (j * M + i) * sizeof(element)

1.3 Address Calculation Example

Given: A[5][4] stored in row-major, base address = 2000, element size = 2 bytes. Find address of A[3][2].

Address = 2000 + (3*4 + 2) * 2 = 2000 + 14*2 = 2028

1.4 General Formula (Non-zero Lower Bounds)

For A[lr..ur][lc..uc] where lr, lc are lower bounds:

Row-major: Base + ((i - lr) * (uc - lc + 1) + (j - lc)) * size

Column-major: Base + ((j - lc) * (ur - lr + 1) + (i - lr)) * size

OperationTime Complexity
Access by indexO(1)
Search (unsorted)O(n)
Search (sorted)O(log n)
Insert at endO(1)
Insert at positionO(n)
Delete at positionO(n)

2.3 Array-Based Stack Implementation

#define MAX 100
int stack[MAX];
int top = -1;

void push(int x) {
    if (top == MAX - 1) { printf("Overflow\n"); return; }
    stack[++top] = x;
}

int pop() {
    if (top == -1) { printf("Underflow\n"); return -1; }
    return stack[top--];
}

int peek() {
    if (top == -1) return -1;
    return stack[top];
}

2.4 Application: Parenthesis Matching

bool isBalanced(char *expr) {
    Stack s;
    for (int i = 0; expr[i]; i++) {
        if (expr[i] == '(' || expr[i] == '{' || expr[i] == '[')
            push(&s, expr[i]);
        else if (expr[i] == ')' || expr[i] == '}' || expr[i] == ']') {
            if (isEmpty(&s)) return false;
            char top = pop(&s);
            if (!matches(top, expr[i])) return false;
        }
    }
    return isEmpty(&s);
}

2.5 Application: Infix to Postfix Conversion

Algorithm:

  1. Scan expression left to right
  2. If operand, add to output
  3. If operator, pop operators with higher or equal precedence from stack to output, then push current operator
  4. If '(', push to stack
  5. If ')', pop to output until '(' is found
  6. Pop remaining operators to output

Example: A + B * C - D

TokenStackOutput
AA
++A
B+A B
*+ *A B
C+ *A B C
--A B C * +
D-A B C * + D
EndA B C * + D -

2.6 Application: Postfix Evaluation

Scan left to right. Push operands. On operator, pop two operands, compute, push result.

Example: 2 3 4 * + 5 - = 2 + 12 - 5 = 9

3.3 Circular Queue

A circular queue uses a fixed-size array with wrap-around to avoid wasting space.

#define MAX 100
int queue[MAX];
int front = -1, rear = -1;

void enqueue(int x) {
    if ((rear + 1) % MAX == front) { printf("Full\n"); return; }
    if (front == -1) front = 0;
    rear = (rear + 1) % MAX;
    queue[rear] = x;
}

int dequeue() {
    if (front == -1) { printf("Empty\n"); return -1; }
    int val = queue[front];
    if (front == rear) front = rear = -1;  // queue becomes empty
    else front = (front + 1) % MAX;
    return val;
}

Circular Queue Capacity: A circular queue of size N can hold at most N-1 elements (one slot is wasted to distinguish full from empty). Alternatively, maintain a count variable to use all N slots.

3.4 Deque (Double-Ended Queue)

Supports insertion and deletion at both ends.

4. Stack and Queue Variants

4.1 Two Stacks in One Array

Stack 1 grows from left (index 0), Stack 2 grows from right (index N-1). Full when tops meet.

4.2 Queue Using Two Stacks

// Enqueue: push to stack1
// Dequeue: if stack2 empty, pop all from stack1 to stack2
//          then pop from stack2
// Amortized O(1) per operation

4.3 Stack Using Two Queues

// Push: enqueue to q1
// Pop: dequeue all except last from q1 to q2
//      dequeue last from q1 (this is the result)
//      swap q1 and q2
// O(n) per pop operation

5. Number of Possible BSTs / Stack Permutations

The number of distinct permutations obtainable by passing elements 1..n through a stack equals the n-th Catalan number:

Cn = (2n)! / ((n+1)! × n!) = C(2n,n) / (n+1)

6. GATE Exam Tips

  • For 2D array address questions, identify row-major vs column-major and lower bounds.
  • Circular queue with array size N holds N-1 elements (unless using a count variable).
  • Number of valid stack permutations = Catalan number.
  • To check if a permutation is achievable via stack: no element should come before a smaller element that was pushed after a larger element still in the stack.
  • Postfix/Prefix evaluation and infix conversion are frequently tested.
  • Queue using two stacks: amortized O(1). Stack using two queues: O(n) per pop.

1. Search Algorithms Overview

Search algorithms are the backbone of AI problem solving. They explore a state space to find a path from an initial state to a goal state. Understanding their properties is critical for GATE.

1.1 Breadth-First Search (BFS)

BFS explores all nodes at the current depth before moving to the next level. It uses a FIFO queue.

  • Complete: Yes (if branching factor b is finite)
  • Optimal: Yes (if all step costs are equal)
  • Time Complexity: O(bd)
  • Space Complexity: O(bd) — stores all nodes at current level

1.2 Depth-First Search (DFS)

DFS explores as deep as possible before backtracking. It uses a LIFO stack (or recursion).

  • Complete: No (can get stuck in infinite paths; yes if state space is finite)
  • Optimal: No
  • Time Complexity: O(bm) where m is max depth
  • Space Complexity: O(bm) — linear in depth

1.3 Iterative Deepening Depth-First Search (IDDFS)

IDDFS combines BFS completeness with DFS space efficiency. It runs DFS with increasing depth limits.

  • Complete: Yes
  • Optimal: Yes (uniform cost steps)
  • Time Complexity: O(bd)
  • Space Complexity: O(bd) — same as DFS
GATE Tip: IDDFS is the preferred uninformed search when space is limited and the solution depth is unknown. The overhead of re-expanding nodes is only O(b/(b-1)) times more than BFS.

1.4 A* Search

A* uses an evaluation function f(n) = g(n) + h(n) where g(n) is the path cost and h(n) is the heuristic estimate to the goal.

  • Complete: Yes (if step costs ≥ some positive ε)
  • Optimal: Yes (if h(n) is admissible; for graph search, h must also be consistent)
  • Time Complexity: O(bd) worst case
  • Space Complexity: O(bd) — stores all generated nodes

1.5 Hill Climbing

Hill climbing is a local search algorithm that greedily moves toward neighbors with better heuristic values. It does not maintain a search tree.

  • Can get stuck at local maxima, plateaus, and ridges
  • Variants: steepest-ascent, stochastic, random-restart

1.6 Simulated Annealing

Simulated annealing allows "bad" moves with a probability that decreases over time (controlled by temperature T). This helps escape local optima.

Probability of accepting worse move:
  P = e^(-(delta_E / T))

where delta_E = cost(new) - cost(current) > 0
As T decreases, probability of accepting bad moves approaches 0.

3. Heuristic Functions

3.1 Admissibility

A heuristic h(n) is admissible if it never overestimates the true cost to reach the goal:

h(n) ≤ h*(n) for all n
where h*(n) is the true optimal cost from n to the goal.

An admissible heuristic guarantees A* finds the optimal solution in tree search.

3.2 Consistency (Monotonicity)

A heuristic is consistent if for every node n and successor n':

h(n) ≤ c(n, a, n') + h(n')   (triangle inequality)

Consistency implies admissibility. A consistent heuristic guarantees A* optimality in graph search (with closed list).

3.3 Dominance

Heuristic h2 dominates h1 if h2(n) ≥ h1(n) for all n. A dominant heuristic expands fewer nodes while remaining optimal (if both are admissible).

Example: For the 8-puzzle:
h1 = number of misplaced tiles (admissible)
h2 = sum of Manhattan distances (admissible)
h2 dominates h1 because h2(n) ≥ h1(n) always.

5. Game Playing: Minimax Algorithm

Minimax is used for adversarial search in two-player zero-sum games. MAX tries to maximize the utility; MIN tries to minimize it.

5.1 Minimax Procedure

function MINIMAX(node, depth, isMaximizing):
    if depth == 0 or node is terminal:
        return heuristic value of node

    if isMaximizing:
        bestVal = -infinity
        for each child of node:
            val = MINIMAX(child, depth-1, false)
            bestVal = max(bestVal, val)
        return bestVal
    else:
        bestVal = +infinity
        for each child of node:
            val = MINIMAX(child, depth-1, true)
            bestVal = min(bestVal, val)
        return bestVal

6. Alpha-Beta Pruning

Alpha-beta pruning optimizes minimax by eliminating branches that cannot influence the final decision.

  • α = best value MAX can guarantee (initialized to -∞)
  • β = best value MIN can guarantee (initialized to +∞)
  • Prune when: α ≥ β

6.1 Alpha-Beta Example

         MAX (alpha=-inf, beta=+inf)
        /     \
     MIN       MIN
    /   \     /   \
   3     5   2     9

Step 1: Evaluate left MIN node:
  - See 3: MIN updates beta=3
  - See 5: min(3,5)=3 -> MIN returns 3
  - MAX updates alpha=3

Step 2: Evaluate right MIN node:
  - See 2: MIN updates beta=2
  - Now alpha(3) >= beta(2) -> PRUNE! Don't evaluate 9
  - MIN returns 2

Step 3: MAX chooses max(3, 2) = 3
Key Insight: With perfect ordering, alpha-beta pruning can reduce the effective branching factor from b to sqrt(b), allowing search to go twice as deep in the same time.

8. Common GATE Questions Pattern

Q: In A* search, how many nodes are expanded?
Approach: Build the search tree, compute f(n) = g(n) + h(n) for each node, and expand nodes in order of increasing f-value. Count the nodes removed from the priority queue (expanded), not those merely generated.
Q: Which nodes are pruned by alpha-beta?
Approach: Trace the minimax tree left-to-right, tracking alpha and beta values at each node. When alpha ≥ beta, mark remaining children as pruned.
Common Mistake: Confusing "generated" vs "expanded" nodes. A node is generated when created, but expanded when its successors are generated. BFS time complexity counts expanded nodes.

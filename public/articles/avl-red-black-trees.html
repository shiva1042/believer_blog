AVL Trees & Red-Black Trees

💻 Data Structures

1. AVL Trees

An AVL tree is a self-balancing Binary Search Tree where the balance factor of every node is -1, 0, or +1.

Balance Factor (BF) = Height(Left Subtree) - Height(Right Subtree)

1.1 AVL Tree Properties

  • All BST properties hold (left < root < right for each node).
  • For every node, |BF| ≤ 1.
  • Height of AVL tree with n nodes: O(log n). More precisely, h < 1.44 * log2(n+2).
  • Minimum nodes for height h: N(h) = N(h-1) + N(h-2) + 1, with N(0)=1, N(1)=2. (Fibonacci-like)

1.2 Rotations

When an insertion or deletion causes the balance factor of any node to become +2 or -2, we perform rotations to restore balance.

LL Rotation (Left-Left Case)

Imbalance at node A with BF = +2, and left child B has BF = +1. Perform Right Rotation at A.

graph TD subgraph Before A1[A bf=+2] --> B1[B bf=+1] A1 --> AR[Ar] B1 --> BL[Bl height h] B1 --> BR[Br height h-1] end
graph TD subgraph After Right Rotation B2[B bf=0] --> BL2[Bl] B2 --> A2[A bf=0] A2 --> BR2[Br] A2 --> AR2[Ar] end

RR Rotation (Right-Right Case)

Imbalance at node A with BF = -2, and right child B has BF = -1. Perform Left Rotation at A.

This is the mirror of LL rotation.

LR Rotation (Left-Right Case)

Imbalance at A with BF = +2, but left child B has BF = -1. Requires a double rotation: Left Rotate B, then Right Rotate A.

graph TD subgraph Before LR A3[A bf=+2] --> B3[B bf=-1] A3 --> AR3[Ar] B3 --> BL3[Bl] B3 --> C3[C] C3 --> CL3[Cl] C3 --> CR3[Cr] end

Step 1: Left rotate at B. Step 2: Right rotate at A. Result: C becomes the new root of this subtree.

RL Rotation (Right-Left Case)

Mirror of LR. Imbalance at A with BF = -2, right child B has BF = +1. Right Rotate B, then Left Rotate A.

1.4 AVL Insertion Example

Insert sequence: 10, 20, 30, 15, 25

Insert 10:  Tree = [10]
Insert 20:  Tree = 10 -> right: 20.  Balanced.
Insert 30:  10(bf=-2) -> 20 -> 30.  RR case at 10.
  Left rotate at 10: root becomes 20, left=10, right=30.
Insert 15:  20 -> 10(right=15), 30.  All balanced.
Insert 25:  20 -> 10(right=15), 30(left=25).
  30 has bf=+1, 20 has bf=0. Balanced.

1.5 AVL Deletion

  1. Perform standard BST deletion.
  2. Walk up from the deleted node to the root, checking balance factors.
  3. If any node becomes unbalanced (BF = +2 or -2), perform the appropriate rotation.
  4. Unlike insertion, deletion may require multiple rotations (up to O(log n)).

2. Red-Black Trees

A Red-Black tree is a self-balancing BST where each node has a color (red or black) and the tree satisfies specific properties.

2.1 Red-Black Tree Properties

  1. Every node is either Red or Black.
  2. The root is always Black.
  3. Every NULL leaf (NIL) is Black.
  4. No two consecutive Red nodes (a Red node cannot have a Red parent or Red child).
  5. Every path from a node to its descendant NIL leaves has the same number of Black nodes (Black-height).

2.2 Height Bounds

  • Black-height of root = bh. Then tree has at least 2^bh - 1 internal nodes.
  • Height h ≤ 2 * log2(n + 1). So worst case is roughly 2 * log(n), compared to 1.44 * log(n) for AVL.

2.3 Red-Black Insertion

New nodes are always inserted as Red. Then fix violations:

CaseConditionAction
Case 1Uncle is RedRecolor parent and uncle to Black, grandparent to Red. Recurse on grandparent.
Case 2Uncle is Black, node is inner child (LR or RL)Rotate node to make it outer child (converts to Case 3).
Case 3Uncle is Black, node is outer child (LL or RR)Rotate grandparent, swap colors of parent and grandparent.
flowchart TD A[Insert new Red node] --> B{Parent is Black?} B -- Yes --> C[Done - no violation] B -- No --> D{Uncle is Red?} D -- Yes --> E[Recolor: parent and uncle Black, grandparent Red] E --> F[Recurse on grandparent] D -- No --> G{Inner child?} G -- Yes --> H[Rotate to make outer child] H --> I[Rotate grandparent + recolor] G -- No --> I I --> J[Done]

2.4 Red-Black Insertion Example

Insert sequence: 10, 20, 30

Insert 10: Root, color Black.  Tree: 10(B)
Insert 20: Red child of 10.   Tree: 10(B) -> right: 20(R).  OK.
Insert 30: 20(R) -> right: 30(R).  Violation: consecutive reds.
  Uncle of 30 (left child of 10) is NIL (Black).
  30 is outer child (RR case) -> Case 3.
  Left rotate at 10, swap colors: 20 becomes Black, 10 becomes Red.
  Tree: 20(B) -> left: 10(R), right: 30(R)

2.5 Red-Black Deletion

Deletion is more complex. The key idea: if a Black node is removed, the Black-height property is violated. Fix by:

  • If the sibling is Red: rotate and recolor to convert to a Black sibling case.
  • If sibling is Black with two Black children: recolor sibling Red, propagate up.
  • If sibling is Black with at least one Red child: rotate and recolor to fix.

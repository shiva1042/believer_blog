Algorithms

Backtracking and Branch & Bound

Based on CLRS concepts and standard algorithm design texts

Backtracking and Branch & Bound are two powerful algorithmic paradigms for solving combinatorial optimization and constraint satisfaction problems. Both explore a state space tree systematically, but they differ in how they prune unpromising branches. This article provides a comprehensive treatment of both techniques with detailed worked examples, essential for GATE CSE.

Part I: Backtracking

1. State Space Tree Concept

A state space tree is a tree that represents all possible states of a problem. Each node represents a partial solution, and edges represent decisions (choices). Leaf nodes represent complete solutions or dead ends. 

State Space Tree Structure:
  Root: Empty solution (no decisions made)
  Internal node: Partial solution (some decisions made)
  Leaf node: Complete solution or infeasible state
  Edge: A decision (e.g., "place queen in column 3")

1.1 Backtracking Strategy

Backtracking performs a depth-first search (DFS) of the state space tree with pruning. At each node, it checks whether the partial solution can possibly lead to a valid complete solution. If not, it backtracks — abandons this branch and returns to the parent node to try the next alternative. 

BACKTRACK(node):
  if node is a complete solution:
      record/output solution
      return
  for each child c of node:
      if is_promising(c):      // bounding function / pruning
          BACKTRACK(c)
      // else: prune this branch (do nothing)

1.2 Bounding Function (Pruning)

The bounding function (also called the pruning function or feasibility function) determines whether a partial solution can be extended to a valid complete solution. A good bounding function prunes large portions of the state space tree, dramatically reducing the search.

Key insight: Without pruning, backtracking degenerates into brute-force enumeration of all possibilities. The effectiveness of backtracking depends entirely on how much the bounding function prunes.

2. N-Queens Problem

Problem: Place N queens on an N×N chessboard so that no two queens attack each other (no two queens in the same row, column, or diagonal).

2.1 Formulation

Represent the solution as an array Q[1..N] where Q[i] = j means the queen in row i is placed in column j. Since each row must have exactly one queen, this eliminates row conflicts by construction.

Constraint checking: Queens at (i, Q[i]) and (k, Q[k]) conflict if:

  • Same column: Q[i] == Q[k]
  • Same diagonal: |i − k| == |Q[i] − Q[k]|
IS-SAFE(Q, row, col):
  for i = 1 to row - 1:
      if Q[i] == col:                      // same column
          return false
      if |i - row| == |Q[i] - col|:       // same diagonal
          return false
  return true

N-QUEENS(Q, row, N):
  if row > N:
      print Q          // found a valid placement
      return
  for col = 1 to N:
      if IS-SAFE(Q, row, col):
          Q[row] = col
          N-QUEENS(Q, row + 1, N)
          // backtrack: Q[row] is overwritten in next iteration

3. Graph Coloring Using Backtracking

Problem: Given an undirected graph G = (V, E) and m colors, assign colors to vertices such that no two adjacent vertices have the same color. 

GRAPH-COLORING(G, color[], v, m):
  if v > |V|:
      print color[]       // valid coloring found
      return true
  for c = 1 to m:
      if IS-SAFE-COLOR(G, color, v, c):
          color[v] = c
          if GRAPH-COLORING(G, color, v+1, m):
              return true
          color[v] = 0    // backtrack
  return false            // no valid color for this vertex

IS-SAFE-COLOR(G, color, v, c):
  for each neighbor u of v:
      if color[u] == c:
          return false
  return true

4. Subset Sum Using Backtracking

Problem: Given a set W = {w1, w2, ..., wn} of positive integers and a target sum S, find all subsets that sum to exactly S. 

SUBSET-SUM-BT(W, n, S, i, current_sum, remaining_sum, include[]):
  if current_sum == S:
      print include[]      // valid subset
      return
  if i > n:
      return
  if current_sum + remaining_sum < S:   // can't reach target
      return                              // pruning!
  if current_sum + W[i] > S:            // including W[i] exceeds target
      return                              // pruning!

  // Include W[i]
  include[i] = 1
  SUBSET-SUM-BT(W, n, S, i+1, current_sum + W[i],
                remaining_sum - W[i], include)

  // Exclude W[i]
  include[i] = 0
  SUBSET-SUM-BT(W, n, S, i+1, current_sum,
                remaining_sum - W[i], include)

5. Hamiltonian Cycle Using Backtracking

HAM-CYCLE(G, path[], pos):
  if pos == |V|:
      if edge(path[pos-1], path[0]) exists:   // back to start?
          print path
          return true
      return false

  for v = 1 to |V|:
      if IS-SAFE-HAM(G, path, pos, v):
          path[pos] = v
          if HAM-CYCLE(G, path, pos + 1):
              return true
          path[pos] = -1    // backtrack
  return false

IS-SAFE-HAM(G, path, pos, v):
  if no edge between path[pos-1] and v:
      return false
  if v is already in path[0..pos-1]:
      return false
  return true

5.2 Time Complexity

Worst case: O(N!) — we try all permutations of vertices. Pruning helps in practice but does not change the worst-case complexity.

Part II: Branch and Bound

6. Branch and Bound Concept

Branch and Bound (B&B) is a systematic enumeration method for solving optimization problems. Like backtracking, it explores a state space tree, but it uses bounds (lower or upper) to prune subtrees that cannot contain the optimal solution.

6.1 Key Differences from Backtracking

Feature Backtracking Branch and Bound
Problem type Feasibility / all solutions Optimization (min/max)
Traversal Always DFS DFS, BFS, or Best-First (LC)
Pruning criterion Feasibility check Bound comparison with best known solution
Bound computation Simple constraint check Computes optimistic bound (lower for min, upper for max)
Exploration order Fixed (DFS) Flexible (can prioritize promising nodes)

6.2 Branch and Bound Strategies

Strategy Data Structure Node Selection Characteristics
FIFO B&B Queue First generated, first explored BFS-like; explores level by level
LIFO B&B Stack Last generated, first explored DFS-like; similar to backtracking
LC (Least Cost) B&B Priority Queue (Min-Heap) Node with best bound Most promising node first; typically most efficient 
BRANCH-AND-BOUND(root):
  best = ∞               // for minimization
  PQ = priority queue
  PQ.insert(root)

  while PQ is not empty:
      node = PQ.extract_best()
      if node.bound ≥ best:     // pruning
          continue                // this subtree cannot improve
      if node is a leaf (complete solution):
          if node.cost < best:
              best = node.cost
              best_solution = node.solution
      else:
          for each child c of node:
              compute c.bound
              if c.bound < best:
                  PQ.insert(c)

  return best_solution

7. 0/1 Knapsack Using Branch and Bound

Problem: Given n items with weights wi and profits pi, and a knapsack capacity W, maximize total profit without exceeding capacity. Each item is either included (1) or excluded (0).

7.1 Upper Bound Calculation

For a node at level i (items 1..i decided), the upper bound is computed using the fractional knapsack relaxation on the remaining items (greedy by profit/weight ratio): 

UPPER-BOUND(node):
  // node.profit = profit so far, node.weight = weight so far
  // Items i+1..n are undecided
  bound = node.profit
  remaining_capacity = W - node.weight
  j = node.level + 1

  // Sort remaining items by pi/wi ratio (done once in preprocessing)
  while j ≤ n and remaining_capacity > 0:
      if w[j] ≤ remaining_capacity:
          bound += p[j]
          remaining_capacity -= w[j]
      else:
          bound += p[j] * (remaining_capacity / w[j])  // fractional
          remaining_capacity = 0
      j++

  return bound

8. TSP Using Branch and Bound

Problem: Find the minimum-cost Hamiltonian cycle in a complete weighted graph.

8.1 Cost Matrix Reduction

The key idea: reduce the cost matrix by subtracting the minimum value from each row and each column. The total amount subtracted is a lower bound on the optimal tour cost, because any tour must use at least one entry from each row and each column. 

REDUCE-MATRIX(M):
  total_reduction = 0

  // Row reduction
  for each row i:
      min_val = min of row i (ignoring ∞)
      subtract min_val from all entries in row i
      total_reduction += min_val

  // Column reduction
  for each column j:
      min_val = min of column j (ignoring ∞)
      subtract min_val from all entries in column j
      total_reduction += min_val

  return total_reduction

9. Job Assignment Problem

Problem: There are n workers and n jobs. The cost of assigning worker i to job j is given by cost[i][j]. Find the assignment that minimizes total cost, with each worker doing exactly one job and each job done by exactly one worker.

9.1 Lower Bound Computation

For each unassigned worker, add the minimum cost among unassigned jobs. This gives a lower bound on the remaining cost. 

LOWER-BOUND(cost, assignment, level):
  bound = cost_so_far
  for i = level to n:
      bound += min(cost[i][j]) for all unassigned jobs j
  return bound

11.1 Decision Guide (Text Version)

  1. Can you solve it greedily? Check for greedy choice property and optimal substructure. If yes, use Greedy. Examples: Activity Selection, Huffman Coding, MST (Kruskal/Prim).
  2. Does it have overlapping subproblems? If you see the same subproblems being solved repeatedly, use DP. Examples: 0/1 Knapsack, LCS, Matrix Chain Multiplication, Floyd-Warshall.
  3. Need all solutions or just feasibility? Use Backtracking. Examples: N-Queens (all solutions), Sudoku solver, Graph Coloring, Hamiltonian Cycle detection.
  4. Need the exact optimum for a hard problem? Use Branch and Bound. It is better than brute force because bounding prunes large portions of the search space. Examples: TSP, Job Assignment, 0/1 Knapsack (when DP table is too large).
GATE Tip: 0/1 Knapsack can be solved by DP (O(nW) pseudo-polynomial), Branch & Bound (exact, exponential worst case), or backtracking. GATE may ask you to compare approaches or trace through a specific method. Know all three for this problem.

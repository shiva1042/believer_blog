1. Introduction to Boolean Algebra

Boolean algebra is a branch of algebra that deals with binary variables and logic operations. It forms the foundation of digital circuit design and computer science. Variables take values from the set {0, 1}, and the primary operations are AND (·), OR (+), and NOT (').

2. Fundamental Laws of Boolean Algebra

2.1 Basic Postulates

LawAND FormOR Form
IdentityA · 1 = AA + 0 = A
NullA · 0 = 0A + 1 = 1
ComplementA · A' = 0A + A' = 1
IdempotentA · A = AA + A = A
Involution(A')' = A

2.2 Commutative Law

The order of operands does not affect the result:

  • A + B = B + A
  • A · B = B · A

2.3 Associative Law

Grouping of operands does not affect the result:

  • (A + B) + C = A + (B + C)
  • (A · B) · C = A · (B · C)

2.4 Distributive Law

  • A · (B + C) = A·B + A·C
  • A + (B · C) = (A + B) · (A + C)

2.5 Absorption Law

  • A + A·B = A
  • A · (A + B) = A

2.6 De Morgan's Theorems

Theorem 1: (A + B)' = A' · B'

Theorem 2: (A · B)' = A' + B'

These theorems generalize to n variables: the complement of any expression is obtained by swapping AND/OR and complementing each variable.

2.7 Consensus Theorem

AB + A'C + BC = AB + A'C (the term BC is redundant).

3. Boolean Expressions and Canonical Forms

3.1 Minterms and Maxterms

A minterm is a product term containing all variables in either complemented or uncomplemented form. A maxterm is a sum term with all variables.

3.2 Sum of Products (SOP)

A canonical SOP is an OR of minterms where the function evaluates to 1.

Example: F(A,B,C) = Σm(1,3,5,7) = A'B'C + A'BC + AB'C + ABC = C

3.3 Product of Sums (POS)

A canonical POS is an AND of maxterms where the function evaluates to 0.

Example: F(A,B,C) = ΠM(0,2,4,6) = (A+B+C)(A+B'+C)(A'+B+C)(A'+B'+C) = C

Relationship: If F = Σm(1,3,5,7), then F = ΠM(0,2,4,6). The minterm and maxterm sets are complements within the full set {0,...,2n-1}.

4. Karnaugh Map (K-map) Minimization

A K-map is a graphical method for simplifying Boolean expressions. Adjacent cells differ by exactly one variable (Gray code ordering).

4.1 Two-Variable K-map

Example: F(A,B) = Σm(0,1,3)

A \ B01
011
101

Groups: {m0,m1} gives A', {m1,m3} gives B. Result: F = A' + B

4.2 Three-Variable K-map

Example: F(A,B,C) = Σm(0,2,4,5,6)

A \ BC00011110

Groups: {m0,m2,m4,m6} = C', {m4,m5} = AB'. Result: F = C' + AB'

4.3 Four-Variable K-map

Example: F(A,B,C,D) = Σm(0,1,2,5,8,9,10)

AB \ CD00011110
001101
010100
110000
101101

Groups: {m0,m1,m8,m9} = B'C', {m0,m2,m8,m10} = B'D'. Result: F = B'C' + B'D' + A'B'C'D (with m5 covered separately as A'BC'D)

4.4 Don't Care Conditions

Don't care conditions (denoted by 'X' or 'd') represent input combinations that either never occur or whose output does not matter. In K-maps, don't cares can be treated as either 0 or 1 to form larger groups and achieve better minimization.

Key Rule: Don't cares should be included in groups only if they help form larger groups. They must not appear in the final expression as standalone terms.

4.5 K-map Minimization Process

5. Quine-McCluskey Method

The Quine-McCluskey (QM) method is a tabular method for minimizing Boolean functions, especially useful for functions with more than 4 variables where K-maps become impractical.

5.1 Algorithm Steps

  1. Step 1: List all minterms in binary and group them by the number of 1s.
  2. Step 2: Compare adjacent groups and combine minterms that differ in exactly one bit, replacing the differing bit with a dash (-).
  3. Step 3: Repeat Step 2 until no further combinations are possible. Unchecked terms are prime implicants.
  4. Step 4: Construct a prime implicant chart and identify essential prime implicants.
  5. Step 5: Select a minimum set of prime implicants that covers all minterms.

6. GATE Exam Tips

  • Always verify your K-map groupings wrap around edges and corners.
  • Groups must be powers of 2 (1, 2, 4, 8, 16).
  • When converting between SOP and POS, remember: SOP minterms + POS maxterms = all terms.
  • Quine-McCluskey is systematic but time-consuming; prefer K-maps for up to 4 variables in exams.
  • De Morgan's theorem is frequently tested both directly and as part of gate-level circuit simplification.

7. Practice Problems

  1. Simplify using K-map: F(A,B,C,D) = Σm(0,1,2,4,5,6,8,9,12,13,14)
  2. Express in canonical POS: F(A,B,C) = A'B + BC'
  3. Apply De Morgan's theorem: [(AB)' + C]' · D
  4. Use Quine-McCluskey to minimize: F(A,B,C,D) = Σm(2,6,8,9,10,11,14,15)

Theory of Computation

Closure Properties and Decision Problems

Closure properties and decision problems form the backbone of formal language theory and are among the most heavily tested topics in GATE CSE. Understanding which operations preserve which language classes and which questions are algorithmically answerable for each class gives you a powerful toolkit for reasoning about automata and languages. This article covers Chapters 4, 7, and 9 of Hopcroft, Motwani, and Ullman's Introduction to Automata Theory, Languages, and Computation.

1. Closure Properties — Overview

A class of languages C is said to be closed under an operation op if, whenever we apply op to languages in C, the result is also in C. Formally, for a unary operation: if L ∈ C, then op(L) ∈ C. For a binary operation: if L1, L2 ∈ C, then L1 op L2 ∈ C.

Why does this matter for GATE? Closure properties let you prove or disprove that a given language belongs to a certain class. If you know L is CFL and R is regular, then L ∩ R is CFL — a fact used in many proofs and GATE questions.

2. Closure Properties of Regular Languages

Regular languages enjoy closure under all commonly studied operations. This is one of the reasons they are so well-behaved.

2.1 Union

Theorem: If L1 and L2 are regular, then L1 ∪ L2 is regular.

Proof sketch (NFA construction): Given NFAs N1 and N2 for L1 and L2, create a new NFA with a new start state q0 that has ε-transitions to the start states of both N1 and N2. The accepting states are the union of the accepting states of both NFAs.

graph LR q0((q0)) -->|ε| s1((start N1)) q0 -->|ε| s2((start N2)) s1 -->|...| f1((accept N1)) s2 -->|...| f2((accept N2))

2.2 Intersection

Theorem: If L1 and L2 are regular, then L1 ∩ L2 is regular.

Proof (Product/Cross-product Construction): Given DFAs D1 = (Q1, Σ, δ1, q01, F1) and D2 = (Q2, Σ, δ2, q02, F2), construct DFA D = (Q1 × Q2, Σ, δ, (q01, q02), F1 × F2) where:

δ((p, q), a) = (δ1(p, a), δ2(q, a))

A string is accepted if and only if both components reach accepting states simultaneously, i.e., the final state is in F1 × F2.

Worked Example: Let L1 = strings over {0,1} with even number of 0s, and L2 = strings with odd number of 1s.

  • D1 has states {E0, O0} (even/odd count of 0s), start = E0, accept = {E0}.
  • D2 has states {E1, O1} (even/odd count of 1s), start = E1, accept = {O1}.
  • Product DFA has states: {(E0,E1), (E0,O1), (O0,E1), (O0,O1)}.
  • Start state: (E0, E1). Accept states: {(E0, O1)}.
StateInput 0Input 1
(E0, E1)(O0, E1)(E0, O1)
(E0, O1)(O0, O1)(E0, E1)
(O0, E1)(E0, E1)(O0, O1)
(O0, O1)(E0, O1)(O0, E1)
GATE Tip: The product construction is the most common proof technique tested. The number of states in the product DFA is |Q1| × |Q2|. For union, change the accept set to (F1 × Q2) ∪ (Q1 × F2).

2.3 Complement

Theorem: If L is regular, then Σ* − L (the complement) is regular.

Proof: Given a complete DFA D = (Q, Σ, δ, q0, F), construct D' = (Q, Σ, δ, q0, Q − F). Simply swap accepting and non-accepting states.

Critical detail: The DFA must be complete (every state has a transition for every symbol). If starting from an NFA, first convert to DFA, ensure completeness (add a dead state if needed), then complement. Complementing an NFA directly by flipping accept states does NOT work.

2.4 Concatenation

Theorem: If L1 and L2 are regular, then L1 · L2 is regular.

Proof: Given NFAs N1 and N2, add ε-transitions from every accepting state of N1 to the start state of N2. The start state of the new NFA is the start state of N1, and the accepting states are those of N2.

2.5 Kleene Star

Theorem: If L is regular, then L* is regular.

Proof: Given NFA N for L, create a new start state q0 (which is also accepting, to accept ε). Add ε-transitions from q0 to the old start state, and from every old accepting state back to the old start state.

2.6 Reversal

Theorem: If L is regular, then LR = {wR | w ∈ L} is regular.

Proof: Given DFA D for L:

  1. Reverse all transitions (if δ(p, a) = q, add transition from q to p on a).
  2. Make the old start state the (only) accepting state.
  3. Create a new start state with ε-transitions to all old accepting states.

The resulting NFA accepts LR.

2.7 Homomorphism

Definition: A homomorphism h: Σ → Δ* maps each symbol to a string. It extends to strings: h(a1a2...an) = h(a1)h(a2)...h(an). For a language: h(L) = {h(w) | w ∈ L}.

Theorem: Regular languages are closed under homomorphism.

Proof: Given a regular expression r for L, replace each symbol a in r with h(a). The result is a valid regular expression for h(L).

Example: Let L = {01, 10} and h(0) = ab, h(1) = ε. Then h(L) = {ab, ab} = {ab}.

2.8 Inverse Homomorphism

Definition: h-1(L) = {w ∈ Σ* | h(w) ∈ L}.

Theorem: Regular languages are closed under inverse homomorphism.

Proof: Given DFA D = (Q, Δ, δ, q0, F) for L, construct DFA D' = (Q, Σ, δ', q0, F) where δ'(q, a) = δ̂(q, h(a)). That is, on reading symbol a, we simulate reading the entire string h(a) in the original DFA.

Key insight: Inverse homomorphism preserves regularity even though the mapping is "backwards." The number of states does not change — only the transition function is modified.

3. Closure Properties of Context-Free Languages

Context-free languages are not as well-behaved as regular languages. Several key operations break closure.

3.1 Operations Under Which CFLs Are Closed

Union

Given CFGs G1 with start symbol S1 and G2 with start symbol S2 (with disjoint variable sets), create a new grammar with start symbol S → S1 | S2.

Concatenation

Create S → S1S2.

Kleene Star

Create S → SS1 | ε.

Reversal

Reverse the right-hand side of every production in the grammar. If A → α, create A → αR.

Homomorphism

Replace each terminal a in every production with h(a).

Inverse Homomorphism

CFLs are also closed under inverse homomorphism (proof uses PDA construction).

Substitution

If L is a CFL and for each a ∈ Σ, s(a) is a CFL, then s(L) = {s(w) | w ∈ L} is a CFL.

3.2 Operations Under Which CFLs Are NOT Closed

Intersection

CFLs are NOT closed under intersection.

Counterexample:

  • L1 = {anbncm | n, m ≥ 0} is context-free (match a's and b's, c's are free).
  • L2 = {ambncn | n, m ≥ 0} is context-free (a's are free, match b's and c's).
  • L1 ∩ L2 = {anbncn | n ≥ 0} is not context-free (by the pumping lemma).

Complement

CFLs are NOT closed under complement.

Proof by contradiction: If CFLs were closed under complement, then since L1 ∩ L2 = (L1c ∪ L2c)c (De Morgan's), and CFLs are closed under union, CFLs would be closed under intersection. But we just showed they are not. Contradiction.

GATE trap: Many students assume that because regular languages are closed under intersection and complement, CFLs must be too. This is false. Always check!

3.3 Intersection of a CFL with a Regular Language

Theorem: If L is a CFL and R is a regular language, then L ∩ R is a CFL.

Proof (Product Construction for PDA × DFA): Given PDA P for L and DFA D for R, construct a new PDA that simulates both in parallel. The state set is the cross product of states, and the stack operations come from the original PDA. A configuration is accepted if both the PDA component is in an accepting state and the DFA component is in an accepting state.

GATE favorite: This property is extremely useful for proving languages are not context-free. Strategy: assume L is CFL, intersect with a carefully chosen regular language R to get a known non-CFL, and reach a contradiction.

Example Application: Prove L = {anbncn | n ≥ 1} is not CFL.

  1. Suppose L is CFL.
  2. Let R = a*b*c* (regular).
  3. Then L ∩ R = L itself should be CFL.
  4. But L is not CFL (pumping lemma). Contradiction.

(In this case, the intersection trick is not needed, but the technique shines for more complex languages where direct pumping is hard.)

4. Closure Properties of Recursive and Recursively Enumerable Languages

4.1 Recursive (Decidable) Languages

A language is recursive (decidable) if there exists a Turing machine that halts on all inputs and accepts exactly the strings in the language.

OperationClosed?Proof Idea
UnionYesRun both TMs; accept if either accepts
IntersectionYesRun both TMs; accept if both accept
ComplementYesSwap accept/reject of the TM
ConcatenationYesNondeterministically guess split point; simulate both
Kleene StarYesNondeterministically guess all split points
HomomorphismYesApply inverse homomorphism to input, test membership
Inverse HomomorphismYesApply homomorphism to input, test membership
ReversalYesReverse input, run original TM
Key fact: Recursive languages are closed under ALL boolean operations (union, intersection, complement, difference). This follows because TMs that always halt can be composed freely.

4.2 Recursively Enumerable (RE) Languages

A language is RE if there exists a TM that accepts every string in the language (but may loop forever on strings not in the language).

OperationClosed?Proof Idea
UnionYesRun both TMs in parallel (dovetailing); accept if either accepts
IntersectionYesRun both TMs in parallel; accept if both accept
ComplementNoIf RE were closed under complement, RE ∩ co-RE = Recursive for all RE languages
ConcatenationYesNondeterministically guess split; simulate both
Kleene StarYesNondeterministically guess splits
HomomorphismYesStandard construction
Inverse HomomorphismYesStandard construction
ReversalYesReverse input tape, simulate
DifferenceNoL − M could yield complement if M = Σ*
Critical theorem: A language L is recursive (decidable) if and only if both L and Lc are recursively enumerable. That is: Recursive = RE ∩ co-RE.

4.3 co-RE Languages

A language is co-RE if its complement is RE. co-RE languages are closed under union, intersection, but NOT complement.

6. Decision Problems for Regular Languages

All standard decision problems for regular languages are decidable. This is because DFAs are finite objects that can be fully explored.

6.1 Membership Problem

Question: Given a DFA D and a string w, is w ∈ L(D)?

Algorithm: Simulate D on w. Accept if the final state is accepting.

Complexity: O(|w|) time, O(1) space (just track current state).

6.2 Emptiness Problem

Question: Is L(D) = ∅?

Algorithm: Perform BFS/DFS from the start state. If any accepting state is reachable, L(D) ≠ ∅.

Complexity: O(|Q| + |δ|) — linear in the size of the DFA.

function isEmpty(DFA D):
    visited = {q0}
    queue = [q0]
    while queue is not empty:
        state = queue.dequeue()
        if state in F:
            return false   // language is NOT empty
        for each symbol a in Sigma:
            next = delta(state, a)
            if next not in visited:
                visited.add(next)
                queue.enqueue(next)
    return true   // no accepting state reachable

6.3 Finiteness Problem

Question: Is L(D) finite?

Algorithm:

  1. Remove all states not reachable from the start state.
  2. Remove all states from which no accepting state is reachable.
  3. Check if the remaining transition graph has a cycle (using DFS).
  4. If there is a cycle, the language is infinite. Otherwise, it is finite.

Alternative (Pumping Lemma approach): A regular language L recognized by a DFA with n states is infinite if and only if it contains a string w with n ≤ |w| < 2n.

6.4 Equivalence Problem

Question: Given DFAs D1 and D2, is L(D1) = L(D2)?

Algorithm (Symmetric Difference):

L(D1) = L(D2)  iff  (L(D1) Δ L(D2)) = ∅
                iff  (L(D1) ∩ L(D2)^c) ∪ (L(D1)^c ∩ L(D2)) = ∅

Construct the symmetric difference automaton (using product construction) and test for emptiness.

Alternative (Table-filling / Myhill-Nerode): Minimize both DFAs and check if the minimal DFAs are isomorphic.

6.5 Subset (Containment) Problem

Question: Is L(D1) ⊆ L(D2)?

Algorithm: Check if L(D1) ∩ L(D2)c = ∅. Build the product DFA for the intersection and test emptiness.

6.6 Universality Problem

Question: Is L(D) = Σ*?

Algorithm: Check if L(D)c = ∅. Complement the DFA and test emptiness.

Summary: Regular Language Decision Problems

7. Decision Problems for Context-Free Languages

7.1 Membership Problem — CYK Algorithm

Question: Given CFG G and string w, is w ∈ L(G)?

Algorithm: The Cocke-Younger-Kasami (CYK) algorithm uses dynamic programming on a grammar in Chomsky Normal Form (CNF).

Steps:

  1. Convert G to CNF (all productions of the form A → BC or A → a).
  2. Build a triangular table T[i][j] = set of variables that derive the substring w[i..j].
  3. Base case: T[i][i] = {A | A → w[i] is a production}.
  4. Inductive step: A ∈ T[i][j] if there exists k with i ≤ k < j and production A → BC where B ∈ T[i][k] and C ∈ T[k+1][j].
  5. Accept if S ∈ T[1][n].

Complexity: O(n3 · |G|) where n = |w|.

Worked Example: Let G have productions:

S → AB | BC
A → BA | a
B → CC | b
C → AB | a

Parse the string w = baaba:

b (1)a (2)a (3)b (4)a (5)
1{B}{S,A}{}{S,A,C}{S,A,C}
2{A,C}{S,C,B}{B}{S,A,C}
3{A,C}{S,A}{B}
4{B}{S,A,C}
5{A,C}

Since S ∈ T[1][5], the string baaba is in the language.

7.2 Emptiness Problem

Question: Is L(G) = ∅?

Algorithm: Mark terminals as "generating." Repeatedly mark any variable A as generating if there is a production A → α where all symbols in α are generating. The language is non-empty iff the start symbol S is marked generating.

Complexity: O(|G|).

7.3 Finiteness Problem

Question: Is L(G) finite?

Algorithm:

  1. Remove useless symbols (non-generating and non-reachable).
  2. Remove unit productions and ε-productions.
  3. Build a dependency graph among variables (edge from A to B if B appears in a production for A).
  4. The language is infinite iff this graph has a cycle.

8. Decision Problems for Recursive and RE Languages

8.1 The Halting Problem

Problem: Given a Turing machine M and input w, does M halt on w?

Result: Undecidable (proved by Alan Turing via diagonalization).

Proof sketch:

  1. Assume there exists a TM H that decides the halting problem: H(<M>, w) accepts if M halts on w, rejects otherwise.
  2. Construct TM D: on input <M>, run H(<M>, <M>). If H accepts, loop forever. If H rejects, accept.
  3. Consider D(<D>):
    • If D halts on <D>, then H accepts, so D loops. Contradiction.
    • If D doesn't halt on <D>, then H rejects, so D accepts (halts). Contradiction.
  4. Therefore, H cannot exist.

The halting problem is RE (we can simulate and accept if the machine halts) but not recursive.

8.2 Rice's Theorem

Theorem: Every non-trivial property of the language of a Turing machine is undecidable.

Formal statement: Let P be a property of RE languages (a set of RE languages). If P is non-trivial (i.e., some RE languages have property P and some do not), then the language LP = {<M> | L(M) ∈ P} is undecidable.

Examples of properties covered by Rice's theorem:

  • "Is L(M) empty?" — undecidable
  • "Is L(M) finite?" — undecidable
  • "Is L(M) = Σ*?" — undecidable
  • "Is L(M) regular?" — undecidable
  • "Is L(M) context-free?" — undecidable
  • "Does L(M) contain the string 01?" — undecidable
What Rice's theorem does NOT cover: Properties of the Turing machine itself (not its language). For example, "Does M have exactly 5 states?" is decidable — just examine the description of M. Rice's theorem applies only to semantic properties (properties of L(M)), not syntactic properties (properties of M's description).

10. Common GATE Tricks and Traps

Trick 1: Using Closure to Prove a Language Is Not in a Class

If you suspect L is not CFL, try to show that intersecting L with a regular language gives a known non-CFL.

Trick 2: Complement Confusion

Students often assume: "If L is CFL, then Lc is CFL." This is FALSE. CFLs are not closed under complement.

Trick 3: The "Intersection of Two CFLs" Trap

A GATE question might give you two CFLs and ask about their intersection. Remember: the intersection of two CFLs is NOT necessarily a CFL. But it is always recursively enumerable (in fact, recursive if the CFLs are given by CFGs).

Trick 4: Confusing Machine Properties with Language Properties

Rice's theorem says that non-trivial language properties of TMs are undecidable. But properties of the machine description (like "does M have 100 states?") ARE decidable.

Trick 5: DCFL Equivalence Is Decidable

One of the deep results in formal language theory (proved by Senizergues in 2001): equivalence of DPDAs is decidable. This is often a surprising fact tested in GATE.

Trick 6: Membership in RE vs Decidability

Membership in RE means "given w, is w in L?" — this is the halting problem in disguise. For a specific RE language given by a TM, you can enumerate but not decide.

Trick 7: Product Construction State Count

If DFA1 has m states and DFA2 has n states, the product DFA has at most m × n states. The minimal DFA for the intersection could have fewer, but the upper bound is m × n.

Trick 8: Empty and Sigma-star

Both ∅ and Σ* are regular languages. Any class containing all regular languages has these as members. This helps in many closure arguments.

11. Worked GATE-Style Examples

Example 1: Closure Argument

Q: Let L1 = {ambncn | m, n ≥ 0} and L2 = {anbncm | m, n ≥ 0}. What is the type of L1 ∩ L2?

Solution:

  • L1 is CFL (PDA matches b's and c's, ignores a's).
  • L2 is CFL (PDA matches a's and b's, ignores c's).
  • L1 ∩ L2 = {anbncn | n ≥ 0}, which is NOT context-free.
  • It IS context-sensitive (an LBA can check it), and recursive.

Answer: CSL (context-sensitive) / Recursive, but not CFL.

Example 2: Decision Problem

Q: Which of the following is decidable?

  1. Whether a given CFG is ambiguous
  2. Whether the language of a given CFG is regular
  3. Whether the language of a given CFG is empty
  4. Whether two CFGs generate the same language

Solution: (A) Ambiguity of CFG — undecidable. (B) Regularity of CFL — undecidable. (C) Emptiness of CFL — decidable (mark generating symbols). (D) Equivalence — undecidable.

Answer: C

Example 3: Rice's Theorem Application

Q: Is the following problem decidable? "Given a TM M, does L(M) contain at least 10 strings?"

Solution: This is a non-trivial property of L(M): some RE languages have 10+ strings (e.g., Σ*), some don't (e.g., ∅). By Rice's theorem, it is undecidable.

Example 4: Applying Inverse Homomorphism

Q: Let h(a) = 01, h(b) = 10, and L = {0110, 0101}. Find h-1(L).

Solution: We need all strings w ∈ {a,b}* such that h(w) ∈ L.

  • h(ab) = 0110 ∈ L. So ab ∈ h-1(L).
  • h(aa) = 0101 ∈ L. So aa ∈ h-1(L).
  • h(ba) = 1001 ∉ L.
  • h(bb) = 1010 ∉ L.

Answer: h-1(L) = {ab, aa}.

1. Overview of Code Optimization

Code optimization transforms intermediate or target code to improve performance (execution time, memory usage) while preserving the program's semantics. Optimization never changes what the program computes — only how efficiently it computes it.

2. Basic Blocks and Control Flow Graphs

A basic block is a maximal sequence of consecutive instructions with:

  • One entry point (no instruction within is a target of a jump, except the first).
  • One exit point (no instruction within is a jump, except possibly the last).

Constructing Basic Blocks

  1. Identify leaders: first instruction, target of any jump, instruction following a jump.
  2. Each leader starts a basic block that continues until the next leader.
Example:
  (1) i = 1               --> Block B1 (leader: first instruction)
  (2) if i > n goto (8)
  (3) t1 = a[i]           --> Block B2 (leader: target after conditional)
  (4) t2 = t1 * t1
  (5) sum = sum + t2
  (6) i = i + 1
  (7) goto (2)
  (8) ...                  --> Block B3 (leader: target of goto)

A Control Flow Graph (CFG) has basic blocks as nodes and edges representing possible flow of control between blocks.

3. Local Optimization Techniques

3.1 Constant Folding

Evaluate constant expressions at compile time rather than at runtime.

Before:  x = 2 + 3          After:  x = 5
Before:  y = x * 0          After:  y = 0
Before:  z = x * 1          After:  z = x

3.2 Constant Propagation

If a variable has a known constant value at a program point, replace uses of that variable with the constant.

Before:                  After:
  x = 5                   x = 5
  y = x + 3               y = 5 + 3    (then fold to y = 8)
  z = x * y               z = 5 * 8    (then fold to z = 40)

3.3 Strength Reduction

Replace expensive operations with cheaper equivalent ones.

BeforeAfterSavings
x = y * 2x = y + y (or x = y << 1)Multiply to add/shift
x = y * 8x = y << 3Multiply to shift
x = y / 4x = y >> 2Divide to shift
x = y ** 2x = y * yExponent to multiply

3.4 Dead Code Elimination

Remove code whose results are never used (dead definitions).

Before:                  After:
  x = a + b               (removed — x never used)
  y = c * d               y = c * d
  z = y + 1               z = y + 1

3.5 Algebraic Simplification and Identity

PatternSimplification

x - x0

x ** 1x

3.6 Copy Propagation

After a copy statement x = y, replace subsequent uses of x with y (if x is not redefined).

Before:                  After:
  x = y                   x = y           (may be eliminated as dead code)
  z = x + 1               z = y + 1

4. Common Subexpression Elimination (CSE)

If an expression E has been previously computed and the values of its operands have not changed, reuse the stored result instead of recomputing E.

Before:                  After:
  t1 = a + b               t1 = a + b
  t2 = a + b               t2 = t1        (reuse)
  t3 = t1 * t2             t3 = t1 * t1

Local CSE

Within a single basic block, use a DAG (Directed Acyclic Graph) representation. Nodes represent operations; shared subexpressions share nodes.

Global CSE

Across basic blocks, use available expressions analysis (a data flow analysis) to determine which expressions are available at each point.

5. Data Flow Analysis Framework

Data flow analysis computes information about the flow of data through a program's CFG. It propagates facts along edges using transfer functions.

General Framework

ComponentDescription
DomainSet of possible data flow values (e.g., sets of expressions, variables)
DirectionForward (entry to exit) or backward (exit to entry)
Transfer FunctionHow a basic block transforms data flow values: OUT[B] = f(IN[B])
Meet OperatorHow to combine values at confluence points (union or intersection)
Boundary ConditionInitial value at entry (forward) or exit (backward) of CFG

Iterative Algorithm

Initialize all blocks
repeat:
  for each block B:
    IN[B] = meet of OUT[predecessors of B]     (forward analysis)
    OUT[B] = transfer_function(IN[B])
until no changes occur (fixed point reached)

6. Common Data Flow Analyses

6.1 Reaching Definitions

A definition d: x = ... reaches a point p if there is a path from d to p on which x is not redefined.

PropertyValue
DirectionForward
DomainSets of definitions
Meet operatorUnion (any path)
Transfer functionOUT[B] = GEN[B] union (IN[B] - KILL[B])
InitializationOUT[entry] = empty set

6.2 Available Expressions

An expression e is available at a point p if on every path from entry to p, e has been computed and none of its operands has been redefined since.

PropertyValue
DirectionForward
DomainSets of expressions
Meet operatorIntersection (all paths)
Transfer functionOUT[B] = GEN[B] union (IN[B] - KILL[B])
InitializationOUT[entry] = empty set; others = universal set

6.3 Live Variable Analysis

A variable v is live at a point p if there exists a path from p to a use of v along which v is not redefined.

PropertyValue
DirectionBackward
DomainSets of variables
Meet operatorUnion
Transfer functionIN[B] = USE[B] union (OUT[B] - DEF[B])
InitializationIN[exit] = empty set
Applications of Liveness: Dead code elimination (if a variable is not live after its definition, the definition is dead), register allocation (live variables need registers; non-live variables do not).

6.4 Use-Def and Def-Use Chains

  • Use-Def Chain (ud-chain): For each use of a variable, links to all definitions that may reach that use.
  • Def-Use Chain (du-chain): For each definition of a variable, links to all uses that it may reach.

These chains are built from reaching definitions analysis and are fundamental to many optimizations.

7. Loop Optimizations

7.1 Loop-Invariant Code Motion

Move computations whose values do not change within the loop to before the loop.

Before:                    After:
  while (i < n) {           t = n * 2;
    x = n * 2;              while (i < n) {
    a[i] = x + i;             a[i] = t + i;
    i++;                       i++;
  }                          }

7.2 Induction Variable Elimination

Replace induction variables (variables that change by a constant amount each iteration) with simpler forms.

Before:                    After:
  j = 0;                     t = &a[0];
  while (j < n) {           while (t < &a[n]) {
    a[j] = 0;                 *t = 0;
    j++;                       t += sizeof(a[0]);
  }                          }

7.3 Loop Unrolling

Replicate the loop body multiple times to reduce loop overhead (fewer comparisons and branches).

9. Key GATE Takeaways

  • Optimization preserves program semantics while improving efficiency.
  • Constant folding evaluates compile-time constants. Constant propagation replaces variables with known values.
  • Dead code elimination removes definitions whose values are never used (requires liveness analysis).
  • Common subexpression elimination avoids redundant computation (requires available expressions).
  • Data flow analysis uses iterative fixed-point algorithms on the CFG.
  • Reaching definitions is forward/union; available expressions is forward/intersection; live variables is backward/union.
  • Loop optimizations (code motion, induction variable elimination) provide the largest speedups in practice.
  • Know how to compute GEN, KILL, IN, and OUT sets for basic blocks.

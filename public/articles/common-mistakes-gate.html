Common Mistakes in GATE CS & How to Avoid Them

📋 GATE Essentials

This article catalogs the most frequent mistakes students make in each GATE CS section, with clear "Mistake to Correct Approach" tables and examples.

1. Engineering Mathematics Mistakes

MistakeCorrect Approach
Confusing P(A|B) with P(B|A) in Bayes' theoremAlways write out the full formula. Label what is given and what is asked.
Forgetting to subtract overcounted elements in inclusion-exclusion|A ∪ B ∪ C| = |A|+|B|+|C| - |A∩B| - |B∩C| - |A∩C| + |A∩B∩C|
Confusing eigenvalue of A with eigenvalue of A^(-1)If λ is eigenvalue of A, then 1/λ is eigenvalue of A^(-1)
Wrong determinant sign when expanding along a rowSign pattern: (+, -, +, -, ...). Use cofactor expansion carefully.
Treating dependent and independent events incorrectlyIndependent: P(A∩B) = P(A)*P(B). Mutually exclusive: P(A∩B) = 0. These are different concepts.
Wrong limits of integration for continuous distributionsFor exponential distribution, always integrate from 0 to infinity (not -infinity).

2. Digital Logic Mistakes

MistakeCorrect Approach
2's complement overflow: adding two positive numbers getting negativeOverflow occurs when sign of both operands is same but result sign differs. Check the carry into and out of MSB.
K-map grouping: not making largest possible groupsAlways make groups of largest power of 2. Groups can wrap around edges.
Confusing active-high with active-low signalsActive-low means the signal is asserted when 0. Check the bubble notation.
Wrong IEEE 754 bias valueSingle precision: bias = 127. Double precision: bias = 1023. Exponent_stored = Exponent_actual + bias.
Treating don't-care conditions as 0 in K-mapsDon't-cares can be treated as 0 or 1 to make larger groups. Use them to minimize.

3. Data Structures Mistakes

MistakeCorrect Approach
Confusing height vs depth of a treeHeight of a node = longest path to a leaf. Depth = distance from root. Height of tree = height of root.
Assuming BST search is always O(log n)Only balanced BST is O(log n). Skewed BST is O(n). Specify "balanced" when answering.
Wrong in-order successor for BST deletionIn-order successor is the leftmost node in the right subtree, NOT just the right child.
Off-by-one errors in array indexing for heapsIf root at index 1: left child = 2i, right child = 2i+1. If root at index 0: left = 2i+1, right = 2i+2.
Confusing stack and queue behavior in BFS/DFSBFS uses Queue (FIFO). DFS uses Stack (LIFO). Mixing them changes the traversal completely.
Hash table: forgetting to count probes correctlyIn linear probing, count the initial hash check as the first probe.

4. Algorithms Mistakes

MistakeCorrect Approach
Applying Master theorem when it does not applyMaster theorem requires T(n) = aT(n/b) + f(n). It does NOT work for T(n) = T(n-1) + n or unequal subproblems.
Confusing O, Ω, and Θ notationsO = upper bound, Ω = lower bound, Θ = tight bound. "Running time is O(n^2)" means at most n^2, not exactly.
Dijkstra's with negative weightsDijkstra does NOT work with negative edge weights. Use Bellman-Ford instead.
Greedy vs DP confusionGreedy: local optimal choices lead to global optimal. DP: overlapping subproblems + optimal substructure. Greedy does not always work.
Forgetting that Quick Sort worst case is O(n^2)When pivot is always min or max element (sorted/reverse-sorted input), Quick Sort degrades to O(n^2).
Wrong recurrence for divide and conquerCount the work at each level carefully. Merge sort: T(n) = 2T(n/2) + O(n), not O(1).

5. Theory of Computation Mistakes

MistakeCorrect Approach
Confusing NFA vs DFA: "NFA is more powerful"NFA and DFA have EQUAL power (both recognize regular languages). NFA just needs fewer states sometimes.
Wrong pumping lemma applicationPumping lemma is for DISPROVING regularity, not proving it. You cannot use it to prove a language is regular.
Assuming all CFL are closed under intersectionCFL is NOT closed under intersection or complement. Regular languages are closed under both.
Confusing decidable vs recognizable (RE)Decidable = TM halts on all inputs (accepts or rejects). RE = TM accepts strings in L but may loop on strings not in L.
Epsilon transitions in NFA to DFA conversionCompute epsilon-closure for every state before processing transitions. Missing this leads to wrong DFA.

6. Compiler Design Mistakes

MistakeCorrect Approach
FIRST/FOLLOW computation: missing epsilon propagationIf A → BC and B can derive epsilon, then FIRST(C) is also in FIRST(A). Trace all nullable productions.
Confusing SLR, CLR, and LALR parser powerPower hierarchy: LR(0) < SLR(1) < LALR(1) < CLR(1). Every SLR grammar is LALR, but not vice versa.
Wrong handle identification in shift-reduce parsingHandle is the rightmost production body that matches in a rightmost derivation (bottom-up). Not just any matching substring.
Left recursion in LL parsersLL parsers cannot handle left recursion. Eliminate it: A → Aα | β becomes A → βA', A' → αA' | ε

7. Operating System Mistakes

MistakeCorrect Approach
Banker's algorithm: confusing Need vs AllocationNeed = Max - Allocation. Available = Total - Sum(Allocation). Check Need ≤ Available for safe sequence.
Page replacement: counting the initial page faultsWhen frames are empty, loading a page IS a page fault (compulsory miss). Do not skip initial loads.
Belady's anomaly: assuming more frames always helpsFIFO can show Belady's anomaly (more frames = more faults). LRU and Optimal do NOT exhibit this.
Semaphore: wrong initial valueBinary semaphore for mutex: init to 1. Counting semaphore for n resources: init to n. Signaling semaphore: init to 0.
Deadlock: forgetting one of the 4 conditionsAll 4 must hold simultaneously: Mutual Exclusion, Hold and Wait, No Preemption, Circular Wait.
Round Robin: wrong handling of arrival at same time as quantum expiryNewly arriving process goes to the end of the queue. The preempted process also goes to the end, AFTER the new arrival.

8. Database Mistakes

MistakeCorrect Approach
Confusing candidate key vs super keySuper key = any set of attributes that uniquely identifies a tuple. Candidate key = minimal super key (no proper subset is a super key).
BCNF vs 3NF confusionBCNF: for every FD X→Y, X must be a superkey. 3NF: same, OR Y is part of a candidate key. 3NF is weaker.
Wrong join result estimationNatural join of R and S: result can have 0 to |R|*|S| tuples. Do not assume it equals max(|R|, |S|).
SQL: GROUP BY without aggregate in SELECTEvery column in SELECT must either be in GROUP BY or inside an aggregate function (SUM, COUNT, etc.).
Relational algebra: confusing selection vs projectionσ (sigma) = selection (rows). π (pi) = projection (columns). Many students swap these.

9. Computer Networks Mistakes

MistakeCorrect Approach
Confusing transmission delay with propagation delayTransmission delay = L/R (packet size / bandwidth). Propagation delay = d/s (distance / speed). They are independent.
Subnetting: forgetting to subtract 2 from host countUsable hosts = 2^(host bits) - 2. The network address and broadcast address are not assignable to hosts.
TCP window size vs sequence number confusionWindow size determines how many bytes can be sent before ACK. Sequence number identifies each byte in the stream.
Sliding window: wrong efficiency calculationEfficiency = W / (1 + 2a) if W < 1+2a, else 1. Here a = T_prop/T_trans. Do not forget the min(1, ...) condition.
Subnet mask: wrong calculation of network bits/24 means first 24 bits are network. Subnet mask = 255.255.255.0. Always convert CIDR to binary if confused.

10. Calculation Mistakes to Avoid

TypeExamplePrevention
Unit conversion errorsMixing KB (1024) with Kb (bits) in cache problemsAlways convert everything to same unit first. Write units at each step.
Log base confusionlog2(8) = 3, not log10(8)GATE CS almost always uses log base 2. Assume log2 unless stated otherwise.
Off-by-one in countingFence post: n posts need n-1 gapsVerify with a small example (n=2 or n=3).
Power of 2 arithmetic2^10 = 1024, not 1000Memorize: 2^10=1024, 2^16=65536, 2^20=1048576
Negative number representation-5 in 2's complement (8-bit) = 11111011, not 10000101Method: invert bits of +5 (00000101 → 11111010) then add 1 → 11111011

11. Time Management Pitfalls

PitfallHow to Avoid
Spending 10+ minutes on a single 2-mark questionSet a strict 6-minute cap. Mark and move on. Return only if time permits.
Attempting questions in printed order (top to bottom)First sweep: solve all easy ones. Second sweep: medium ones. Third: hard ones.
Rechecking already-correct answers repeatedlyTrust your first instinct if you solved it confidently. Only recheck uncertain ones.
Leaving NAT questions unansweredNAT has NO negative marking. Always enter an answer, even a guess.
Not using the on-screen calculatorPractice with the virtual calculator before exam day. It saves time on complex arithmetic.
Getting stuck on the first hard question and panickingHard questions appear randomly. Skip them and build confidence with easier ones first.

12. Conceptual Traps — Quick Reference

  • NFA vs DFA: Same power (both recognize regular languages). NFA is NOT more powerful.
  • 2's complement overflow: Only occurs when adding same-sign numbers. Check if result sign differs from operand signs.
  • Deadlock vs Starvation: Deadlock = circular waiting, no progress. Starvation = process waits indefinitely but others progress.
  • Paging vs Segmentation: Paging = fixed-size frames, no external fragmentation. Segmentation = variable-size, has external fragmentation.
  • TCP vs UDP: TCP = reliable, connection-oriented. UDP = unreliable, connectionless. DNS uses both (UDP for queries, TCP for zone transfers).
  • B-Tree vs B+ Tree: B-Tree stores data in all nodes. B+ Tree stores data only in leaves; internal nodes have only keys.
  • P vs NP: P ⊆ NP (every P problem is also NP). Whether P = NP is unknown. NP-hard is NOT necessarily in NP.
Final Advice: After every mock test, review your mistakes and categorize them using the tables above. Most students lose 10-15 marks to avoidable errors. Fixing these can dramatically improve your score.

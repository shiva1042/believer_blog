1. Asymptotic Notations

Asymptotic notations describe the growth rate of functions as input size n approaches infinity. They abstract away constant factors and lower-order terms.

1.1 Big-O Notation (Upper Bound)

Definition: f(n) = O(g(n)) if there exist positive constants c and n0 such that f(n) ≤ c * g(n) for all n ≥ n0.

Big-O represents the worst-case upper bound on growth rate. It tells us the function grows no faster than g(n).

Example: f(n) = 3n2 + 5n + 2
  f(n) = O(n2) because 3n2 + 5n + 2 ≤ 10n2 for n ≥ 1
  Also f(n) = O(n3), O(n4), etc. (but we prefer the tightest bound)

1.2 Big-Omega Notation (Lower Bound)

Definition: f(n) = Omega(g(n)) if there exist positive constants c and n0 such that f(n) ≥ c * g(n) for all n ≥ n0.

Big-Omega represents the best-case lower bound. The function grows at least as fast as g(n).

1.3 Big-Theta Notation (Tight Bound)

Definition: f(n) = Theta(g(n)) if there exist positive constants c1, c2, and n0 such that c1 * g(n) ≤ f(n) ≤ c2 * g(n) for all n ≥ n0.

Big-Theta gives a tight bound: the function grows at exactly the same rate as g(n). f(n) = Theta(g(n)) if and only if f(n) = O(g(n)) AND f(n) = Omega(g(n)).

1.4 Little-o Notation (Strict Upper Bound)

Definition: f(n) = o(g(n)) if for every positive constant c, there exists n0 such that f(n) < c * g(n) for all n ≥ n0.

Equivalently, lim (n->infinity) f(n)/g(n) = 0. This means f(n) grows strictly slower than g(n).

Example: n = o(n2), but n2 is not o(n2).

1.5 Little-omega Notation (Strict Lower Bound)

Definition: f(n) = omega(g(n)) if for every positive constant c, there exists n0 such that f(n) > c * g(n) for all n ≥ n0.

Equivalently, lim (n->infinity) f(n)/g(n) = infinity. The function grows strictly faster than g(n).

3. Best, Worst, and Average Case Analysis

3.1 Best Case

The input configuration that causes the algorithm to execute the minimum number of operations.

Example: Linear search where the target is the first element: O(1).

3.2 Worst Case

The input configuration that causes the maximum number of operations. This is the most commonly analyzed case as it provides a guarantee.

Example: Quick sort with the smallest (or largest) element always chosen as pivot on a sorted array: O(n2).

3.3 Average Case

Expected number of operations over all possible inputs, assuming a probability distribution (usually uniform).

Example: Quick sort with random pivot selection: O(n log n) expected time.

Average case of Linear Search:
  Assuming element is present and equally likely at any position:
  Average comparisons = (1 + 2 + 3 + ... + n) / n = (n + 1) / 2 = O(n)

4. Master Theorem for Recurrences

The Master Theorem solves recurrences of the form:

T(n) = aT(n/b) + f(n)  where a ≥ 1, b > 1

Let c_crit = log_b(a). Then:

CaseConditionResult
Case 1 f(n) = O(n^(c_crit - epsilon)) for some epsilon > 0 T(n) = Theta(n^c_crit)
Case 2 f(n) = Theta(n^c_crit * log^k(n)), k ≥ 0 T(n) = Theta(n^c_crit * log^(k+1)(n))
Case 3 f(n) = Omega(n^(c_crit + epsilon)) for some epsilon > 0, and af(n/b) ≤ cf(n) for c < 1 T(n) = Theta(f(n))

5. Other Methods for Solving Recurrences

5.1 Substitution Method

Guess the form of the solution and prove by induction.

T(n) = 2T(n/2) + n
Guess: T(n) = O(n log n)
Assume T(k) ≤ ck log k for all k < n
T(n) = 2c(n/2)log(n/2) + n
     = cn(log n - 1) + n
     = cn log n - cn + n
     ≤ cn log n     (when c ≥ 1)

5.2 Recursion Tree Method

Draw the tree of recursive calls, compute the cost at each level, and sum across all levels.

6. Amortized Analysis

Amortized analysis determines the average cost per operation over a sequence of operations, even though individual operations may be expensive.

6.1 Aggregate Method

Compute the total cost of n operations and divide by n.

Example: Dynamic array (vector) doubling
  Insert n elements. Resizing occurs at sizes 1, 2, 4, 8, ...
  Total cost = n + (1 + 2 + 4 + ... + n) ≤ n + 2n = 3n
  Amortized cost per insert = 3n/n = O(1)

6.2 Accounting Method

Assign different charges to different operations. Some operations are overcharged (credit stored), others use stored credit.

6.3 Potential Method

Define a potential function Phi that maps the state of the data structure to a real number. The amortized cost of operation i is:

amortized_cost_i = actual_cost_i + Phi(D_i) - Phi(D_{i-1})

GATE Tip: Amortized analysis is NOT average-case analysis. It guarantees the per-operation cost over any sequence of operations, without probabilistic assumptions. Common examples: dynamic arrays, splay trees, Fibonacci heaps.

8. Key GATE Takeaways

  • Big-O is an upper bound, Big-Omega is a lower bound, Big-Theta is a tight bound.
  • The Master Theorem is the fastest way to solve divide-and-conquer recurrences in GATE.
  • Amortized O(1) for dynamic array insert is a frequently tested concept.
  • Always simplify: drop constants and lower-order terms. O(3n2 + 5n) = O(n2).
  • log base does not matter in Big-O: O(log2 n) = O(log10 n) = O(ln n) since they differ by a constant.

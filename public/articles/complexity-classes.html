Theory of Computation

Complexity Classes — P, NP, PSPACE, and Beyond

Computational complexity theory classifies problems not by whether they can be solved, but by how efficiently they can be solved. While computability theory asks "Is this problem solvable at all?", complexity theory asks "How much time or space does it take?" This article covers the material from Chapters 10–11 of Hopcroft, Motwani, and Ullman, with a focus on what appears in GATE CSE examinations.

1. Foundational Concepts

1.1 Measuring Complexity

We measure the complexity of a decision problem (a yes/no question) in terms of the resources consumed by the best algorithm (Turing machine) that solves it, as a function of input size n.

  • Time complexity: The number of steps a TM takes before halting.
  • Space complexity: The number of tape cells a TM uses (beyond the input).

We use worst-case analysis: T(n) = max over all inputs of size n of the number of steps.

1.2 Deterministic vs Nondeterministic TMs

A deterministic TM (DTM) has exactly one possible action in each configuration. A nondeterministic TM (NTM) can have multiple possible actions; it accepts if any computation path leads to acceptance.

For time complexity:

  • DTM time: Number of steps in the single computation.
  • NTM time: Length of the shortest accepting computation path (if one exists).
Key relationship: Any NTM running in time T(n) can be simulated by a DTM in time 2O(T(n)). This exponential blowup is the crux of the P vs NP question.

2. Time Complexity Classes

2.1 P — Polynomial Time

Definition:

P = ⋃k≥0 DTIME(nk)

P is the class of all decision problems solvable by a deterministic Turing machine in polynomial time.

Informal meaning: Problems in P are considered "efficiently solvable" or "tractable."

Examples of problems in P:

2.2 NP — Nondeterministic Polynomial Time

Definition (NTM view):

NP = ⋃k≥0 NTIME(nk)

NP is the class of decision problems solvable by a nondeterministic Turing machine in polynomial time.

Definition (Certificate/Verifier view): A language L is in NP if there exists a polynomial-time deterministic verifier V such that:

x ∈ L  ⇔  ∃ certificate c, |c| ≤ p(|x|), such that V(x, c) accepts

The certificate (also called a "witness" or "proof") must be:

  • Of polynomial length in the input size.
  • Verifiable in polynomial time by a deterministic machine.
Intuition: NP problems are those where a proposed solution can be checked quickly, even if finding the solution might be hard. For example, given a claimed Hamiltonian cycle, you can verify it in O(n) time, but finding one might require exponential time.

Examples of problems in NP:

ProblemCertificateVerification
SAT (Boolean satisfiability)A satisfying assignmentEvaluate formula: O(|formula|)
Hamiltonian CycleThe cycle itselfCheck it visits all vertices once: O(V)
Graph Coloring (k-coloring)The coloringCheck no adjacent same-color: O(E)
CliqueThe clique verticesCheck all edges exist: O(k2)
Subset SumThe subsetSum the elements: O(n)
Vertex CoverThe cover setCheck all edges covered: O(E)
Integer Linear ProgrammingA feasible solutionCheck constraints: O(constraints)
Travelling Salesman (decision)The tourCompute tour length: O(n)

2.3 co-NP

Definition: co-NP = {L | Lc ∈ NP}

A language is in co-NP if its complement is in NP, i.e., there exist short proofs of "NO" instances.

Examples:

  • TAUTOLOGY: "Is this Boolean formula true for ALL assignments?" The complement (SATISFIABILITY of the negation) is in NP, so TAUTOLOGY ∈ co-NP.
  • Primality: Actually in both NP and co-NP (and in P, by AKS).
  • UNSAT: "Is this formula unsatisfiable?" This is co-NP (complement of SAT).

Relationship with NP:

  • P ⊆ NP ∩ co-NP (every problem in P is in both NP and co-NP).
  • Whether NP = co-NP is an open question.
  • If any NP-complete problem is in co-NP, then NP = co-NP.

2.4 NP-Hard and NP-Complete

Definition (NP-Hard): A problem H is NP-hard if every problem in NP can be polynomial-time reduced to H. Formally: ∀ L ∈ NP: L ≤p H.

Definition (NP-Complete): A problem is NP-complete if it is:

  1. In NP, AND
  2. NP-hard.
NPC = NP ∩ NP-hard
Critical distinction: NP-hard problems need NOT be in NP. The Halting Problem is NP-hard (every NP problem reduces to it) but it is not in NP (it's not even decidable!). NP-complete problems must be in NP.

Significance of NP-completeness:

  • If ANY NP-complete problem can be solved in polynomial time, then P = NP.
  • If ANY NP-complete problem cannot be solved in polynomial time, then P ≠ NP.
  • All NP-complete problems are polynomial-time equivalent to each other.

2.5 The Cook-Levin Theorem

Theorem (Cook, 1971; Levin, 1973): SAT (Boolean Satisfiability) is NP-complete.

Proof idea: Given any NP language L with NTM M running in time p(n), encode the entire computation of M on input x as a Boolean formula φ such that φ is satisfiable iff M accepts x. The formula has variables representing the state, head position, and tape contents at each time step, with clauses enforcing the transition function.

3.3 EXPTIME

Definition: EXPTIME = ⋃k≥0 DTIME(2nk)

Problems solvable in exponential time by a deterministic TM.

Key result: P ⊊ EXPTIME (by the Time Hierarchy Theorem). This is one of the few proven separations.

4. Space Complexity Classes

4.1 Definitions

Space complexity measures the number of work tape cells used (excluding the read-only input tape for sublinear classes).

DSPACE(f(n)) = languages decidable by a DTM using O(f(n)) space
NSPACE(f(n)) = languages decidable by an NTM using O(f(n)) space

4.2 L — Logarithmic Space

Definition: L = DSPACE(log n)

Problems solvable using only O(log n) additional space (beyond the read-only input).

Examples:

  • Undirected s-t connectivity (Reingold's theorem, 2004)
  • Checking if a string is a palindrome (using two pointers)
  • Checking graph properties that need only a constant number of pointers into the input
Why logarithmic? With O(log n) space, you can store a constant number of pointers/indices into the input (each index requires log n bits). This is exactly what you need for many graph traversals.

4.3 NL — Nondeterministic Logarithmic Space

Definition: NL = NSPACE(log n)

Key example: PATH = {(G, s, t) | there is a directed path from s to t in G} is NL-complete.

Proof that PATH is in NL: Nondeterministically guess the next vertex on the path, keeping only the current vertex in memory (O(log n) bits).

Immerman-Szelepcsényi Theorem: NL = co-NL. More generally, NSPACE(f(n)) = co-NSPACE(f(n)) for f(n) ≥ log n. This means non-reachability is also in NL.

4.4 PSPACE

Definition: PSPACE = ⋃k≥0 DSPACE(nk)

Problems solvable using polynomial space.

4.5 NPSPACE

Definition: NPSPACE = ⋃k≥0 NSPACE(nk)

4.6 Savitch's Theorem

Theorem (Savitch, 1970): For any function f(n) ≥ log n:

NSPACE(f(n)) ⊆ DSPACE(f(n)2)

Corollary: NPSPACE = PSPACE

Nondeterministic polynomial space equals deterministic polynomial space. The squaring of space is polynomial if we started with polynomial space.

Proof idea: To check if configuration C1 leads to configuration C2 in at most 2k steps, guess a midpoint configuration Cm and recursively check:

  1. C1 leads to Cm in at most 2k-1 steps, AND
  2. Cm leads to C2 in at most 2k-1 steps.

The recursion depth is O(log(2f(n))) = O(f(n)), and each level stores one configuration of size O(f(n)), giving total space O(f(n)2).

GATE significance: Savitch's theorem is frequently tested. The key takeaway: nondeterminism doesn't help much for space (only a quadratic blowup), unlike time (where the blowup is believed to be exponential for NP vs P).

4.7 PSPACE-Complete Problems

Definition: A problem is PSPACE-complete if it is in PSPACE and every PSPACE problem reduces to it in polynomial time.

TQBF (True Quantified Boolean Formulas):

Given a fully quantified Boolean formula:

∀x1 ∃x2 ∀x3 ... Qnxn φ(x1, ..., xn)

Is this formula true?

TQBF is PSPACE-complete. It is the canonical PSPACE-complete problem, analogous to SAT for NP.

Other PSPACE-complete problems:

ProblemDescription
TQBFIs a quantified Boolean formula true?
Generalized GeographyDoes Player 1 have a winning strategy?
Generalized ChessDoes White have a forced win on an n×n board?
Regular Expression EquivalenceDo two regexes with squaring describe the same language?
Sticker SystemsReachability in certain string rewriting systems
Pattern: Many two-player games, when generalized to n×n boards, become PSPACE-complete. The alternation of ∀ (opponent moves) and ∃ (your moves) maps naturally to TQBF.

5.2 Space and Time Hierarchy Theorems

Time Hierarchy Theorem: For any time-constructible function f(n):

DTIME(f(n)) ⊊ DTIME(f(n) · log2(f(n)))

In particular, DTIME(nk) ⊊ DTIME(nk+1), which gives us P ⊊ EXPTIME.

Space Hierarchy Theorem: For any space-constructible function f(n):

DSPACE(f(n)) ⊊ DSPACE(f(n) · log(f(n)))

In particular, DSPACE(log n) ⊊ DSPACE(n), giving L ⊊ PSPACE.

6. Reductions

6.1 Many-One (Karp) Reductions

Definition: Language A is polynomial-time many-one reducible to language B (written A ≤p B) if there exists a polynomial-time computable function f such that:

x ∈ A  ⇔  f(x) ∈ B

Properties:

  • Transitivity: If A ≤p B and B ≤p C, then A ≤p C.
  • If A ≤p B and B ∈ P, then A ∈ P.
  • If A ≤p B and A ∉ P, then B ∉ P.
  • Karp reductions preserve NP membership: if A ≤p B and B ∈ NP, then A ∈ NP.

6.2 Turing (Cook) Reductions

Definition: A ≤T B if there is a polynomial-time algorithm for A that can make queries to an oracle for B.

Key differences from Karp reductions:

PropertyKarp ReductionTuring Reduction
TypeOne function call, output is the answerMultiple oracle calls allowed
Answer usageMust preserve yes/no directlyCan negate, combine oracle answers
PowerWeakerStrictly more powerful
NPC definitionStandard (uses Karp)Gives "NP-hard under Turing reductions"
ComplementA ≤p B does NOT imply Ac ≤p BA ≤T B implies Ac ≤T B
GATE focus: GATE primarily uses Karp (many-one) reductions. When a question says "polynomial-time reduction," it almost always means Karp reduction unless stated otherwise.

6.3 Direction of Reduction

To show problem B is NP-hard, reduce a known NP-hard problem A to B: A ≤p B.

The reduction goes FROM the known hard problem TO the target problem.

Common mistake: Students often reduce in the wrong direction. Remember: A ≤p B means "A is no harder than B." To show B is hard, show something known-hard reduces TO B.

8. Undecidable vs Intractable

It is critical to distinguish two very different notions of "hard":

UndecidableIntractable (NP-hard)
Can it be solved at all?No algorithm existsAlgorithm exists, but likely not efficient
Halts on all inputs?No TM always haltsAlgorithm halts, but may take exponential time
ExampleHalting Problem3-SAT, TSP
Can we verify?Not alwaysYes, in polynomial time (for NP)
ApproachNo general solution possibleHeuristics, approximations, special cases
Language classNot in RecursiveIn Recursive but (likely) not in P
GATE distinction: An undecidable problem has NO algorithm at all. An intractable problem HAS an algorithm, but every known algorithm takes super-polynomial time. NP-complete problems are decidable (they are in NP ⊆ EXPTIME), just believed to not be in P.

9. Complement Classes and Their Significance

9.1 Overview

9.2 Why Complements Matter

If a class equals its complement class, it means the class is "symmetric" — proving membership is as easy as proving non-membership. For NP, the asymmetry is key:

  • NP has short proofs for YES instances.
  • co-NP has short proofs for NO instances.
  • If NP ≠ co-NP, then no NP-complete problem has short proofs of non-membership.

9.3 Implications of P = NP

If P = NP, then:

  • NP = co-NP (since P = co-P).
  • Every problem in NP has a polynomial-time algorithm.
  • The entire polynomial hierarchy collapses to P.
  • Cryptography based on one-way functions would be broken.

10. The Polynomial Hierarchy (Brief Overview)

The polynomial hierarchy (PH) generalizes NP and co-NP:

Σ0P = Π0P = Δ0P = P
Σ1P = NP
Π1P = co-NP
Σk+1P = NPΣkP  (NP with a ΣkP oracle)
ΠkP = co-ΣkP

PH = ⋃k≥0 ΣkP

Key facts:

  • PH ⊆ PSPACE.
  • If PH has a complete problem, then PH collapses to some finite level.
  • If P = NP, then PH = P (total collapse).
  • Σ2P problems have the form: ∃y ∀z φ(x, y, z) where φ is polynomial-time decidable.
GATE relevance: The polynomial hierarchy is rarely tested in depth, but you should know that it exists, that PH ⊆ PSPACE, and that P = NP would collapse it entirely.

12. Common GATE Question Patterns

Pattern 1: "Which class does this problem belong to?"

Given a problem description, identify whether it is in P, NP-complete, PSPACE-complete, or undecidable.

Strategy: Look for known NP-complete kernels (3-SAT structure, clique-like, subset-sum-like). Check if it has a polynomial-time algorithm. Check if it involves quantifier alternation (PSPACE). Check if it involves TM behavior (undecidable).

Pattern 2: "If P = NP, which of the following is true?"

If P = NP, then: NP = co-NP, PH = P, NPC = P (all NPC problems would be in P), and every NP problem is polynomial.

Pattern 3: "Which of these reductions is correct?"

Check: (1) Is the direction correct? (from known hard to new problem), (2) Is it polynomial-time computable? (3) Does it preserve yes/no instances?

Pattern 4: "What is the relationship between these classes?"

Know the chain: L ⊆ NL ⊆ P ⊆ NP ⊆ PSPACE ⊆ EXPTIME. Know which equalities are open vs proven.

Pattern 5: Confusing NP-hard with NP-complete

NP-hard problems need NOT be in NP. The optimization version of TSP ("find the shortest tour") is NP-hard but not in NP (it's not even a decision problem). The Halting Problem is NP-hard but undecidable.

Pattern 6: Savitch's Theorem Applications

If a problem is in NPSPACE (nondeterministic polynomial space), then it is also in PSPACE (deterministic polynomial space) by Savitch's theorem.

Pattern 7: Problems in NP ∩ co-NP but not known to be in P

Examples: Factoring (widely believed), Graph Isomorphism (now known to be in quasi-polynomial time). These are candidates for problems in NP ∩ co-NP \ P, if such problems exist.

13. Worked GATE-Style Examples

Example 1

Q: Let L be a language in NP. Consider the following statements:

  1. L is decidable.
  2. L can be solved in polynomial time.
  3. L can be solved in exponential time.
  4. The complement of L is in NP.

Which statements are necessarily true?

Solution:

  • (1) TRUE: NP ⊆ EXPTIME ⊆ Decidable. All NP problems are decidable.
  • (2) UNKNOWN: Only true if P = NP, which is unproven.
  • (3) TRUE: NP ⊆ EXPTIME, so every NP problem has an exponential-time (or better) algorithm.
  • (4) UNKNOWN: Only true if NP = co-NP, which is unproven.

Answer: (1) and (3) are necessarily true.

Example 2

Q: Problem X is NP-complete. Problem Y is in P. What can we say about X ≤p Y?

Solution: If X ≤p Y and Y ∈ P, then X ∈ P. But X is NP-complete, so this would mean P = NP. Since we believe P ≠ NP, it is unlikely (but not proven impossible) that X ≤p Y. If P = NP, then yes, such a reduction exists.

Example 3

Q: Show that if TAUTOLOGY is in NP, then NP = co-NP.

Solution:

  1. TAUTOLOGY is the complement of SAT. Since SAT is NP-complete, TAUTOLOGY is co-NP-complete.
  2. If TAUTOLOGY ∈ NP, then a co-NP-complete problem is in NP.
  3. Therefore, every co-NP problem reduces to TAUTOLOGY (which is in NP), so co-NP ⊆ NP.
  4. By symmetry (complement both sides), NP ⊆ co-NP.
  5. Hence NP = co-NP.

Example 4

Q: A problem requires O(n2) space on a nondeterministic TM. What can we say about its deterministic space complexity?

Solution: By Savitch's theorem, NSPACE(n2) ⊆ DSPACE(n4). So the problem can be solved deterministically in O(n4) space.

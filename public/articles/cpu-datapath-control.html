1. Arithmetic Logic Unit (ALU)

The ALU is the computational core of the CPU. It performs arithmetic operations (add, subtract, multiply, divide) and logical operations (AND, OR, XOR, NOT, shift) on binary data.

1.1 ALU Operations

CategoryOperations
ArithmeticADD, SUB, MUL, DIV, INC, DEC, NEG
LogicalAND, OR, XOR, NOT, NAND, NOR
ShiftLogical Shift Left/Right, Arithmetic Shift Left/Right, Rotate Left/Right
ComparisonCMP (sets flags based on subtraction result)

1.2 ALU Status Flags

FlagSymbolConditionUse
ZeroZResult = 0Equality testing
CarryCCarry/borrow out of MSBUnsigned overflow detection
OverflowVSigned result exceeds rangeSigned overflow detection
Sign/NegativeN or SMSB of result = 1Negative result indicator

Overflow vs Carry: The carry flag indicates unsigned overflow. The overflow flag (V) indicates signed overflow. They are independent. V = Cn XOR Cn-1 (carry into MSB XOR carry out of MSB).

OperationActionFill BitUse

Logical Shift RightShift bits right0 fills MSBUnsigned divide by 2
Arithmetic Shift RightShift bits rightSign bit fills MSBSigned divide by 2
Rotate LeftShift left, MSB wraps to LSBMSB wrapsCircular operations
Rotate RightShift right, LSB wraps to MSBLSB wrapsCircular operations

3. Control Unit

The control unit generates the sequence of control signals that orchestrate instruction execution. It uses the opcode and flags to determine what actions to take at each step.

3.1 Hardwired Control Unit

Control logic is implemented directly with combinational and sequential circuits (gates, flip-flops, decoders).

AdvantageDisadvantage
Very fast executionComplex to design
Optimized for specific ISADifficult to modify/extend
Used in RISC processorsInflexible for complex instruction sets

3.2 Microprogrammed Control Unit

Control signals are stored as microinstructions in a control memory (control store). Each machine instruction is executed by a sequence of microinstructions.

AdvantageDisadvantage
Easy to design and modifySlower than hardwired
Supports complex instruction setsRequires control memory
Used in CISC processorsAdditional fetch overhead for microinstructions

3.3 Comparison

FeatureHardwiredMicroprogrammed
SpeedFasterSlower
FlexibilityLowHigh
Cost (for complex ISA)HighLower
Design complexityHighLower
Typical useRISCCISC

3.4 Horizontal vs Vertical Microprogramming

FeatureHorizontalVertical
Microinstruction widthWide (many bits)Narrow (fewer bits)
ParallelismHigh (multiple signals per word)Low (one or few operations)
EncodingMinimal (each bit = one signal)Encoded (needs decoder)
Control memory sizeLarge (wide words)Smaller (narrow words but more words)

Every instruction goes through a series of steps called the instruction cycle.

5. Single-Cycle vs Multi-Cycle Implementation

FeatureSingle-CycleMulti-Cycle
Clock periodSet by longest instructionSet by longest stage
CPI (Cycles Per Instruction)13-5 (varies)
Hardware utilizationLow (components idle most of cycle)Higher (components reused)
PerformanceFast for uniform instructionsBetter for variable-length operations

Performance formula: CPU Time = Instruction Count × CPI × Clock Period. Improving any factor improves performance.

6. GATE Exam Tips

  • Know the difference between hardwired and microprogrammed control for comparison questions.
  • Be able to trace micro-operations for basic instructions (ADD, LOAD, STORE, BRANCH).
  • Understand that overflow flag uses signed arithmetic while carry flag uses unsigned arithmetic.
  • Arithmetic shift right preserves the sign bit; logical shift right fills with 0.
  • Single-cycle: CPI=1 but long clock period. Multi-cycle: shorter clock but variable CPI.
  • Horizontal microprogramming = wide word, more parallelism, no decoding needed.

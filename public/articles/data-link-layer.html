1. Framing

The data link layer packages raw bits from the physical layer into frames. Framing defines the boundaries of each frame.

MethodDescriptionIssue
Character CountA field in the header specifies the number of characters in the frameIf count is corrupted, synchronization is lost permanently

Bit Stuffing Example:
Original data: 01111111 10
After stuffing: 011111011 10 (a 0 is inserted after the fifth consecutive 1)
Receiver removes the 0 after every five consecutive 1s.

2. Error Detection

2.1 Parity Check

TypeDescriptionDetection Capability
Single-bit ParityAdd one parity bit to make total number of 1s even (even parity) or odd (odd parity)Detects odd number of bit errors; misses even number
Two-dimensional ParityArrange data in a matrix; compute parity for each row and columnDetects all 1, 2, and 3-bit errors; most 4-bit errors

2.2 Checksum

Data is divided into fixed-length words. All words are summed (with carry wraparound). The one's complement of the sum is the checksum. The receiver adds all words including the checksum; if result is all 1s, no error is detected.

Internet Checksum (used by IP, TCP, UDP) uses 16-bit one's complement arithmetic.

2.3 Cyclic Redundancy Check (CRC)

CRC is the most powerful error detection code used at the data link layer. It treats the bit string as a polynomial and divides it by a generator polynomial.

CRC Calculation Steps:
  1. Given data D and generator polynomial G of degree r
  2. Append r zero bits to D: D × 2r
  3. Divide (D × 2r) by G using XOR (modulo-2) division
  4. The remainder R (r bits) is the CRC
  5. Transmitted frame = D followed by R
  6. Receiver divides received frame by G; if remainder is 0, no error
CRC StandardGenerator PolynomialDetects
CRC-8x8 + x2 + x + 1All 1, 2-bit errors
CRC-16x16 + x15 + x2 + 1All 1, 2-bit and odd-bit errors
CRC-32Used in Ethernet, ZIPAll burst errors of length ≤ 32
CRC Detection Capabilities:
  • All single-bit errors (if G has more than one term)
  • All double-bit errors (if G has a factor with 3 terms)
  • All odd number of bit errors (if G has factor (x+1))
  • All burst errors of length ≤ r (degree of G)

3. Error Correction: Hamming Code

Hamming code can detect and correct single-bit errors. For m data bits, r redundancy bits are needed where 2r ≥ m + r + 1.

Parity bits are placed at positions that are powers of 2 (1, 2, 4, 8, ...). Each parity bit covers specific positions based on the binary representation of position numbers. The error position is found by XORing all parity check results (syndrome).

Hamming Distance: The minimum number of bit positions in which two codewords differ. For d-bit error detection, minimum Hamming distance = d + 1. For d-bit error correction, minimum Hamming distance = 2d + 1.

4.2 CSMA/CD (Collision Detection)

Used in Ethernet. Stations detect collisions while transmitting and abort immediately, saving time.

flowchart TD A["Frame ready to send"] --> B{"Channel idle?"} B -->|"No"| C["Wait (backoff)"] C --> B B -->|"Yes"| D["Begin transmission"] D --> E{"Collision detected?"} E -->|"No"| F["Transmission complete\nSuccess!"] E -->|"Yes"| G["Send jam signal\n(48 bits)"] G --> H["Increment attempt counter"] H --> I{"Max attempts\nreached?"} I -->|"No"| J["Binary Exponential Backoff\nWait random(0..2^k - 1) slots"] J --> B I -->|"Yes"| K["Report failure\nAbort"]
Minimum Frame Size for CSMA/CD:
The sender must be able to detect a collision before finishing transmission. The frame must take at least 2 × propagation delay to transmit.
Min frame size = 2 × propagation delay × bandwidth
For Ethernet (10 Mbps, max 2500m, propagation speed 2 × 108 m/s): Min frame = 2 × (2500 / 2×108) × 107 = 250 bits. Actual Ethernet minimum = 512 bits (64 bytes).

4.3 CSMA/CA (Collision Avoidance)

Used in wireless networks (IEEE 802.11) where collision detection is difficult due to the hidden terminal problem. Uses RTS/CTS handshake and NAV (Network Allocation Vector) to reserve the channel.

5. Ethernet (IEEE 802.3)

5.1 MAC Addressing

A MAC address is a 48-bit (6-byte) hardware address uniquely assigned to each network interface card. Written as six hex pairs: AA:BB:CC:DD:EE:FF.

Address TypeDescription
UnicastFirst bit = 0. Identifies a single interface.
BroadcastAll 1s: FF:FF:FF:FF:FF:FF. Sent to all stations on LAN.
MulticastFirst bit = 1 (but not broadcast). Sent to a group of stations.

6. Ethernet Bridging and Spanning Tree

6.1 Bridges and Switches

A bridge (or Layer 2 switch) connects LAN segments and forwards frames based on MAC addresses. It maintains a forwarding table (MAC address → port mapping), learned through backward learning (observing source addresses of incoming frames).

6.2 Spanning Tree Protocol (STP)

When bridges create loops in the network topology, broadcast storms can occur. The Spanning Tree Protocol (IEEE 802.1D) prevents loops by creating a loop-free logical topology.

  1. Elect a root bridge (lowest bridge ID)
  2. Each non-root bridge selects a root port (port with shortest path to root)
  3. Each LAN segment selects a designated port (bridge port with shortest path to root for that segment)
  4. All other ports are blocked

7. Summary and GATE Key Points

Key Points for GATE:
  • CRC can detect all single-bit, double-bit, odd-bit errors, and all burst errors of length ≤ r.
  • Hamming code: number of redundancy bits r satisfies 2r ≥ m + r + 1.
  • Pure ALOHA max throughput = 18.4%; Slotted ALOHA = 36.8%. Slotted ALOHA is exactly 2x Pure ALOHA.
  • CSMA/CD minimum frame size = 2 × propagation_delay × bandwidth.
  • Ethernet minimum frame size = 64 bytes (512 bits); maximum = 1518 bytes.
  • Binary Exponential Backoff: after kth collision, choose random wait from [0, 2k − 1] slots (k ≤ 10).
  • Bridges learn MAC addresses via backward learning. STP prevents loops.
  • CSMA/CD is for wired (Ethernet); CSMA/CA is for wireless (Wi-Fi).

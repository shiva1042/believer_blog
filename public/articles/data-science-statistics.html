1.2 Measures of Spread

Variance:  sigma^2 = (1/n) * SUM(x_i - mean)^2       (population)
                   = (1/(n-1)) * SUM(x_i - mean)^2   (sample)

Standard Deviation: sigma = sqrt(variance)

Range = max - min

IQR = Q3 - Q1  (Interquartile Range)

Example: Data = {2, 4, 4, 4, 5, 5, 7, 9}
  Mean = 40/8 = 5
  Variance = [(9+1+1+1+0+0+4+16)/8] = 32/8 = 4
  Std Dev = sqrt(4) = 2

2. Hypothesis Testing

2.1 Framework

2.2 Key Concepts

Null Hypothesis (H0):  "No effect" or "no difference" (status quo)
Alt Hypothesis (H1):   What we want to prove

Significance level (alpha): Probability of rejecting H0 when it's true
  Common values: 0.05, 0.01, 0.10

p-value: Probability of observing data as extreme as seen,
         assuming H0 is true.
  If p-value < alpha -> reject H0
  If p-value >= alpha -> fail to reject H0

2.3 Type I and Type II Errors

H0 TrueH0 False
Reject H0Type I Error (α)Correct (Power = 1-β)
Fail to Reject H0CorrectType II Error (β)

2.4 t-Test

One-sample t-test: Is the population mean equal to mu_0?
  t = (x_bar - mu_0) / (s / sqrt(n))
  df = n - 1

Two-sample t-test (independent):
  t = (x1_bar - x2_bar) / sqrt(s1^2/n1 + s2^2/n2)

Example: Sample mean = 52, H0: mu = 50, s = 4, n = 16
  t = (52 - 50) / (4 / sqrt(16)) = 2 / 1 = 2.0
  df = 15
  For alpha = 0.05 (two-tailed), t_critical ≈ 2.131
  t = 2.0 < 2.131, so fail to reject H0.

2.5 Chi-Square Test

Test for independence between two categorical variables.

chi^2 = SUM[ (O - E)^2 / E ]

where O = observed frequency, E = expected frequency
  E = (row_total * col_total) / grand_total

df = (rows - 1) * (cols - 1)

Example:
              Male    Female    Total
Pass           30       45       75
Fail           20        5       25
Total          50       50      100

Expected (Pass, Male) = 75*50/100 = 37.5
Expected (Pass, Female) = 75*50/100 = 37.5
Expected (Fail, Male) = 25*50/100 = 12.5
Expected (Fail, Female) = 25*50/100 = 12.5

chi^2 = (30-37.5)^2/37.5 + (45-37.5)^2/37.5
      + (20-12.5)^2/12.5 + (5-12.5)^2/12.5
      = 1.5 + 1.5 + 4.5 + 4.5 = 12.0

df = (2-1)(2-1) = 1
Critical value at alpha=0.05: 3.841
chi^2 = 12.0 > 3.841 -> Reject H0 (variables are dependent)

3. Correlation vs Causation

AspectCorrelationCausation
DefinitionStatistical relationship between variablesOne variable causes change in another
DirectionBidirectionalUnidirectional
Proven byObservational dataControlled experiments
CoefficientPearson r ∈ [-1, 1]N/A
Pearson Correlation:
  r = SUM[(x_i - x_bar)(y_i - y_bar)] /
      sqrt(SUM[(x_i-x_bar)^2] * SUM[(y_i-y_bar)^2])

  r = +1: perfect positive correlation
  r = -1: perfect negative correlation
  r =  0: no linear correlation
Key Insight: Correlation does NOT imply causation. Ice cream sales and drowning deaths are correlated (both increase in summer) but ice cream doesn't cause drowning!

4. Principal Component Analysis (PCA)

PCA reduces dimensionality by projecting data onto directions of maximum variance.

4.1 PCA Algorithm

Step 1: Standardize the data (zero mean, unit variance)
Step 2: Compute covariance matrix C = (1/n) X^T X
Step 3: Find eigenvalues and eigenvectors of C
Step 4: Sort eigenvectors by decreasing eigenvalue
Step 5: Select top k eigenvectors (principal components)
Step 6: Project data: Z = X * W_k  (W_k = matrix of top k eigenvectors)

4.2 Explained Variance

Explained variance ratio for component i:
  EVR_i = lambda_i / SUM(lambda_j)

Example: Eigenvalues = [4.2, 1.8, 0.8, 0.2]
  Total = 7.0
  PC1 explains: 4.2/7.0 = 60%
  PC1+PC2 explain: 6.0/7.0 = 85.7%
  Choose k such that cumulative EVR >= 95%

5. Clustering

5.1 K-Means Clustering

Algorithm:
1. Initialize k centroids randomly
2. Assign each point to nearest centroid
3. Recompute centroids as mean of assigned points
4. Repeat steps 2-3 until convergence

Objective: Minimize SUM_{i=1}^{k} SUM_{x in C_i} ||x - mu_i||^2

5.2 K-Means Example

Points: (1,1), (1,2), (2,1), (8,8), (8,9), (9,8), k=2

Iteration 1:
  Init centroids: C1=(1,1), C2=(8,8)
  Assign: {(1,1),(1,2),(2,1)} -> C1,  {(8,8),(8,9),(9,8)} -> C2
  Update: C1 = (4/3, 4/3) = (1.33, 1.33)
          C2 = (25/3, 25/3) = (8.33, 8.33)

Iteration 2: Same assignments -> Converged!

6. Evaluation Metrics for Classification

6.1 Confusion Matrix

Predicted PositivePredicted Negative
Actual PositiveTrue Positive (TP)False Negative (FN)
Actual NegativeFalse Positive (FP)True Negative (TN)

6.2 Metrics from Confusion Matrix

Accuracy  = (TP + TN) / (TP + TN + FP + FN)
Precision = TP / (TP + FP)        -- of predicted positives, how many are correct?
Recall    = TP / (TP + FN)        -- of actual positives, how many did we find?
F1 Score  = 2 * (Precision * Recall) / (Precision + Recall)
Specificity = TN / (TN + FP)

6.3 Detailed Example

Spam classifier results on 200 emails:
  TP = 80 (spam correctly identified)
  FP = 10 (ham marked as spam)
  FN = 20 (spam missed)
  TN = 90 (ham correctly identified)

Accuracy  = (80+90)/200 = 85%
Precision = 80/(80+10) = 88.9%   (of emails flagged spam, 88.9% were actually spam)
Recall    = 80/(80+20) = 80%     (caught 80% of all spam)
F1        = 2*(0.889*0.8)/(0.889+0.8) = 0.842 = 84.2%

6.4 ROC Curve and AUC

ROC Curve: Plot of TPR (Recall) vs FPR at various thresholds
  TPR = TP / (TP + FN) = Recall
  FPR = FP / (FP + TN) = 1 - Specificity

AUC (Area Under ROC Curve):
  AUC = 1.0: Perfect classifier
  AUC = 0.5: Random classifier (diagonal line)
  AUC < 0.5: Worse than random

Higher AUC = better model (threshold-independent metric)

7. When to Use Which Metric

ScenarioBest MetricReason
Balanced classesAccuracyWorks well when classes are equal
Imbalanced classesF1 ScoreAccuracy can be misleading
Medical diagnosisRecallMissing disease is costly (minimize FN)

GATE Tip: Be comfortable computing confusion matrix metrics by hand. Also know that accuracy is misleading for imbalanced datasets (e.g., 99% accuracy on data where 99% is one class).

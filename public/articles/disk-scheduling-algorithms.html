Disk Scheduling Algorithms

⚙️ Operating System

1. Introduction

Disk scheduling algorithms determine the order in which disk I/O requests are serviced. The goal is to minimize the total seek time (time to move the disk arm to the desired track/cylinder).

Disk Access Time = Seek Time + Rotational Latency + Transfer Time

Since seek time is the largest component and the only one affected by scheduling, algorithms focus on minimizing total head movement.

2. Common Setup for Examples

We will use the following example throughout all algorithms:

  • Disk queue (request order): 98, 183, 37, 122, 14, 124, 65, 67
  • Initial head position: 53
  • Disk range: 0 to 199 (200 cylinders)

3. FCFS (First Come First Served)

Requests are served in the order they arrive. Simple but not optimal.

Calculation

Head movement: 53 -> 98 -> 183 -> 37 -> 122 -> 14 -> 124 -> 65 -> 67

Seek distances:
|53-98|   = 45
|98-183|  = 85
|183-37|  = 146
|37-122|  = 85
|122-14|  = 108
|14-124|  = 110
|124-65|  = 59
|65-67|   = 2

Total seek time = 45 + 85 + 146 + 85 + 108 + 110 + 59 + 2 = 640 cylinders

4. SSTF (Shortest Seek Time First)

Always service the request closest to the current head position. Greedy approach.

Calculation

Start at 53.
Closest to 53: 65 (distance 12)
Closest to 65: 67 (distance 2)
Closest to 67: 37 (distance 30)
Closest to 37: 14 (distance 23)
Closest to 14: 98 (distance 84)
Closest to 98: 122 (distance 24)
Closest to 122: 124 (distance 2)
Closest to 124: 183 (distance 59)

Order: 53 -> 65 -> 67 -> 37 -> 14 -> 98 -> 122 -> 124 -> 183
Total = 12 + 2 + 30 + 23 + 84 + 24 + 2 + 59 = 236 cylinders
Note: SSTF can cause starvation for requests far from the head if new close requests keep arriving.

5. SCAN (Elevator Algorithm)

The head moves in one direction, servicing all requests until it reaches the end of the disk, then reverses direction.

Calculation (initially moving toward 0)

Start at 53, moving toward 0.
Service requests in decreasing order until 0:
53 -> 37 -> 14 -> 0 (reach end)
Reverse direction, service in increasing order:
0 -> 65 -> 67 -> 98 -> 122 -> 124 -> 183

Seek distances:
|53-37| = 16, |37-14| = 23, |14-0| = 14
|0-65| = 65, |65-67| = 2, |67-98| = 31
|98-122| = 24, |122-124| = 2, |124-183| = 59

Total = 16 + 23 + 14 + 65 + 2 + 31 + 24 + 2 + 59 = 236 cylinders

6. C-SCAN (Circular SCAN)

Like SCAN, but after reaching one end, the head jumps to the other end without servicing requests on the return trip. Provides more uniform wait times.

Calculation (moving toward 199)

Start at 53, moving toward 199.
Service in increasing order:
53 -> 65 -> 67 -> 98 -> 122 -> 124 -> 183 -> 199 (reach end)
Jump to 0 (no service during jump):
199 -> 0 -> 14 -> 37

Seek distances (counting the jump as movement):
|53-65| = 12, |65-67| = 2, |67-98| = 31, |98-122| = 24
|122-124| = 2, |124-183| = 59, |183-199| = 16
|199-0| = 199 (jump), |0-14| = 14, |14-37| = 23

Total = 12 + 2 + 31 + 24 + 2 + 59 + 16 + 199 + 14 + 23 = 382 cylinders
Note: In some formulations, the jump from end to beginning is not counted as seek time (treated as instantaneous). In that case, subtract 199 to get 183.

7. LOOK

Like SCAN, but the head only goes as far as the last request in each direction (does not go all the way to 0 or 199).

Calculation (initially moving toward 0)

Start at 53, moving toward 0.
Go to lowest request (14), not all the way to 0:
53 -> 37 -> 14
Reverse:
14 -> 65 -> 67 -> 98 -> 122 -> 124 -> 183

Seek distances:
|53-37| = 16, |37-14| = 23
|14-65| = 51, |65-67| = 2, |67-98| = 31
|98-122| = 24, |122-124| = 2, |124-183| = 59

Total = 16 + 23 + 51 + 2 + 31 + 24 + 2 + 59 = 208 cylinders

8. C-LOOK (Circular LOOK)

Like C-SCAN, but only goes as far as the last request in each direction.

Calculation (moving toward 199)

Start at 53, moving toward 199.
Go to highest request (183):
53 -> 65 -> 67 -> 98 -> 122 -> 124 -> 183
Jump to lowest request (14):
183 -> 14 -> 37

Seek distances:
|53-65| = 12, |65-67| = 2, |67-98| = 31, |98-122| = 24
|122-124| = 2, |124-183| = 59
|183-14| = 169 (jump), |14-37| = 23

Total = 12 + 2 + 31 + 24 + 2 + 59 + 169 + 23 = 322 cylinders

11. Key Points for GATE

  • Always clarify the initial direction of head movement for SCAN/LOOK variants.
  • For SCAN and C-SCAN, the head goes to the end of the disk (0 or max cylinder).
  • For LOOK and C-LOOK, the head goes only to the last request in that direction.
  • SSTF is a greedy algorithm. It does not guarantee optimal seek time globally.
  • FCFS is the only algorithm where request arrival order matters.
  • In NAT questions, count each seek distance carefully. Off-by-one errors are common.
GATE Tip: The most frequently asked question type is: "Given a request queue and initial head position, compute total seek time for [algorithm]." Practice the calculation for each algorithm until it becomes mechanical.

Divide and Conquer

⚡ Section 5: Algorithms

1. The Divide and Conquer Paradigm

Divide and Conquer (D&C) is an algorithmic paradigm that works in three steps:

  1. Divide: Break the problem into smaller subproblems of the same type.
  2. Conquer: Solve each subproblem recursively. If the subproblem is small enough, solve it directly (base case).
  3. Combine: Merge the solutions of subproblems to form the solution to the original problem.
Key Property: D&C works when subproblems are independent (non-overlapping). If subproblems overlap, use Dynamic Programming instead.

3. Merge Sort

Algorithm

MergeSort(A, low, high):
  if low < high:
    mid = (low + high) / 2
    MergeSort(A, low, mid)       // Divide: left half
    MergeSort(A, mid+1, high)    // Divide: right half
    Merge(A, low, mid, high)     // Combine: merge sorted halves

4. Quick Sort

Algorithm

QuickSort(A, low, high):
  if low < high:
    pivot_index = Partition(A, low, high)  // Divide
    QuickSort(A, low, pivot_index - 1)     // Conquer: left
    QuickSort(A, pivot_index + 1, high)    // Conquer: right

Partition(A, low, high):
  pivot = A[high]
  i = low - 1
  for j = low to high - 1:
    if A[j] ≤ pivot:
      i = i + 1
      swap A[i] and A[j]
  swap A[i+1] and A[high]
  return i + 1

5. Binary Search as D&C

Binary search follows the D&C paradigm with a single subproblem:

  • Divide: Compare target with middle element.
  • Conquer: Recurse on the left or right half (only one subproblem).
  • Combine: No combination needed — the answer comes directly from the subproblem.

Recurrence: T(n) = T(n/2) + O(1) = O(log n). Also called "decrease and conquer" since only one subproblem is generated.

6. Strassen's Matrix Multiplication

Standard matrix multiplication of two n x n matrices requires O(n3) time. Strassen's algorithm reduces this by reducing the number of recursive multiplications.

Approach

For two 2x2 matrices:

  • Standard: 8 multiplications, 4 additions.
  • Strassen: 7 multiplications, 18 additions/subtractions.

Strassen computes 7 products (M1 through M7) of specific linear combinations of submatrices, then combines them to get the result.

Complexity Analysis

Standard:   T(n) = 8T(n/2) + O(n2)  =>  O(n3)      [a=8, b=2, c_crit=3]
Strassen:   T(n) = 7T(n/2) + O(n2)  =>  O(n^2.81)  [a=7, b=2, c_crit=log2(7)=2.81]

GATE Tip: Strassen's is mainly theoretical. It is impractical for small matrices due to large constants. The key exam point: reducing 8 multiplications to 7 yields O(n^(log2 7)) = O(n^2.81).

7. Closest Pair of Points

Given n points in a 2D plane, find the pair with minimum distance.

D&C Algorithm

  1. Divide: Sort points by x-coordinate. Split into two halves by a vertical line through the median.
  2. Conquer: Recursively find the closest pair in each half. Let d = min(d_left, d_right).
  3. Combine: Check pairs that span the dividing line (strip of width 2d). For each point in the strip, compare with at most 7 subsequent points (sorted by y-coordinate).

Complexity

Recurrence: T(n) = 2T(n/2) + O(n log n)   [if sorting strip each time]
            T(n) = 2T(n/2) + O(n)          [with pre-sorting optimization]

With optimization: T(n) = O(n log n)
Critical insight: The strip can contain at most O(n) points, but each point needs to be compared with only O(1) (at most 7) other points. This keeps the combine step at O(n).

9. Other D&C Algorithms

9.1 Karatsuba Multiplication

Multiplies two n-digit numbers using 3 multiplications of n/2-digit numbers instead of 4.

T(n) = 3T(n/2) + O(n) = O(n^(log2 3)) = O(n^1.585), better than O(n2) schoolbook multiplication.

9.2 Maximum Subarray Problem

Find the contiguous subarray with the maximum sum. D&C approach: the max subarray is either entirely in the left half, entirely in the right half, or crossing the midpoint.

T(n) = 2T(n/2) + O(n) = O(n log n). Note: Kadane's algorithm solves this in O(n) using DP.

9.3 Selection Problem (Quickselect)

Find the k-th smallest element. Uses partition like quicksort but only recurses into one half.

Average: O(n). Worst: O(n2). Median-of-medians guarantees O(n) worst case.

10. Key GATE Takeaways

  • D&C subproblems must be independent. Overlapping subproblems signal DP.
  • Master theorem is the primary tool for analyzing D&C recurrences.
  • Merge sort: always O(n log n), stable, not in-place. Quick sort: O(n log n) average, O(n2) worst, in-place, not stable.
  • Strassen's reduces matrix multiplication from O(n3) to O(n^2.81) by using 7 recursive multiplications.
  • Closest pair: O(n log n) with strip optimization (compare each point with at most 7 neighbors).
  • Recursion tree method is useful when the Master Theorem does not apply.

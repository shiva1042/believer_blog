Dynamic Programming

⚡ Section 5: Algorithms

1. Fundamentals of Dynamic Programming

Dynamic Programming (DP) is an algorithmic paradigm that solves complex problems by breaking them into simpler overlapping subproblems, solving each subproblem only once, and storing the results for future use.

1.1 Two Key Properties

Overlapping Subproblems: The recursive solution involves solving the same subproblems repeatedly. DP stores these results to avoid redundant computation.

Optimal Substructure: An optimal solution to the problem can be constructed from optimal solutions to its subproblems.

2. 0/1 Knapsack Problem

Given n items with weights w[i] and values v[i], and a knapsack capacity W, find the maximum value that can be obtained by selecting a subset of items such that total weight does not exceed W. Each item is either taken entirely or not at all.

Recurrence

K(i, w) = max value using items 1..i with capacity w

K(i, w) = K(i-1, w)                           if w[i] > w
K(i, w) = max(K(i-1, w), K(i-1, w-w[i]) + v[i])  otherwise
K(0, w) = 0 for all w
K(i, 0) = 0 for all i

Example: Items = {(w=2,v=3), (w=3,v=4), (w=4,v=5), (w=5,v=6)}, W = 5

Answer: K(4, 5) = 7 (take items 1 and 2: weight = 2+3 = 5, value = 3+4 = 7).

Time Complexity: O(nW). Space: O(nW), can be reduced to O(W) using rolling array.

GATE Tip: 0/1 Knapsack is pseudo-polynomial — polynomial in n and W, but W may be exponential in the input size (number of bits to represent W).

3. Longest Common Subsequence (LCS)

Given two sequences X[1..m] and Y[1..n], find the length of the longest subsequence common to both.

Recurrence

LCS(i, j) = length of LCS of X[1..i] and Y[1..j]

LCS(i, 0) = 0, LCS(0, j) = 0
If X[i] == Y[j]:  LCS(i, j) = LCS(i-1, j-1) + 1
Else:              LCS(i, j) = max(LCS(i-1, j), LCS(i, j-1))

Example: X = "ABCBDAB", Y = "BDCAB"

4. Matrix Chain Multiplication

Given a chain of matrices A1, A2, ..., An with dimensions p0 x p1, p1 x p2, ..., p(n-1) x pn, find the optimal parenthesization that minimizes scalar multiplications.

Recurrence

m(i, j) = minimum multiplications to compute Ai...Aj

m(i, i) = 0
m(i, j) = min over k from i to j-1 of:
           m(i, k) + m(k+1, j) + p(i-1) * p(k) * p(j)

Example: Dimensions = [10, 30, 5, 60]

Matrices: A1(10x30), A2(30x5), A3(5x60)

  • (A1 * A2) * A3: 10*30*5 + 10*5*60 = 1500 + 3000 = 4500
  • A1 * (A2 * A3): 30*5*60 + 10*30*60 = 9000 + 18000 = 27000

Optimal: (A1 * A2) * A3 with 4500 multiplications.

Time: O(n3). Space: O(n2).

5. Floyd-Warshall Algorithm

Computes shortest paths between all pairs of vertices in a weighted graph (including negative weights, but no negative cycles).

Recurrence

dist(i, j, k) = shortest path from i to j using vertices {1, 2, ..., k} as intermediates

dist(i, j, 0) = weight(i, j) if edge exists, infinity otherwise
dist(i, j, k) = min(dist(i, j, k-1), dist(i, k, k-1) + dist(k, j, k-1))

Algorithm

FloydWarshall(W, n):
  dist = W  // Initialize with edge weights
  for k = 1 to n:
    for i = 1 to n:
      for j = 1 to n:
        dist[i][j] = min(dist[i][j], dist[i][k] + dist[k][j])
  return dist

Time: O(n3). Space: O(n2). Detects negative cycles: Check if any dist[i][i] < 0.

6. More Classic DP Problems

6.1 Longest Increasing Subsequence (LIS)

LIS(i) = length of LIS ending at index i
LIS(i) = 1 + max(LIS(j)) for all j < i where A[j] < A[i]
Time: O(n2) basic, O(n log n) with patience sorting / binary search.

6.2 Edit Distance (Levenshtein Distance)

dp(i, j) = edit distance between X[1..i] and Y[1..j]

If X[i] == Y[j]: dp(i,j) = dp(i-1, j-1)
Else: dp(i,j) = 1 + min(dp(i-1,j),     // delete
                         dp(i,j-1),     // insert
                         dp(i-1,j-1))   // replace
Time: O(mn), Space: O(mn)

6.3 Coin Change Problem

Minimum coins to make amount V from denominations d1, d2, ..., dk:

dp(v) = min over all coins c where c ≤ v of: dp(v - c) + 1
dp(0) = 0
Time: O(Vk)

8. Key GATE Takeaways

  • DP requires both overlapping subproblems and optimal substructure.
  • 0/1 Knapsack: O(nW) pseudo-polynomial. LCS: O(mn). Matrix Chain: O(n3).
  • Floyd-Warshall is DP for all-pairs shortest paths: O(V3).
  • Top-down is easier to code; bottom-up allows space optimization.
  • Identify the state variables and write the recurrence relation first.
  • If the problem asks "minimum/maximum" or "count number of ways," think DP.

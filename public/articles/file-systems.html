1. File Concept

A file is a named collection of related information stored on secondary storage. The OS abstracts the physical properties of storage devices to define a logical unit — the file.

1.1 File Attributes

AttributeDescription
NameHuman-readable identifier (the only attribute in human-readable form)
IdentifierUnique tag (number) within the file system
TypeInformation needed if the system supports different types
LocationPointer to the file's location on the device
SizeCurrent size in bytes, words, or blocks
ProtectionAccess control information (read, write, execute permissions)
TimestampsCreation, last modification, last access times
OwnerUser who created or owns the file

1.2 File Operations

OperationDescription
createAllocate space, create directory entry
openLocate directory entry, load metadata into memory, return file descriptor
readRead data from current file position
writeWrite data at current file position
seekReposition the current file position pointer
deleteFree allocated space, remove directory entry
truncateErase contents but keep attributes and directory entry

Most OS maintain an open-file table (per-process and system-wide) to track currently open files, avoiding repeated directory searches.

2. Directory Structure

A directory is a file-system structure that contains information about files. Different organizational schemes exist:

StructureDescriptionProsCons
Single-LevelAll files in one directorySimpleNaming collisions, no grouping
Two-LevelSeparate directory per userIsolates usersNo grouping within a user's files
Tree-StructuredHierarchical directories with subdirectoriesLogical grouping, path namesNo sharing of files/directories
Acyclic GraphTree + shared files/directories via linksAllows sharingDangling pointers on deletion; requires reference counting or back-pointers
General GraphAllows cycles (symbolic links can create cycles)Maximum flexibilityMust handle cycles in traversal (garbage collection needed)
Hard link vs Symbolic link: A hard link is another directory entry pointing to the same inode (file data). A symbolic (soft) link is a special file containing the path to the target. Hard links cannot cross file systems or link to directories; symbolic links can.

3. File Allocation Methods

File allocation methods determine how disk blocks are allocated to files.

3.1 Comparison of Allocation Methods

3.2 Contiguous Allocation

Each file occupies a set of contiguous blocks on disk. The directory entry stores the starting block and length.

ProsCons
Simple to implementExternal fragmentation
Excellent sequential and direct accessFile size must be known at creation
Minimal disk seeksDifficult to grow files

Direct access: Block i of a file is at starting_block + i. O(1) access time.

3.3 Linked Allocation

Each file is a linked list of disk blocks. Each block contains a pointer to the next block. The directory entry stores the first and last block pointers.

ProsCons
No external fragmentationNo efficient direct access (must traverse links)
Files can grow dynamicallyPointer overhead in each block
Simple to implementReliability: one bad pointer loses rest of file
FAT (File Allocation Table): An improvement over linked allocation. The pointers are collected into a separate table (FAT) at the beginning of the disk. The FAT is cached in memory, making direct access much faster. Used in FAT12, FAT16, FAT32 file systems.

3.4 Indexed Allocation

Each file has an index block (also called an inode in Unix). The index block contains an array of pointers to all the data blocks of the file.

ProsCons
Supports direct access efficientlyWasted space for small files (index block overhead)
No external fragmentationIndex block size limits max file size
No sequential traversal neededMay need multi-level index for large files

3.5 Unix Inode Structure

Unix uses a multi-level indexed scheme:

Pointer TypeCountBlocks Addressable
Direct pointers1212 blocks
Single indirect1n blocks (where n = block_size / pointer_size)
Double indirect1n2 blocks
Triple indirect1n3 blocks
Example: Block size = 4 KB, pointer size = 4 bytes. n = 4096/4 = 1024.
  • Direct: 12 × 4 KB = 48 KB
  • Single indirect: 1024 × 4 KB = 4 MB
  • Double indirect: 10242 × 4 KB = 4 GB
  • Triple indirect: 10243 × 4 KB = 4 TB
  • Maximum file size ≈ 4 TB

4. Free Space Management

The file system must track which disk blocks are free and available for allocation.

4.1 Bitmap (Bit Vector)

Each bit represents a block: 1 = free, 0 = allocated (or vice versa). Block number calculation: (number of bits per word) × (number of 0-value words) + offset of first 1 bit.

ProsCons
Simple and efficient for finding contiguous free blocksBitmap can be large for big disks
Fast using hardware bit-manipulation instructionsMust be kept on disk and cached in memory
Example: A 1 TB disk with 4 KB blocks needs 1 TB / 4 KB = 228 bits = 32 MB for the bitmap.

4.2 Linked List

All free blocks are linked together. The first free block pointer is stored in a special location. Simple but traversing the list to find free space is slow.

4.3 Other Methods

MethodDescription
GroupingFirst free block stores addresses of n free blocks; the last of these points to another group of n free blocks
CountingStore (start_block, count) pairs for contiguous runs of free blocks; efficient when blocks are contiguously allocated and freed

5. Summary and GATE Key Points

Key Points for GATE:
  • Contiguous allocation supports fast direct access but causes external fragmentation.
  • Linked allocation eliminates fragmentation but has O(n) direct access time.
  • Indexed allocation (Unix inodes) provides O(1) direct access with multi-level indexing for large files.
  • FAT improves linked allocation by storing all pointers in a table cached in memory.
  • For Unix inode: max file size = (12 + n + n2 + n3) × block_size, where n = block_size / pointer_size.
  • Bitmap free space management needs (total_blocks / 8) bytes of storage.
  • Acyclic graph directory structure allows file sharing; hard links and symbolic links are the mechanisms.
  • Number of disk accesses for reading block i: contiguous = 1, linked = i+1, indexed = 2 (index block + data block).

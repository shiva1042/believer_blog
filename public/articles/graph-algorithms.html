Graph Algorithms

⚡ Section 5: Algorithms

1. Graph Traversals

1.1 Breadth-First Search (BFS)

BFS explores vertices level by level using a queue. It visits all vertices at distance k before any vertex at distance k+1.

BFS(G, source):
  for each vertex u in G:
    color[u] = WHITE, dist[u] = infinity, parent[u] = NIL
  color[source] = GRAY, dist[source] = 0
  Q = empty queue
  Enqueue(Q, source)
  while Q is not empty:
    u = Dequeue(Q)
    for each neighbor v of u:
      if color[v] == WHITE:
        color[v] = GRAY
        dist[v] = dist[u] + 1
        parent[v] = u
        Enqueue(Q, v)
    color[u] = BLACK

Time: O(V + E). Space: O(V). Applications: Shortest path in unweighted graphs, connected components, bipartite checking.

1.2 Depth-First Search (DFS)

DFS explores as deep as possible before backtracking, using a stack (or recursion).

DFS(G):
  for each vertex u in G:
    color[u] = WHITE, parent[u] = NIL
  time = 0
  for each vertex u in G:
    if color[u] == WHITE:
      DFS-Visit(G, u)

DFS-Visit(G, u):
  time++; discover[u] = time; color[u] = GRAY
  for each neighbor v of u:
    if color[v] == WHITE:
      parent[v] = u
      DFS-Visit(G, v)
  color[u] = BLACK; time++; finish[u] = time

Time: O(V + E). Space: O(V). Applications: Topological sort, cycle detection, SCCs, articulation points, bridges.

2. Minimum Spanning Tree (MST)

A minimum spanning tree of a connected, weighted, undirected graph connects all vertices with the minimum total edge weight using exactly V-1 edges.

2.1 Kruskal's Algorithm

  1. Sort all edges by weight in ascending order.
  2. For each edge (u, v) in sorted order: if u and v are in different components, add the edge to MST.
  3. Use Union-Find (Disjoint Set) for efficient component tracking.

Time: O(E log E) for sorting + O(E * alpha(V)) for union-find = O(E log E) = O(E log V).

2.2 Prim's Algorithm

  1. Start from any vertex. Add it to the MST set.
  2. Repeatedly add the minimum-weight edge that connects a vertex in the MST to a vertex outside.
  3. Use a min-heap (priority queue) for efficiency.

Time: O(E log V) with binary heap, O(E + V log V) with Fibonacci heap.

3. Shortest Path Algorithms

3.1 Dijkstra's Algorithm

Finds shortest paths from a single source to all vertices. Works only with non-negative edge weights.

Dijkstra(G, source):
  dist[source] = 0; dist[v] = infinity for all other v
  PQ = min-priority queue with all vertices
  while PQ is not empty:
    u = ExtractMin(PQ)
    for each neighbor v of u:
      if dist[u] + weight(u,v) < dist[v]:
        dist[v] = dist[u] + weight(u,v)
        parent[v] = u
        DecreaseKey(PQ, v, dist[v])

Time: O(V2) with array, O((V+E) log V) with binary heap, O(V log V + E) with Fibonacci heap.

Dijkstra's Example

3.2 Bellman-Ford Algorithm

Handles negative edge weights and detects negative cycles.

BellmanFord(G, source):
  dist[source] = 0; dist[v] = infinity for all other v
  for i = 1 to V-1:          // Relax all edges V-1 times
    for each edge (u, v, w):
      if dist[u] + w < dist[v]:
        dist[v] = dist[u] + w
        parent[v] = u
  // Check for negative cycles
  for each edge (u, v, w):
    if dist[u] + w < dist[v]:
      report "Negative cycle exists"

Time: O(VE). Space: O(V).

3.3 Floyd-Warshall Algorithm

Computes all-pairs shortest paths using dynamic programming.

Time: O(V3). Space: O(V2). Handles negative weights but not negative cycles.

4. Topological Sorting

A linear ordering of vertices of a DAG (Directed Acyclic Graph) such that for every edge (u, v), u appears before v.

Methods

  1. DFS-based: Run DFS and output vertices in reverse order of finish times. Time: O(V+E).
  2. Kahn's Algorithm (BFS-based): Repeatedly remove vertices with in-degree 0. Time: O(V+E).
Kahn's Algorithm:
  Compute in-degree for all vertices
  Q = queue of all vertices with in-degree 0
  while Q is not empty:
    u = Dequeue(Q)
    output u
    for each neighbor v of u:
      in-degree[v]--
      if in-degree[v] == 0:
        Enqueue(Q, v)
  if count of output vertices < V:
    "Graph has a cycle" (not a DAG)
GATE Tip: Topological sort is only possible for DAGs. If DFS finds a back edge, the graph has a cycle and topological ordering is impossible. The number of possible topological orderings can vary.

5. Strongly Connected Components (SCCs)

An SCC is a maximal set of vertices such that there is a path from every vertex to every other vertex in the set. Applies to directed graphs.

Kosaraju's Algorithm

  1. Run DFS on the original graph G. Record finish times.
  2. Compute the transpose graph G^T (reverse all edges).
  3. Run DFS on G^T in decreasing order of finish times from step 1.
  4. Each DFS tree in step 3 is an SCC.

Time: O(V + E). Alternative: Tarjan's algorithm (single DFS pass).

7. Key GATE Takeaways

  • BFS and DFS both run in O(V + E) time.
  • BFS uses a queue and gives shortest path in unweighted graphs. DFS uses a stack (recursion).
  • Kruskal's is better for sparse graphs; Prim's for dense graphs. Both produce MST in O(E log V).
  • Dijkstra requires non-negative weights. Use Bellman-Ford for negative weights.
  • Floyd-Warshall is O(V3) and solves all-pairs shortest paths.
  • Topological sort applies only to DAGs. Back edge in DFS implies a cycle.
  • Kosaraju's and Tarjan's algorithms find SCCs in O(V+E).

Greedy Algorithms

⚡ Section 5: Algorithms

1. Greedy Strategy Fundamentals

A greedy algorithm makes the locally optimal choice at each step, hoping to find a global optimum. It never reconsiders past decisions.

1.1 Greedy Choice Property

A globally optimal solution can be arrived at by making locally optimal (greedy) choices. At each step, we make the choice that looks best at the moment without worrying about future consequences.

1.2 Optimal Substructure

An optimal solution to the problem contains within it optimal solutions to subproblems. This property is shared with dynamic programming, but greedy algorithms differ in that they make a single choice and proceed, whereas DP explores all choices.

Key Difference from DP: Greedy makes one choice per step and never backtracks. DP considers all possible choices and picks the best after computing results for each.

2. Activity Selection Problem

Given n activities with start times s_i and finish times f_i, select the maximum number of non-overlapping activities.

Greedy Strategy

  1. Sort activities by finish time in ascending order.
  2. Select the first activity.
  3. For each subsequent activity, select it if its start time is greater than or equal to the finish time of the last selected activity.
ActivitySelection(s, f, n):
  Sort activities by finish time
  selected = {a_1}
  last_finish = f[1]
  for i = 2 to n:
    if s[i] ≥ last_finish:
      selected = selected + {a_i}
      last_finish = f[i]
  return selected

Result: {A1, A3, A6} — 3 activities selected. Time Complexity: O(n log n) for sorting.

3. Huffman Coding

Huffman coding is a greedy algorithm for lossless data compression. It assigns variable-length prefix codes to characters based on their frequency — more frequent characters get shorter codes.

Algorithm

  1. Create a leaf node for each character with its frequency.
  2. Insert all nodes into a min-heap (priority queue).
  3. While more than one node remains in the heap:
    1. Extract the two nodes with minimum frequency.
    2. Create a new internal node with frequency = sum of the two.
    3. Make the two extracted nodes children of the new node.
    4. Insert the new node into the heap.
  4. The remaining node is the root of the Huffman tree.

4. Fractional Knapsack

Given items with weights w_i and values v_i, and a knapsack of capacity W, maximize total value. Unlike 0/1 knapsack, we can take fractions of items.

Greedy Strategy

  1. Compute value-to-weight ratio (v_i / w_i) for each item.
  2. Sort items by ratio in decreasing order.
  3. Take items greedily: take the entire item if it fits, otherwise take the fraction that fills the remaining capacity.

Example

ItemWeightValueRatioAmount Taken
A10606.0Full (10)
B201005.0Full (20)
C301204.0Partial (20/30)

Knapsack capacity W = 50. Total value = 60 + 100 + (20/30)*120 = 60 + 100 + 80 = 240.

Time Complexity: O(n log n) for sorting.

Important: Greedy works for fractional knapsack but NOT for 0/1 knapsack. The 0/1 version requires dynamic programming.

5. Job Sequencing with Deadlines

Given n jobs, each with a deadline d_i and profit p_i, schedule jobs on a single machine to maximize total profit. Each job takes one unit of time.

Greedy Strategy

  1. Sort jobs by profit in decreasing order.
  2. For each job, find the latest available time slot before its deadline.
  3. If a slot is available, schedule the job; otherwise, skip it.

6. More Greedy Problems

6.1 Minimum Number of Coins (Change Making)

Greedy works for canonical coin systems (e.g., US coins: 25, 10, 5, 1). Always pick the largest denomination that does not exceed the remaining amount.

Caution: Greedy fails for non-canonical systems. For coins {1, 3, 4}, making change for 6: greedy gives 4+1+1 (3 coins), optimal is 3+3 (2 coins).

6.2 Minimum Number of Platforms

Given arrival and departure times of trains, find the minimum number of platforms required. Sort events, use greedy counting of overlapping intervals.

8. Proving Greedy Correctness

Two common proof techniques:

  1. Greedy stays ahead: Show at every step, the greedy solution is at least as good as any other solution.
  2. Exchange argument: Start with an optimal solution and show it can be transformed into the greedy solution without worsening it.

9. Key GATE Takeaways

  • Greedy algorithms require both greedy choice property and optimal substructure.
  • Activity selection: sort by finish time. Fractional knapsack: sort by value/weight ratio.
  • Huffman coding produces an optimal prefix-free code. Time: O(n log n).
  • Greedy fails for 0/1 knapsack — use DP instead.
  • Always verify greedy correctness with exchange argument or "greedy stays ahead" proof.
  • MST algorithms (Kruskal's, Prim's) and Dijkstra's algorithm are greedy algorithms.

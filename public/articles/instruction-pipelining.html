Instruction Pipelining

🖥️ Section 3: Computer Organization

1. Introduction

Pipelining is a technique where multiple instructions are overlapped in execution. Like an assembly line in a factory, different stages of different instructions execute simultaneously. Pipelining increases throughput (instructions completed per unit time) without reducing the latency of individual instructions.

2. Pipeline Stages

A classic 5-stage RISC pipeline:

StageAbbreviationAction
Instruction FetchIFFetch instruction from memory using PC
Instruction DecodeIDDecode instruction, read registers
ExecuteEXALU operation or address computation
Memory AccessMEMRead/write data memory (for loads/stores)
Write BackWBWrite result to register file

3. Performance Formulas

3.1 Execution Time

MetricWithout PipelineWith Pipeline
Time for n instructions, k stagesn × k × t(k + n - 1) × t
Clock period (t)Sum of all stage delaysmax(stage delays) + latch delay
CPI (ideal)k1

3.2 Speedup

Speedup = Time without pipeline / Time with pipeline

S = (n × k × t) / ((k + n - 1) × t) = (n × k) / (k + n - 1)

As n approaches infinity: S → k (maximum speedup equals number of stages).

3.3 Throughput

Throughput = n / ((k + n - 1) × t)

As n approaches infinity: Throughput → 1/t (one instruction per clock cycle).

Example: A 5-stage pipeline, each stage takes 2 ns. Execute 100 instructions.
Without pipeline: 100 × 5 × 2 = 1000 ns
With pipeline: (5 + 100 - 1) × 2 = 208 ns
Speedup = 1000/208 = 4.81x (approaching the ideal 5x)

3.4 Effective CPI with Stalls

Effective CPI = Ideal CPI + Stall cycles per instruction

Effective CPI = 1 + (stall frequency × stall cycles)

4. Pipeline Hazards

Hazards are situations that prevent the next instruction from executing in its designated clock cycle. There are three types:

4.1 Structural Hazards

Occur when hardware resources are insufficient to support all concurrent instructions. Example: a single memory port shared by IF and MEM stages.

CauseSolution
Single memory for instruction and dataSeparate instruction and data caches (Harvard architecture)
Single-port register fileMulti-port register file or read/write in different half-cycles
Shared functional unitDuplicate the functional unit

4.2 Data Hazards

Occur when an instruction depends on the result of a previous instruction still in the pipeline.

Types of Data Dependencies

TypeAbbreviationExampleDescription
Read After WriteRAWADD R1,R2,R3; SUB R4,R1,R5True dependency: SUB needs R1 written by ADD
Write After ReadWARADD R1,R2,R3; SUB R2,R4,R5Anti-dependency: SUB writes R2 before ADD reads it
Write After WriteWAWADD R1,R2,R3; SUB R1,R4,R5Output dependency: both write R1

In a standard 5-stage in-order pipeline, only RAW hazards cause stalls. WAR and WAW hazards can occur in out-of-order or longer pipelines.

RAW Hazard Example

I1: ADD R1, R2, R3 writes R1 in WB (cycle 5). I2: SUB R4, R1, R5 needs R1 in ID (cycle 3). This causes a 2-cycle stall without forwarding.

4.3 Control Hazards

Occur with branch instructions because the next instruction address is not known until the branch condition is evaluated.

ProblemPenalty
Unconditional branch1-2 cycles (until target computed)
Conditional branch1-3 cycles (until condition resolved)

5. Hazard Solutions

5.1 Data Forwarding (Bypassing)

The ALU result is forwarded directly from the EX/MEM or MEM/WB pipeline register to the ALU input, bypassing the register file write.

flowchart TD subgraph Without_Forwarding["Without Forwarding: 2 Stall Cycles"] A1["I1: IF ID EX MEM WB"] A2["I2: IF ID -- -- EX MEM WB"] end subgraph With_Forwarding["With Forwarding: 0 Stall Cycles"] B1["I1: IF ID EX MEM WB"] B2["I2: IF ID EX MEM WB"] B1 -. "Forward from EX/MEM" .-> B2 end

Load-Use Hazard: Even with forwarding, a LOAD followed by a dependent instruction requires one stall cycle because the data is available after MEM, not after EX.

LOAD R1, 0(R2) followed by ADD R3, R1, R4 requires 1 stall (bubble).

5.2 Stalling (Pipeline Bubbles)

Insert NOP (no-operation) cycles into the pipeline to delay the dependent instruction. The hardware "freezes" the pipeline stages before the hazard and inserts bubbles.

5.4 Delayed Branching

The compiler fills the branch delay slot(s) with useful instructions that execute regardless of the branch outcome.

6. Pipeline Performance with Stalls

Example: 5-stage pipeline, 20% of instructions are loads, 50% of loads are followed by a dependent instruction (1 stall cycle with forwarding), 15% of instructions are branches with 2-cycle penalty and 80% prediction accuracy.

Stalls per instruction = (0.20 × 0.50 × 1) + (0.15 × 0.20 × 2) = 0.10 + 0.06 = 0.16

Effective CPI = 1 + 0.16 = 1.16

Speedup over non-pipelined = 5 / 1.16 = 4.31x

7. Pipeline Depth Trade-offs

More Stages (Deeper Pipeline)Fewer Stages (Shallower Pipeline)
Higher clock frequencyLower clock frequency
More hazard penaltiesFewer hazard penalties
More pipeline registers (overhead)Less overhead
Higher branch misprediction costLower branch misprediction cost

8. GATE Exam Tips

  • Master the speedup formula: S = nk / (k + n - 1). As n → infinity, S → k.
  • RAW is the most important hazard in a standard 5-stage pipeline.
  • With forwarding: LOAD-use hazard still needs 1 stall. All other RAW hazards are resolved.
  • Without forwarding: up to 2 stalls for ALU-dependent and up to 2 stalls for load-dependent instructions.
  • Branch penalty = cycles wasted when branch is mispredicted.
  • Effective CPI = 1 + sum of (frequency × penalty) for each hazard type.
  • Pipeline clock period = max(stage delay) + register overhead.

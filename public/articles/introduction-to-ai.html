Introduction to Artificial Intelligence

Section: Artificial Intelligence

1. What is Artificial Intelligence?

Artificial Intelligence (AI) is the branch of computer science concerned with building systems that can perform tasks normally requiring human intelligence. These tasks include reasoning, learning, perception, natural language understanding, and problem solving.

There are four classical approaches to defining AI, organized along two dimensions: thinking vs. acting and humanly vs. rationally:

HumanlyRationally
ThinkingCognitive modeling — systems that think like humans (e.g., cognitive science)Laws of thought — systems that think rationally (e.g., logic-based AI)
ActingTuring Test — systems that act like humansRational agents — systems that act rationally (dominant modern view)

1.1 The Turing Test

Proposed by Alan Turing in 1950, the Turing Test evaluates a machine's ability to exhibit intelligent behavior indistinguishable from a human. A machine passes the test if a human interrogator cannot reliably distinguish the machine from a human based on their responses. The total Turing Test also requires computer vision and robotics capabilities.

3. Types of Artificial Intelligence

3.1 Narrow AI (Weak AI)

Designed to perform a specific task. All current AI systems fall into this category. Examples: speech recognition, image classification, recommendation systems, game-playing AI.

3.2 General AI (Strong AI / AGI)

A hypothetical system with human-level intelligence that can understand, learn, and apply knowledge across diverse domains. AGI does not yet exist.

3.3 Super AI

A hypothetical AI that surpasses human intelligence in all domains including creativity, problem-solving, and social skills. This remains a theoretical concept.

4. Intelligent Agents

An agent is anything that perceives its environment through sensors and acts upon the environment through actuators. An agent function maps percept sequences to actions.

Agent Function: f : P* → A
where P* = set of all possible percept sequences, A = set of actions

A rational agent selects actions that maximize its expected performance measure, given the percept sequence and its built-in knowledge.

4.1 PEAS Framework

To design a rational agent, we specify its PEAS description:

ComponentDescriptionExample (Self-Driving Car)
Performance MeasureHow success is evaluatedSafety, reaching destination, comfort, fuel efficiency
EnvironmentThe external world the agent operates inRoads, traffic, pedestrians, weather
ActuatorsMechanisms for taking actionsSteering, accelerator, brake, signals
SensorsMechanisms for perceiving the environmentCameras, LIDAR, GPS, speedometer
Example — PEAS for a Medical Diagnosis System:
  • Performance: Correct diagnosis rate, patient health outcomes
  • Environment: Patient, hospital, medical records
  • Actuators: Display diagnosis, prescribe treatment
  • Sensors: Keyboard input of symptoms, lab reports, medical imaging

5. Types of Agents

5.1 Simple Reflex Agent

Selects actions based solely on the current percept, ignoring percept history. Uses condition-action rules (if-then rules). Works only in fully observable environments.

if condition then action
Example: if car-in-front-brakes then apply-brakes

5.2 Model-Based Reflex Agent

Maintains an internal state (model of the world) that tracks aspects of the environment not directly visible. Can handle partially observable environments. The internal state is updated using knowledge of how the world evolves and how actions affect the world.

5.3 Goal-Based Agent

Has explicit goals and selects actions that achieve those goals. Uses search and planning to determine the sequence of actions needed. More flexible than reflex agents because behavior can change by modifying goals.

5.4 Utility-Based Agent

Uses a utility function that maps states to real numbers, representing the degree of happiness/satisfaction. When multiple action sequences achieve the goal, the agent selects the one with the highest expected utility. Handles trade-offs between conflicting goals.

U : S → R (utility function maps states to real-valued utility)

5.5 Learning Agent

Has four conceptual components:

  • Learning element: Makes improvements by learning from experience
  • Performance element: Selects external actions (the agent function)
  • Critic: Provides feedback on how the agent is performing
  • Problem generator: Suggests exploratory actions for new experiences

  • Poker: Partially observable, multi-agent, stochastic, sequential, static, discrete, known

  • Crossword puzzle: Fully observable, single agent, deterministic, sequential, static, discrete, known

7. Key Concepts for GATE

  • A rational agent is not omniscient — it acts based on available percepts and knowledge.
  • Rationality depends on: performance measure, prior knowledge, possible actions, and percept sequence to date.
  • The most complex environment type is: partially observable, multi-agent, stochastic, sequential, dynamic, continuous, and unknown.
  • Simple reflex agents fail in partially observable environments; model-based agents address this limitation.
  • Utility-based agents are the most general framework, subsuming goal-based agents.

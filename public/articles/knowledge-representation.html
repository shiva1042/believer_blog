Knowledge Representation and Reasoning

Section: Artificial Intelligence

1. Propositional Logic Recap

Propositional logic deals with propositions (statements that are true or false) and logical connectives. While powerful, it has significant limitations: it cannot express relationships between objects, quantify over variables, or represent partial information naturally.

ConnectiveSymbolMeaning
Negation¬pNOT p
Conjunctionp ∧ qp AND q
Disjunctionp ∨ qp OR q
Implicationp → qif p then q
Biconditionalp ↔ qp if and only if q

Limitations of propositional logic: Cannot represent "All humans are mortal" or "There exists a prime number greater than 100" without enumerating all instances.

2. First-Order Logic (FOL)

First-order logic (also called predicate logic) extends propositional logic with variables, quantifiers, predicates, and functions, enabling us to express general statements about objects and their relationships.

2.1 Components of FOL

ComponentDescriptionExamples
ConstantsSpecific objectsJohn, 5, BlockA
VariablesRange over objectsx, y, z
PredicatesRelations/propertiesHuman(x), Loves(x, y), Greater(x, y)
FunctionsMappings to objectsFather(x), Plus(x, y)
QuantifiersScope over variables∀ (universal), ∃ (existential)
ConnectivesLogical operators¬, ∧, ∨, →, ↔

2.2 Quantifiers

Universal: ∀x P(x) — "For all x, P(x) is true"
Existential: ∃x P(x) — "There exists an x such that P(x) is true"

Key equivalences:

¬(∀x P(x)) ≡ ∃x ¬P(x)
¬(∃x P(x)) ≡ ∀x ¬P(x)
∀x P(x) typically pairs with → : ∀x (Human(x) → Mortal(x))
∃x P(x) typically pairs with ∧ : ∃x (Student(x) ∧ Smart(x))
Example Translations:
  • "All dogs are animals" → ∀x (Dog(x) → Animal(x))
  • "Some students love AI" → ∃x (Student(x) ∧ Loves(x, AI))
  • "Everyone who is a parent loves their children" → ∀x ∀y (Parent(x, y) → Loves(x, y))
  • "No robot is human" → ∀x (Robot(x) → ¬Human(x)) or ¬∃x (Robot(x) ∧ Human(x))

3. Inference Rules

3.1 Modus Ponens

From: p → q and p
Conclude: q

Generalized Modus Ponens for FOL: if we have p1 ∧ p2 ∧ ... ∧ pn → q and all premises p1', p2', ..., pn' that unify with p1, p2, ..., pn, we can conclude the substituted q.

3.2 Modus Tollens

From: p → q and ¬q
Conclude: ¬p

3.3 Resolution

Resolution is a powerful and complete inference rule for clausal form (disjunctions of literals):

From: (p ∨ r) and (¬p ∨ q)
Conclude: (r ∨ q) [resolvent]

For FOL, resolution requires unification to match complementary literals. Resolution is refutation-complete: to prove a statement, negate it and show that resolution derives the empty clause (contradiction).

4. Unification

Unification finds a substitution that makes two expressions identical. The most general unifier (MGU) is the simplest such substitution.

Unification Examples:
  • Knows(John, x) and Knows(John, Jane) → MGU: {x/Jane}
  • Knows(John, x) and Knows(y, Bill) → MGU: {y/John, x/Bill}
  • Knows(John, x) and Knows(x, Bill) → Fails (x cannot be both John and Bill)
  • Knows(John, x) and Knows(y, Mother(y)) → MGU: {y/John, x/Mother(John)}
Occur check: x and f(x) cannot unify (x would have to contain itself).

5. Forward and Backward Chaining

5.1 Forward Chaining

Data-driven reasoning. Starts with known facts and applies inference rules to derive new facts until the goal is reached or no more rules apply.

  • Applies all applicable rules in each iteration
  • Sound and complete for definite clause knowledge bases
  • May derive many irrelevant conclusions
  • Used in: production systems, business rule engines

5.2 Backward Chaining

Goal-driven reasoning. Starts with the goal and works backward, finding rules whose conclusions match the goal and setting up their premises as new sub-goals.

  • Only considers relevant rules
  • Much more focused than forward chaining
  • May suffer from infinite loops without cycle detection
  • Used in: Prolog, expert systems, diagnostic systems
Example:
Knowledge base: (1) A → B, (2) B ∧ C → D, (3) A is true, (4) C is true.
Forward chaining: Start with {A, C}. Apply rule 1: derive B → {A, B, C}. Apply rule 2: derive D → {A, B, C, D}.
Backward chaining for D: Goal = D. Rule 2 can derive D if B ∧ C. Sub-goal B: Rule 1 can derive B if A. Sub-goal A: A is a known fact. Sub-goal C: C is a known fact. Therefore D is proved.

6. Semantic Networks

A semantic network represents knowledge as a graph where nodes represent concepts/objects and edges represent relationships between them.

RelationshipMeaningExample
IS-AClass membership / subclassDog IS-A Animal
HAS-APart-whole / possessionCar HAS-A Engine
INSTANCE-OFIndividual is member of classRex INSTANCE-OF Dog

Property inheritance: Properties are inherited through IS-A links. If "Animal HAS property Breathes" and "Dog IS-A Animal", then Dog inherits the property Breathes. Exceptions can override inherited properties.

7. Frames

Frames are structured knowledge representation schemes. Each frame represents a class or instance and contains slots (attributes) with fillers (values).

Example Frame:
Frame: Dog
  IS-A: Animal
  Legs: 4
  Sound: Bark
  Has-Tail: True
  Diet: Omnivore

Frame: Rex
  INSTANCE-OF: Dog
  Breed: Labrador
  Color: Golden
  Age: 5

Frames support default values (can be overridden), inheritance through IS-A hierarchies, and procedural attachments (if-needed, if-added, if-removed demons).

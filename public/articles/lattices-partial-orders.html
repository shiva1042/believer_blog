Lattices, Partial Orders, and Boolean Algebra

This article provides a comprehensive treatment of partially ordered sets, lattices, and Boolean algebra as covered in Rosen's Discrete Mathematics and Its Applications (Chapters 9 and 12). These topics are heavily tested in GATE CSE under Engineering Mathematics. Understanding Hasse diagrams, special elements, lattice types, and Boolean algebra is essential for scoring well.

1. Relations and Partial Orders — Foundations

1.1 Binary Relations Recap

A binary relation R on a set S is a subset of S × S. We write a R b (or (a, b) ∈ R) to mean that a is related to b.

1.2 Properties of Relations

Property Definition Example on Z
Reflexive ∀a ∈ S, a R a a ≤ a ✓
Antisymmetric If a R b and b R a, then a = b a ≤ b and b ≤ a ⇒ a = b ✓
Transitive If a R b and b R c, then a R c a ≤ b and b ≤ c ⇒ a ≤ c ✓
Symmetric If a R b, then b R a ≤ is NOT symmetric
Total/Linear ∀a, b: a R b or b R a ≤ on Z is total ✓

1.3 Partial Order (Poset)

A relation ≤ on a set S is a partial order if it is:

  1. Reflexive: a ≤ a for all a ∈ S.
  2. Antisymmetric: a ≤ b and b ≤ a implies a = b.
  3. Transitive: a ≤ b and b ≤ c implies a ≤ c.

The pair (S, ≤) is called a partially ordered set (poset).

Mnemonic — RAT: Reflexive, Antisymmetric, Transitive.

2. Total Orders vs. Partial Orders

A total order (also called a linear order) is a partial order where every two elements are comparable:

∀a, b ∈ S: a ≤ b or b ≤ a

Feature Partial Order Total Order
Comparability Some pairs may be incomparable Every pair is comparable
Hasse diagram shape Can have "branches" (width > 1) Always a single chain (path)
Example (P({a,b,c}), ⊆) (Z, ≤)
Maximal = Greatest? Not necessarily Yes, always

3. Hasse Diagrams

A Hasse diagram is a visual representation of a finite poset that removes redundant information:

  1. Remove self-loops (reflexive edges).
  2. Remove transitive edges: if a ≤ b ≤ c, do not draw the edge a — c.
  3. Orient upward: if a ≤ b, place b above a. No arrows needed.

Example: Divisibility on {1, 2, 3, 4, 6, 12}

The poset ({1, 2, 3, 4, 6, 12}, |) has the following Hasse diagram:

graph BT 1["1"] --> 2["2"] 1 --> 3["3"] 2 --> 4["4"] 2 --> 6["6"] 3 --> 6 4 --> 12["12"] 6 --> 12

Reading the diagram: 1 divides everything (bottom). 12 is divisible by everything (top). 4 and 6 are incomparable (neither divides the other). 2 and 3 are incomparable.

Example: Power Set P({a, b})

The poset (P({a,b}), ⊆):

graph BT E["{∅}"] --> A["{a}"] E --> B["{b}"] A --> AB["{a, b}"] B --> AB

Example: Power Set P({a, b, c})

The poset (P({a,b,c}), ⊆):

graph BT E["{∅}"] --> A["{a}"] E --> B["{b}"] E --> C["{c}"] A --> AB["{a,b}"] A --> AC["{a,c}"] B --> AB B --> BC["{b,c}"] C --> AC C --> BC AB --> ABC["{a,b,c}"] AC --> ABC BC --> ABC

How to Read a Hasse Diagram

  • a ≤ b if and only if there is an upward path from a to b.
  • Two elements are incomparable if there is no upward path between them in either direction.
  • A direct edge (no intermediate elements) means b covers a: a < b and there is no c with a < c < b.

4. Special Elements in Posets

Let (S, ≤) be a poset and A ⊆ S.

Element Definition Uniqueness
Maximal of S No element in S is strictly greater: ¬∃b ∈ S, a < b May be multiple
Minimal of S No element in S is strictly less: ¬∃b ∈ S, b < a May be multiple
Greatest (maximum) of S ∀b ∈ S, b ≤ a Unique if exists
Least (minimum) of S ∀b ∈ S, a ≤ b Unique if exists
Upper Bound of A u ∈ S such that ∀a ∈ A, a ≤ u May be multiple or none
Lower Bound of A l ∈ S such that ∀a ∈ A, l ≤ a May be multiple or none
LUB / Supremum / Join (a ∨ b) Least upper bound: smallest element among all upper bounds of A Unique if exists
GLB / Infimum / Meet (a ∧ b) Greatest lower bound: largest element among all lower bounds of A Unique if exists
Key GATE Distinction: Maximal ≠ Greatest.
A greatest element must be comparable to and greater than or equal to ALL elements. A maximal element just has nothing above it — there may be incomparable elements elsewhere.
In a total order, maximal = greatest (if the set is finite). In a partial order, they can differ.

Worked Example

Consider the poset ({1, 2, 3, 4, 6, 12}, |) and the subset A = {2, 3}.

  • Upper bounds of A: Elements divisible by both 2 and 3, i.e., divisible by lcm(2,3) = 6. UB = {6, 12}.
  • LUB (join) of A: The least element in {6, 12} under divisibility is 6. So 2 ∨ 3 = 6.
  • Lower bounds of A: Elements that divide both 2 and 3, i.e., divisors of gcd(2,3) = 1. LB = {1}.
  • GLB (meet) of A: The greatest element in {1} is 1. So 2 ∧ 3 = 1.

Another Example

In (P({a,b,c}), ⊆), let A = {{a}, {b}}.

  • Upper bounds: Sets containing both a and b: {{a,b}, {a,b,c}}.
  • LUB: {a,b} (the smallest such set). So {a} ∨ {b} = {a, b}.
  • Lower bounds: Sets contained in both {a} and {b}: {∅}.
  • GLB: ∅. So {a} ∧ {b} = ∅.

5. Chains and Antichains

Let (S, ≤) be a poset.

  • A chain is a subset C ⊆ S where every two elements in C are comparable. (A chain is a totally ordered subset.)
  • An antichain is a subset A ⊆ S where no two distinct elements in A are comparable.

Example

In ({1, 2, 3, 4, 6, 12}, |):

  • Chain: {1, 2, 4, 12} — each divides the next.
  • Chain: {1, 3, 6, 12}.
  • Antichain: {4, 3} — 4 does not divide 3 and 3 does not divide 4.
  • Antichain: {4, 6, 3} — no two are related by divisibility? Actually 3 | 6, so {4, 3} is an antichain but {4, 6, 3} is not.
  • Maximum antichain: {4, 3} has size 2. (In fact, the width of this poset is 2.)

Dilworth's Theorem

Theorem: In any finite poset, the minimum number of chains needed to partition the poset equals the maximum size of an antichain (the width of the poset).

Mirsky's Theorem

Theorem: The minimum number of antichains needed to partition the poset equals the length of the longest chain (the height of the poset).

GATE Tip: Questions on chains and antichains often ask for the length of the longest chain or the size of the maximum antichain in a divisibility poset or power set poset.

6. Lattices

6.1 Definition

A poset (L, ≤) is a lattice if for every pair of elements a, b ∈ L, both a ∨ b (join / LUB) and a ∧ b (meet / GLB) exist.

Critical: For a poset to be a lattice, every pair must have both a join and a meet. If even one pair fails, it is NOT a lattice.

6.2 Examples of Lattices

(Z, ≤)max(a, b)min(a, b)Yes

6.3 Non-Lattice Example

Consider the poset with Hasse diagram:

graph BT A["a"] --> C["c"] A --> D["d"] B["b"] --> C B --> D

Here, a ∨ b should be the LUB of {a, b}. The upper bounds are {c, d}, but neither c ≤ d nor d ≤ c (they are incomparable). So there is no least upper bound. This poset is NOT a lattice.

6.4 Properties of Lattices

In any lattice (L, ∧, ∨), the following hold for all a, b, c ∈ L:

PropertyMeet (∧)Join (∨)
Commutativea ∧ b = b ∧ aa ∨ b = b ∨ a
Associative(a ∧ b) ∧ c = a ∧ (b ∧ c)(a ∨ b) ∨ c = a ∨ (b ∨ c)
Idempotenta ∧ a = aa ∨ a = a
Absorptiona ∧ (a ∨ b) = aa ∨ (a ∧ b) = a
Absorption Laws: These are unique to lattices and are often tested. They capture the idea that join and meet "absorb" each other. From these four properties, you can recover the partial order: a ≤ b iff a ∧ b = a (equivalently, a ∨ b = b).

7. Types of Lattices

7.1 Bounded Lattice

A lattice is bounded if it has a greatest element (denoted 1) and a least element (denoted 0):

  • ∀a: a ∨ 1 = 1 and a ∧ 1 = a (1 is the identity for meet)
  • ∀a: a ∧ 0 = 0 and a ∨ 0 = a (0 is the identity for join)

Every finite lattice is bounded (the join of all elements gives 1; the meet of all elements gives 0).

7.2 Complemented Lattice

A bounded lattice is complemented if for every element a, there exists an element a' (called the complement) such that:

  • a ∨ a' = 1
  • a ∧ a' = 0

Example: (P(S), ⊆) is complemented: the complement of A is S \ A.

Example: (D30, |) = {1, 2, 3, 5, 6, 10, 15, 30} is complemented. For instance, complement of 6 is 5 (since lcm(6,5) = 30 and gcd(6,5) = 1).

Warning: In a complemented lattice, complements need not be unique (unless the lattice is also distributive). In a Boolean algebra, complements are always unique.

7.3 Distributive Lattice

A lattice is distributive if for all a, b, c:

  • a ∧ (b ∨ c) = (a ∧ b) ∨ (a ∧ c)
  • a ∨ (b ∧ c) = (a ∨ b) ∧ (a ∨ c)

(In a lattice, either distributive law implies the other.)

Characterization: When is a lattice NOT distributive?

Theorem (Birkhoff): A lattice is non-distributive if and only if it contains a sublattice isomorphic to one of these two:

N5 (Pentagon):

graph BT O["0"] --> A["a"] O --> B["b"] A --> C["c"] C --> I["1"] B --> I

M3 (Diamond):

graph BT O["0"] --> A["a"] O --> B["b"] O --> C["c"] A --> I["1"] B --> I C --> I

M3 is a lattice but NOT distributive: a ∧ (b ∨ c) = a ∧ 1 = a, but (a ∧ b) ∨ (a ∧ c) = 0 ∨ 0 = 0 ≠ a.

GATE Tip: To check if a lattice is distributive, look for sublattices isomorphic to M3 or N5. If neither exists, the lattice is distributive. Also: in a distributive lattice, every element has at most one complement.

7.4 Summary of Lattice Types

8. Boolean Algebra as a Lattice

8.1 Definition

A Boolean algebra is a complemented distributive lattice. Equivalently, it is a bounded lattice (B, ∧, ∨, ', 0, 1) satisfying:

  1. Commutative, associative, and absorption laws (lattice axioms).
  2. Distributive laws: a ∧ (b ∨ c) = (a ∧ b) ∨ (a ∧ c) and its dual.
  3. Complement laws: a ∨ a' = 1 and a ∧ a' = 0.

8.2 Key Theorems

Theorem (Uniqueness of Complement): In a Boolean algebra, every element has exactly one complement.

Proof: Suppose a has two complements b and c. Then:

b = b ∧ 1 = b ∧ (a ∨ c) = (b ∧ a) ∨ (b ∧ c) = 0 ∨ (b ∧ c) = b ∧ c

c = c ∧ 1 = c ∧ (a ∨ b) = (c ∧ a) ∨ (c ∧ b) = 0 ∨ (c ∧ b) = c ∧ b = b ∧ c

Therefore b = b ∧ c = c.  ■

Theorem (De Morgan's Laws): In any Boolean algebra:

  • (a ∨ b)' = a' ∧ b'
  • (a ∧ b)' = a' ∨ b'

Theorem (Involution): (a')' = a.

8.3 The Canonical Boolean Algebra

The power set lattice (P(S), ∪, ∩, complement, ∅, S) is the prototypical Boolean algebra. In fact:

Stone's Representation Theorem: Every finite Boolean algebra is isomorphic to (P(S), ⊆) for some finite set S. Consequently, every finite Boolean algebra has 2n elements for some n ≥ 0.

GATE Corollary: If a Boolean algebra has k elements, then k must be a power of 2. It has exactly n = log2(k) atoms (minimal non-zero elements).

8.4 Atoms

An atom in a Boolean algebra is an element a ≠ 0 such that there is no element b with 0 < b < a. In P(S), the atoms are the singleton sets {x} for x ∈ S.

Every element in a finite Boolean algebra can be uniquely expressed as the join of atoms below it. This is the lattice-theoretic analogue of writing a set as the union of its singleton subsets.

9. Isomorphism of Lattices

Two lattices (L1, ≤1) and (L2, ≤2) are isomorphic if there exists a bijection f: L1 → L2 such that:

a ≤1 b ⇔ f(a) ≤2 f(b)

Equivalently, f preserves both join and meet:

  • f(a ∨ b) = f(a) ∨ f(b)
  • f(a ∧ b) = f(a) ∧ f(b)

Example

(D6, |) = ({1, 2, 3, 6}, |) is isomorphic to (P({a, b}), ⊆).

The mapping: 1 → ∅, 2 → {a}, 3 → {b}, 6 → {a, b}.

Both have the same diamond-shaped Hasse diagram:

graph BT Bot["1 / ∅"] --> L["2 / {a}"] Bot --> R["3 / {b}"] L --> Top["6 / {a,b}"] R --> Top

How to Check Isomorphism (GATE Strategy)

  1. Both lattices must have the same number of elements.
  2. Their Hasse diagrams must have the same structure (same "shape").
  3. Corresponding elements must preserve the order relation.
  4. Quick checks: same number of atoms, same number of elements at each "level."

10. Divisibility Lattices — Detailed Analysis

The lattice (Dn, |) of divisors of n is one of the most frequently tested lattice structures in GATE.

When is Dn a Boolean Algebra?

Theorem: (Dn, |) is a Boolean algebra if and only if n is squarefree (i.e., n is a product of distinct primes).

Reason: If n = p1 · p2 · … · pk (all distinct primes), then Dn is isomorphic to P({p1, …, pk}). Each divisor corresponds to a subset of the prime factors. The number of divisors is 2k.

12. Topological Sort and Linearization

A topological sort (or linear extension) of a poset (S, ≤) is a total order ≤T on S that is compatible with ≤: if a ≤ b in the poset, then a ≤T b in the total order.

Theorem: Every finite poset has at least one linear extension. (This follows from the fact that every finite poset has a minimal element — repeatedly extracting minimal elements gives a topological sort.)

For GATE, topological sort is more commonly encountered in the context of DAGs (directed acyclic graphs) and algorithm design, but the mathematical foundation lies in poset theory.

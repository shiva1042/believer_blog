1. Instruction Format

A machine instruction consists of fields that tell the CPU what operation to perform and on what data. The basic fields are:

FieldDescription
OpcodeSpecifies the operation (ADD, SUB, LOAD, etc.)
Operand(s)Specifies data or address of data
Addressing ModeSpecifies how to interpret the operand field

2. Addressing Modes

Addressing modes determine how the operand field of an instruction is used to calculate the effective address (EA) of the operand in memory.

2.2 Memory Access Count by Mode

Counting rule for instruction execution: Fetch the instruction (1 access) + accesses to get operand. For GATE, operand accesses are typically what is asked:

  • Immediate: 0 extra accesses (data in instruction)
  • Direct: 1 extra access
  • Indirect: 2 extra accesses (one to get address, one to get data)
  • Register: 0 extra accesses (data in register file)
  • Register Indirect: 1 extra access

4. RISC vs CISC

RegistersMany (32+)Few (8-16)

Control unitHardwiredMicroprogrammed
Code sizeLarger (more instructions)Smaller (complex instructions)
ExamplesARM, MIPS, RISC-Vx86, VAX

Key RISC Principle: Simplify the hardware to execute simple instructions very fast. Complex operations are built by the compiler from simple instructions. The load/store architecture means only LOAD and STORE instructions access memory; all other instructions operate on registers.

5.2 Expanding Opcode

When a fixed instruction size must support many instructions with varying operand counts, an expanding opcode scheme is used. Instructions with fewer operands get longer opcodes, and vice versa.

Example: 16-bit instruction, 4-bit register fields. With a 4-bit opcode, you get 15 three-register instructions (one opcode reserved for expansion). The reserved opcode prefix allows encoding more two-register, one-register, and zero-register instructions.

6. GATE Exam Tips

  • Know the effective address formula for each addressing mode cold.
  • Indirect addressing always requires an extra memory access compared to direct.
  • Auto-increment/decrement: increment happens AFTER use, decrement happens BEFORE use.
  • Expanding opcode problems: calculate how many instructions of each type can be encoded given total instruction bits and register field sizes.
  • RISC uses load/store architecture; only LOAD and STORE touch memory.
  • PC-relative addressing is used for position-independent code and branch instructions.

7. Practice Problems

  1. A machine has 16-bit instructions with 4-bit register fields and 3 registers per instruction. How many 3-register instructions can be encoded if the opcode is 4 bits?
  2. Given auto-increment mode with LOAD R1, (R2)+ and R2=1000, word size=4 bytes: what is R2 after execution?
  3. How many memory accesses are needed for: LOAD R1, @(R2) using register indirect with one level of indirection?
  4. In a 0-address machine, write the instruction sequence for: X = (A * B) + (C / D).

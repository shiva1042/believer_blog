Machine Learning Fundamentals

Section: Artificial Intelligence

1. What is Machine Learning?

Machine Learning (ML) is a subfield of AI that focuses on building systems that learn from data and improve their performance without being explicitly programmed. Tom Mitchell's formal definition:

A computer program is said to learn from experience E with respect to some task T
and performance measure P, if its performance on T, as measured by P, improves with experience E.

2. Types of Machine Learning

2.1 Supervised Learning

The algorithm learns from labeled training data — each input has a corresponding desired output. The goal is to learn a mapping from inputs to outputs that generalizes to unseen data.

  • Classification: Output is a discrete category (e.g., spam/not-spam, digit recognition)
  • Regression: Output is a continuous value (e.g., house price prediction, temperature forecasting)

Examples: Linear Regression, Logistic Regression, Decision Trees, SVM, KNN, Naive Bayes, Neural Networks.

2.2 Unsupervised Learning

The algorithm learns from unlabeled data, discovering hidden patterns or structures without guidance.

  • Clustering: Grouping similar data points (e.g., K-Means, DBSCAN)
  • Dimensionality Reduction: Reducing the number of features while preserving information (e.g., PCA)
  • Association Rule Mining: Finding interesting relationships between variables (e.g., Apriori)

2.3 Reinforcement Learning

An agent learns by interacting with an environment, receiving rewards or penalties for its actions, and learning a policy that maximizes cumulative reward.

  • Key concepts: Agent, Environment, State, Action, Reward, Policy
  • Exploration vs. Exploitation: Balancing trying new actions vs. using known good actions
  • Examples: Q-Learning, SARSA, Deep Q-Networks, Policy Gradient methods

3. Bias-Variance Tradeoff

The prediction error of any ML model can be decomposed into three components:

Total Error = Bias² + Variance + Irreducible Noise
ComponentDescriptionEffect
BiasError from overly simplistic assumptions in the modelHigh bias → underfitting (misses relevant patterns)
VarianceError from sensitivity to small fluctuations in training dataHigh variance → overfitting (captures noise as patterns)
Irreducible NoiseInherent noise in the data that no model can captureCannot be reduced by any model
Intuition:
  • High bias, low variance: Linear model on non-linear data — consistently wrong but in a predictable way
  • Low bias, high variance: Deep decision tree — fits training data perfectly but gives very different results on different training sets
  • Sweet spot: A model with the right complexity that balances bias and variance

4. Overfitting and Underfitting

4.1 Overfitting

Model learns noise and fluctuations in the training data. Performs well on training data but poorly on unseen data. Caused by excessive model complexity relative to the amount of training data.

Solutions: Regularization (L1/L2), pruning (decision trees), dropout (neural networks), more training data, simpler model, cross-validation, early stopping.

4.2 Underfitting

Model is too simple to capture the underlying patterns. Performs poorly on both training and test data.

Solutions: More complex model, more features, reduce regularization, longer training.

5. Cross-Validation

Cross-validation is a technique for assessing model generalization performance by partitioning data into training and validation sets multiple times.

5.1 K-Fold Cross-Validation

  1. Split data into K equal folds
  2. For each fold i (i = 1 to K): train on all folds except fold i, validate on fold i
  3. Average the K validation scores
CV Score = (1/K) ∑ Score_i for i = 1 to K

Common choices: K = 5 or K = 10. Leave-One-Out CV (LOOCV) uses K = N (number of samples) — computationally expensive but minimal bias.

5.2 Stratified K-Fold

Ensures each fold has approximately the same proportion of each class label. Important for imbalanced datasets.

6. Evaluation Metrics

6.1 Confusion Matrix

Actual PositiveTrue Positive (TP)False Negative (FN) - Type II Error
Actual NegativeFalse Positive (FP) - Type I ErrorTrue Negative (TN)

6.2 Metrics from Confusion Matrix

Accuracy = (TP + TN) / (TP + TN + FP + FN)

Precision = TP / (TP + FP) [Of predicted positives, how many are correct?]

Recall (Sensitivity/TPR) = TP / (TP + FN) [Of actual positives, how many are found?]

Specificity (TNR) = TN / (TN + FP) [Of actual negatives, how many are correctly identified?]

F1 Score = 2 × (Precision × Recall) / (Precision + Recall) [Harmonic mean]

False Positive Rate (FPR) = FP / (FP + TN) = 1 - Specificity
Example: Medical test with TP=90, FP=10, FN=20, TN=880:
Accuracy = (90+880)/(90+880+10+20) = 970/1000 = 97%
Precision = 90/(90+10) = 90%
Recall = 90/(90+20) = 81.8%
F1 = 2 × 0.9 × 0.818 / (0.9 + 0.818) = 85.7%

6.3 ROC Curve and AUC

The ROC (Receiver Operating Characteristic) curve plots True Positive Rate (Recall) vs. False Positive Rate at various classification thresholds.

  • AUC (Area Under the Curve): Ranges from 0 to 1. AUC = 1 is a perfect classifier. AUC = 0.5 is random guessing.
  • ROC-AUC is threshold-independent and useful for comparing classifiers.
  • Particularly useful when class distribution is balanced.

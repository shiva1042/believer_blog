1. Propositional Logic Resolution

1.1 Converting to CNF (Conjunctive Normal Form)

Steps to convert any formula to CNF:
1. Eliminate biconditional: A <-> B  =>  (A -> B) ^ (B -> A)
2. Eliminate implication:   A -> B   =>  ~A v B
3. Push negations inward (De Morgan's):
   ~(A ^ B) => ~A v ~B
   ~(A v B) => ~A ^ ~B
   ~~A => A
4. Distribute OR over AND:
   A v (B ^ C)  =>  (A v B) ^ (A v C)

Result: conjunction (AND) of clauses, each clause is a disjunction (OR) of literals.

1.2 CNF Conversion Example

Convert: (P -> Q) -> R

Step 1: P -> Q becomes ~P v Q
  Formula: (~P v Q) -> R

Step 2: Eliminate implication:
  ~(~P v Q) v R

Step 3: Push negation inward:
  (P ^ ~Q) v R

Step 4: Distribute OR over AND:
  (P v R) ^ (~Q v R)

CNF clauses: {P v R}, {~Q v R}

1.3 Resolution Procedure

Resolution rule: From clause (A v B) and clause (~B v C),
derive resolvent (A v C).

To prove KB |= alpha:
1. Add ~alpha to KB
2. Convert all formulas to CNF
3. Repeatedly apply resolution
4. If empty clause {} is derived -> proof complete
5. If no new clauses -> alpha not provable

Example: KB = {P -> Q, Q -> R}, prove P -> R
  KB clauses: {~P v Q}, {~Q v R}
  Add ~(P -> R) = P ^ ~R
  Clauses: {P}, {~R}

  All clauses: {~P v Q}, {~Q v R}, {P}, {~R}
  Resolve {P} and {~P v Q}: derive {Q}
  Resolve {Q} and {~Q v R}: derive {R}
  Resolve {R} and {~R}: derive {} (empty clause!)
  Therefore P -> R is proved.

2. First-Order Logic

2.1 Unification

Unification finds a substitution that makes two expressions identical.

Unify(P(x, f(y)), P(a, f(g(z))))
  x = a, y = g(z)
  Substitution: {x/a, y/g(z)}

Unify(P(x, x), P(a, b))
  x = a, then x = b => a must equal b
  If a != b: FAIL (no unifier)

Unify(P(f(x)), P(x))
  f(x) = x => occurs check fails (infinite recursion)
  FAIL

Most General Unifier (MGU): The least restrictive substitution.

2.2 Unification Algorithm

function UNIFY(x, y, theta):
    if theta = failure: return failure
    if x = y: return theta
    if x is a variable: return UNIFY-VAR(x, y, theta)
    if y is a variable: return UNIFY-VAR(y, x, theta)
    if x and y are compound:
        return UNIFY(args(x), args(y), UNIFY(op(x), op(y), theta))
    return failure

function UNIFY-VAR(var, x, theta):
    if {var/val} in theta: return UNIFY(val, x, theta)
    if {x/val} in theta: return UNIFY(var, val, theta)
    if var occurs in x: return failure (occurs check)
    return theta + {var/x}

2.3 Resolution in FOL

Steps:
1. Convert to CNF (includes Skolemization)
2. Standardize variables apart (rename to avoid conflicts)
3. Find complementary literals that unify
4. Apply resolution with unifier

Converting to CNF for FOL:
  a. Eliminate implications and biconditionals
  b. Move negations inward
  c. Standardize variables
  d. Skolemize: Replace existential variables with Skolem functions
     ∃x P(x) => P(A)  (Skolem constant)
     ∀x ∃y P(x,y) => P(x, f(x))  (Skolem function)
  e. Drop universal quantifiers
  f. Distribute OR over AND

Example:
  "Everyone who loves all animals is loved by someone"
  ∀x [∀y Animal(y) -> Loves(x,y)] -> [∃y Loves(y,x)]

  After CNF conversion with Skolemization:
  {~Loves(x, F(x)) v Loves(G(x), x)}
  {Animal(F(x)) v Loves(G(x), x)}

3. Prolog-like Inference

Prolog uses backward chaining with depth-first search.

Knowledge Base:
  parent(tom, bob).
  parent(tom, liz).
  parent(bob, ann).
  parent(bob, pat).

  grandparent(X, Y) :- parent(X, Z), parent(Z, Y).
  sibling(X, Y) :- parent(Z, X), parent(Z, Y), X \= Y.

Query: ?- grandparent(tom, ann).
  Trace:
  1. Match grandparent(tom, ann) with rule head
     X = tom, Y = ann
  2. Need to prove: parent(tom, Z) and parent(Z, ann)
  3. parent(tom, bob) succeeds: Z = bob
  4. parent(bob, ann) succeeds
  5. Answer: true

Query: ?- grandparent(tom, Who).
  Returns: Who = ann ; Who = pat

3.1 Prolog Execution Model

3.2 Cut and Negation

Cut (!): Prevents backtracking past the cut point.
  max(X, Y, X) :- X >= Y, !.
  max(X, Y, Y).

Negation as failure (\+):
  \+ Goal succeeds if Goal fails.

  bachelor(X) :- male(X), \+ married(X).

  This is NOT logical negation! It assumes closed-world:
  anything not provable is false.

4. Fuzzy Logic

Fuzzy logic extends classical logic to handle partial truth values in the range [0, 1].

4.1 Fuzzy Operations

OperationClassical LogicFuzzy Logic
NOT A~A1 - μ(A)
A AND BA ^ Bmin(μ(A), μ(B))
A OR BA v Bmax(μ(A), μ(B))
A IMPLIES BA -> Bmin(1, 1 - μ(A) + μ(B))

4.2 Fuzzy Example

Temperature membership functions:
  Cold: mu_cold(20°C) = 0.8,  mu_cold(30°C) = 0.2
  Hot:  mu_hot(20°C) = 0.1,   mu_hot(30°C) = 0.7

"Temperature is Cold AND NOT Hot" at 25°C:
  mu_cold(25) = 0.5 (interpolated)
  mu_hot(25)  = 0.4 (interpolated)
  NOT Hot = 1 - 0.4 = 0.6
  Cold AND NOT Hot = min(0.5, 0.6) = 0.5

Fuzzy inference: If temperature is hot, then fan speed is high.
  Apply rule with mu_hot = 0.4
  Clip output membership to 0.4
  Defuzzify (e.g., centroid method) to get crisp output.

5. Temporal Logic Basics

Temporal logic adds operators for reasoning about propositions over time.

5.1 Linear Temporal Logic (LTL)

OperatorSymbolMeaning
NextX p / O pp holds in the next state
GloballyG p / [] pp holds in all future states
EventuallyF p / <> pp holds in some future state
Untilp U qp holds until q becomes true

5.2 LTL Examples

"The light is always eventually green":
  G(F(green))

"Once the alarm sounds, the system shuts down within the next state":
  G(alarm -> X(shutdown))

"The request is always eventually followed by a response":
  G(request -> F(response))

"Mutual exclusion: processes p1 and p2 are never both in critical section":
  G(~(cs1 ^ cs2))

"Starvation freedom: if a process requests, it eventually enters CS":
  G(request_i -> F(cs_i))

5.3 CTL (Computation Tree Logic)

CTL adds path quantifiers:
  A = for All computation paths
  E = there Exists a computation path

Combined with temporal operators:
  AG p:  On all paths, p always holds (invariant)
  EF p:  There exists a path where p eventually holds (reachable)
  AF p:  On all paths, p eventually holds (inevitable)
  EG p:  There exists a path where p always holds

Key difference from LTL:
  LTL: implicit universal path quantification
  CTL: explicit path quantifiers required

6. Resolution Example Gallery

Example 1: Prove ~Q from {P, P -> Q -> R, ~R}

Convert to CNF:
  {P}, {~P v ~Q v R}, {~R}

Resolution steps:
  Resolve {P} and {~P v ~Q v R}: {~Q v R}
  Resolve {~Q v R} and {~R}: {~Q}

Result: ~Q is derived. Proved!

Example 2: Prove S from {P v Q, P -> R, Q -> S, R -> S}

CNF: {P v Q}, {~P v R}, {~Q v S}, {~R v S}
Add ~S: {~S}

Resolve {~R v S} and {~S}: {~R}
Resolve {~P v R} and {~R}: {~P}
Resolve {P v Q} and {~P}: {Q}
Resolve {Q} and {~Q v S}: {S}
Resolve {S} and {~S}: {} (empty clause!)

S is proved.
GATE Tip: In resolution problems, always negate what you want to prove, convert everything to CNF, and apply resolution. The order in which you resolve clauses can affect efficiency but not correctness.

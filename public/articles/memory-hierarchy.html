1. Introduction

The memory hierarchy exploits the principle of locality of reference to provide the illusion of a large, fast, and cheap memory. Data that is accessed frequently is kept in smaller, faster (and more expensive) memory levels.

1.1 Locality of Reference

TypeDefinitionExample
Temporal LocalityRecently accessed data will likely be accessed again soonLoop variables, counters
Spatial LocalityNearby memory locations will likely be accessed soonArray traversal, sequential code

2. Cache Memory

Cache is a small, fast SRAM memory that stores copies of frequently accessed main memory data. The CPU checks the cache first before accessing main memory.

2.1 Cache Terminology

TermDefinition
Cache HitRequested data found in cache
Cache MissRequested data not in cache; must fetch from lower level
Hit Rate (h)Fraction of accesses that are hits
Miss Rate (m)1 - h; fraction that are misses
Hit TimeTime to access cache on a hit
Miss PenaltyAdditional time to fetch from lower level on a miss
Block/LineUnit of data transfer between cache and memory
TagBits used to identify which memory block is in a cache line

2.2 Address Breakdown

3. Cache Mapping Techniques

3.1 Direct Mapped Cache

Each memory block maps to exactly one cache line: Cache line = (Block address) mod (Number of lines)

AdvantageDisadvantage
Simple hardwareHigh conflict misses
Fast lookupNo flexibility in placement

Example: 8-line cache, block addresses 0-31. Block 0, 8, 16, 24 all map to line 0. If a program alternates between blocks 0 and 8, every access is a miss (thrashing).

3.2 Fully Associative Cache

A block can be placed in any cache line. All tags must be compared simultaneously.

AdvantageDisadvantage
No conflict missesComplex hardware (parallel comparators)
Maximum flexibilitySlow for large caches

3.3 Set Associative Cache

A compromise: cache is divided into sets, each containing n lines (n-way set associative). A block maps to a specific set but can go in any line within that set.

Set = (Block address) mod (Number of sets)

AssociativitySetsLines per SetDescription
Direct MappedN (= total lines)11-way set associative
2-wayN/22Each set has 2 lines
4-wayN/44Each set has 4 lines
Fully Associative1NN-way set associative

Key Formula: Number of sets = Cache size / (Block size × Associativity)
Tag bits = Address bits - log2(sets) - log2(block size)

4. Cache Performance

4.1 Average Memory Access Time (AMAT)

AMAT = Hit Time + Miss Rate × Miss Penalty

Example: Hit time = 1 cycle, miss rate = 5%, miss penalty = 100 cycles.

AMAT = 1 + 0.05 × 100 = 6 cycles

4.2 Multi-Level Cache AMAT

AMAT = L1 Hit Time + L1 Miss Rate × (L2 Hit Time + L2 Miss Rate × L2 Miss Penalty)

Example: L1: hit time = 1 cycle, miss rate = 5%. L2: hit time = 10 cycles, miss rate = 20% (local). Memory: 100 cycles.

AMAT = 1 + 0.05 × (10 + 0.20 × 100) = 1 + 0.05 × 30 = 2.5 cycles

5. Replacement Policies

When a cache set is full and a new block must be loaded, a replacement policy determines which existing block to evict.

PolicyDescriptionComplexityPerformance
LRU (Least Recently Used)Evict block not used for longest timeHigh (needs tracking)Best (exploits temporal locality)
FIFO (First In First Out)Evict oldest block in setMediumGood
RandomEvict a random blockLowSurprisingly good for high associativity
LFU (Least Frequently Used)Evict least accessed blockHighGood for some workloads

LRU implementation: For 2-way, 1 bit per set suffices. For n-way, tracking requires log2(n!) bits for exact LRU (impractical for large n), so pseudo-LRU approximations are used.

6. Write Policies

6.1 On a Write Hit

PolicyActionProCon
Write-ThroughWrite to both cache and memoryMemory always consistentSlow (every write goes to memory)
Write-BackWrite only to cache; write to memory on evictionFaster (fewer memory writes)Inconsistency risk; needs dirty bit

6.2 On a Write Miss

PolicyActionTypical Pairing
Write-AllocateLoad block into cache, then writeUsed with write-back
No-Write-AllocateWrite directly to memory, don't load to cacheUsed with write-through

6.3 Write Buffer

A small FIFO buffer between cache and memory that holds pending writes. The CPU can continue executing while writes drain to memory in the background. Used primarily with write-through caches.

8. Virtual Memory (Brief)

Virtual memory extends the memory hierarchy to disk, giving each process a private address space. The OS manages page tables that map virtual to physical addresses. The TLB (Translation Lookaside Buffer) is a cache for page table entries.

ConceptCache Analogy

9. GATE Exam Tips

  • Memorize AMAT = Hit Time + Miss Rate × Miss Penalty.
  • Tag bits = total address bits - index bits - offset bits.
  • Total cache bits = lines × (valid bit + dirty bit + tag bits + data bits).
  • Direct mapped: index determines location completely. No replacement needed.
  • Fully associative: no index bits. All bits except offset are tag.
  • Write-back + write-allocate is the most common combination in modern CPUs.
  • For multi-level AMAT, local miss rate is per that level; global miss rate = product of local miss rates.

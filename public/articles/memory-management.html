Memory Management and Virtual Memory

⚙️ Section 8: Operating System

1. Contiguous Memory Allocation

In contiguous allocation, each process occupies a single contiguous block of memory.

1.1 Fixed Partitioning

Memory is divided into fixed-size partitions at boot time. Each partition holds exactly one process.

AspectDetail
Internal FragmentationYes — process smaller than partition wastes space within the partition
External FragmentationNo (partitions are fixed)
Degree of MultiprogrammingLimited by number of partitions

1.2 Variable Partitioning

Partitions are created dynamically to fit each process exactly. No internal fragmentation but external fragmentation accumulates over time.

1.3 Placement Strategies

StrategyDescriptionSpeedFragmentation
First FitAllocate the first hole that is big enoughFastModerate
Best FitAllocate the smallest hole that is big enoughSlow (search all)Leaves tiny unusable holes
Worst FitAllocate the largest holeSlowLeaves large remainder (may be useful)
Next FitLike First Fit but starts from last allocation pointFastSimilar to First Fit

Compaction can reduce external fragmentation by moving processes to create a single large free block, but it is expensive and requires relocation capability.

2. Paging

Paging eliminates external fragmentation by dividing physical memory into fixed-size frames and logical memory into same-size pages. A page table maps pages to frames.

2.1 Address Translation

flowchart LR LA["Logical Address\n(page number | offset)"] --> PT["Page Table"] PT --> PA["Physical Address\n(frame number | offset)"] LA -->|"page number p"| PT PT -->|"frame number f"| PA subgraph Translation direction TB A["p = logical_addr / page_size"] B["d = logical_addr % page_size"] C["physical_addr = f * page_size + d"] end
Example: Logical address space = 16 bytes, page size = 4 bytes, physical memory = 32 bytes.
  • Number of pages = 16 / 4 = 4 (need 2 bits for page number)
  • Number of frames = 32 / 4 = 8 (need 3 bits for frame number)
  • Offset = log2(4) = 2 bits
  • Logical address size = 2 + 2 = 4 bits; Physical address size = 3 + 2 = 5 bits

If page table: {0→5, 1→6, 2→1, 3→2}, then logical address 6 (page 1, offset 2) maps to physical address 6*4+2... no: frame 6, offset 2 = 6×4 + 2 = 26.

2.2 Page Table Structure

FieldPurpose
Frame NumberPhysical frame the page maps to
Valid/Invalid BitWhether the page is in physical memory
Protection BitsRead, write, execute permissions
Reference BitSet when page is accessed (used by replacement algorithms)
Dirty (Modified) BitSet when page is written to (determines if write-back needed)

2.3 Translation Lookaside Buffer (TLB)

The TLB is a fast associative cache for page table entries. It dramatically reduces address translation time.

ScenarioMemory Accesses
TLB Hit1 (direct frame lookup from TLB + actual memory access)
TLB Miss (single-level page table)2 (page table access + actual memory access)
TLB Miss (two-level page table)3 (outer + inner page table + actual access)

Effective Access Time (EAT): EAT = h × (t + m) + (1 − h) × (t + n × m + m), where h = TLB hit ratio, t = TLB access time, m = memory access time, n = page table levels.

2.4 Multi-Level Page Tables

For large address spaces, a single page table is too large. Solution: break the page table into multiple levels (hierarchical paging). A 32-bit address with 4KB pages and 4-byte entries needs a two-level table. A 64-bit system may need 4+ levels.

4. Virtual Memory

Virtual memory allows a process's logical address space to be larger than physical memory. Only needed pages are loaded into memory; the rest reside on disk.

4.1 Demand Paging

Pages are loaded into memory only when referenced (lazy loading). If the referenced page is not in memory, a page fault occurs.

4.2 Page Fault Handling Flow

flowchart TD A["CPU generates\nlogical address"] --> B{"Page in\nmemory?"} B -->|"Yes (valid bit=1)"| C["Access frame\nin memory"] B -->|"No (valid bit=0)"| D["Page Fault\n(trap to OS)"] D --> E{"Free frame\navailable?"} E -->|"Yes"| F["Load page from\ndisk to frame"] E -->|"No"| G["Run page\nreplacement algorithm"] G --> H["Write victim page\nto disk (if dirty)"] H --> F F --> I["Update page table\n(set valid bit)"] I --> J["Restart\ninstruction"]

Effective Access Time with Page Faults: EAT = (1 − p) × ma + p × page_fault_time, where p = page fault rate, ma = memory access time. Page fault time includes disk I/O (typically 8-10 ms).

Belady's Anomaly: With FIFO, increasing the number of frames can sometimes increase page faults. This does NOT occur with LRU or Optimal (they are stack algorithms).

5.1 Clock Algorithm Detail

Pages are arranged in a circular buffer with a clock hand pointer. On a page fault:

  1. Check the page at the clock hand.
  2. If reference bit = 0, replace this page.
  3. If reference bit = 1, set it to 0, advance the hand, and repeat.

6. Thrashing

Thrashing occurs when a process spends more time paging (swapping pages in and out) than executing. This happens when the process does not have enough frames to hold its working set of pages.

ConceptDescription
Working SetThe set of pages a process is actively using within a time window Δ
Working Set ModelAllocate enough frames to cover the working set; if total demand exceeds available frames, suspend a process
Page Fault Frequency (PFF)If page fault rate is too high, allocate more frames; if too low, reduce frames. Establishes upper and lower bounds.
Thrashing indicator: CPU utilization drops sharply while disk activity spikes. The degree of multiprogramming is too high relative to available memory.

7. Summary and GATE Key Points

Key Points for GATE:
  • Number of pages = logical address space / page size. Number of frames = physical memory / frame size.
  • Page table size = number of pages × page table entry size.
  • In a two-level page table, the page number is split into two parts for the outer and inner tables.
  • TLB hit ratio is critical: EAT = h(t+m) + (1-h)(t+2m) for single-level page table.
  • Optimal page replacement gives the minimum page faults (theoretical lower bound).
  • Belady's anomaly applies to FIFO but NOT to LRU or Optimal (stack algorithms).
  • Internal fragmentation occurs in paging (last page); external fragmentation occurs in segmentation and variable partitioning.
  • Thrashing is solved by the working set model or page fault frequency approach.

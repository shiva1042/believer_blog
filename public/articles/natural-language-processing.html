2. Text Preprocessing

2.1 Tokenization

Tokenization splits text into individual units (tokens). Tokens can be words, subwords, or characters.

Input:  "The cat sat on the mat."
Output: ["The", "cat", "sat", "on", "the", "mat", "."]

Sentence tokenization:
Input:  "Hello world. How are you?"
Output: ["Hello world.", "How are you?"]

2.2 Stop Word Removal

Common words (the, is, at, which, on) that carry little meaning are removed to reduce noise.

Before: ["The", "cat", "sat", "on", "the", "mat"]
After:  ["cat", "sat", "mat"]

2.3 Stemming

Reduces words to their root form using rule-based heuristics (may produce non-words).

Porter Stemmer examples:
  "running"    -> "run"
  "flies"      -> "fli"      (not a real word!)
  "organization" -> "organ"  (incorrect root)
  "studies"    -> "studi"

2.4 Lemmatization

Reduces words to their dictionary form (lemma) using vocabulary and morphological analysis.

Lemmatization examples:
  "running"    -> "run"
  "flies"      -> "fly"       (correct!)
  "better"     -> "good"
  "studies"    -> "study"

3. Bag of Words (BoW)

BoW represents text as a vector of word counts, ignoring order and grammar.

Documents:
  D1: "the cat sat"
  D2: "the cat played"
  D3: "the dog sat"

Vocabulary: [cat, dog, played, sat, the]

BoW vectors:
  D1: [1, 0, 0, 1, 1]
  D2: [1, 0, 1, 0, 1]
  D3: [0, 1, 0, 1, 1]
Limitations: BoW loses word order, creates sparse high-dimensional vectors, and treats all words equally regardless of importance.

4. TF-IDF (Term Frequency - Inverse Document Frequency)

TF-IDF weighs words by their importance: frequent in a document but rare across the corpus.

TF(t, d) = (count of t in d) / (total terms in d)

IDF(t) = log(N / df(t))
  where N = total documents, df(t) = documents containing t

TF-IDF(t, d) = TF(t, d) * IDF(t)

4.1 TF-IDF Example

Corpus: 3 documents, N=3
Word "cat": appears in D1, D2 (df=2)
  IDF("cat") = log(3/2) = 0.176

In D1: "the cat sat on the cat" (6 words)
  TF("cat", D1) = 2/6 = 0.333
  TF-IDF("cat", D1) = 0.333 * 0.176 = 0.059

Word "dog": appears only in D3 (df=1)
  IDF("dog") = log(3/1) = 0.477  (higher = more distinctive!)

5.2 GloVe (Global Vectors)

GloVe combines count-based (co-occurrence matrix) and prediction-based methods. It factorizes the word co-occurrence matrix.

Objective: minimize
  J = SUM_{i,j} f(X_ij) * (w_i^T * w_j + b_i + b_j - log(X_ij))^2

where X_ij = co-occurrence count of words i and j
f(x) = weighting function that caps very frequent pairs

6. Language Models

6.1 N-gram Language Model

An n-gram model estimates the probability of a word given the previous (n-1) words.

Unigram:  P(w_i)
Bigram:   P(w_i | w_{i-1})
Trigram:  P(w_i | w_{i-2}, w_{i-1})

Bigram probability estimation:
  P(w_i | w_{i-1}) = count(w_{i-1}, w_i) / count(w_{i-1})

Example: "I want to eat Chinese food"
P(food|Chinese) = count("Chinese food") / count("Chinese")

6.2 Perplexity

Perplexity(W) = P(w1, w2, ..., wN)^(-1/N)
             = 2^H(W)

Lower perplexity = better model.
A model with perplexity 50 is as uncertain as choosing
  uniformly among 50 words at each step.

6.4 Transformer Basics

The Transformer architecture uses self-attention instead of recurrence, enabling parallel processing.

Self-Attention:
  Attention(Q, K, V) = softmax(Q * K^T / sqrt(d_k)) * V

  Q = query matrix
  K = key matrix
  V = value matrix
  d_k = dimension of keys (scaling factor)

7. Named Entity Recognition (NER)

NER identifies and classifies named entities in text into predefined categories.

Input: "Barack Obama was born in Hawaii on August 4, 1961."
Output:
  Barack Obama  -> PERSON
  Hawaii        -> LOCATION
  August 4, 1961 -> DATE
Entity TypeExamples
PERSONAlbert Einstein, Marie Curie
ORGANIZATIONGoogle, MIT, WHO
LOCATIONParis, Mount Everest
DATE/TIMEJanuary 2024, 3:00 PM
MONEY$500, 50 euros

8. Sentiment Analysis

Sentiment analysis determines the emotional tone of text (positive, negative, neutral).

Approaches:
1. Lexicon-based: Use sentiment dictionaries (AFINN, SentiWordNet)
2. ML-based: Train classifiers on labeled data (Naive Bayes, SVM, LSTM)
3. Transformer-based: Fine-tune BERT/GPT on sentiment datasets

Example:
  "This movie was absolutely wonderful!" -> Positive (0.95)
  "Terrible service, never going back."  -> Negative (0.92)
  "The package arrived on Tuesday."      -> Neutral  (0.87)

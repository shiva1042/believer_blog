1. IPv4 Addressing

An IPv4 address is a 32-bit identifier written in dotted-decimal notation (e.g., 192.168.1.1). It is divided into a network portion and a host portion.

1.2 Subnetting

Subnetting divides a network into smaller sub-networks by borrowing bits from the host portion for the subnet portion.

Example: Given 192.168.1.0/24, create 4 subnets.
  • Borrow 2 bits from host: /26 subnet mask (255.255.255.192)
  • Subnets: 192.168.1.0/26, 192.168.1.64/26, 192.168.1.128/26, 192.168.1.192/26
  • Each subnet: 26 − 2 = 62 usable hosts

1.3 CIDR (Classless Inter-Domain Routing)

CIDR eliminates class boundaries. The network prefix length is specified explicitly (e.g., 10.0.0.0/12). This allows more efficient address allocation and route aggregation (supernetting).

3. Fragmentation and Reassembly

When a packet exceeds the MTU (Maximum Transmission Unit) of a link, the router fragments it into smaller packets.

Example: Original datagram: 4000 bytes (including 20-byte header), MTU = 1500 bytes.
FragmentSize (bytes)OffsetMF Flag
11500 (20 hdr + 1480 data)01
21500 (20 hdr + 1480 data)185 (1480/8)1
31040 (20 hdr + 1020 data)370 (2960/8)0

Fragment offset is in units of 8 bytes. Data in each fragment must be a multiple of 8 (except last fragment).

4.2 Dijkstra's Algorithm (Link State)

Each router has complete knowledge of the network topology (obtained via link-state broadcasts). Dijkstra's algorithm computes the shortest path from the source to all other nodes.

  • Complexity: O(n2) or O(n log n) with priority queue, where n = number of nodes
  • Protocol: OSPF (Open Shortest Path First)
  • Advantage: Converges quickly, loop-free
  • Disadvantage: Requires global topology knowledge; high overhead for broadcasting link states

4.3 Bellman-Ford Algorithm (Distance Vector)

Each router knows only the distances to its neighbors. Routers exchange distance vectors periodically with neighbors and update their routing tables using the Bellman-Ford equation:

D(x, y) = min over all neighbors v { cost(x, v) + D(v, y) }
  • Complexity: O(n × E) where E = number of edges
  • Protocol: RIP (Routing Information Protocol)
  • Problem: Count-to-infinity problem (slow convergence on bad news)
  • Solutions: Split horizon, poison reverse, triggered updates

4.5 Flooding

Every incoming packet is sent out on every outgoing link except the one it arrived on. Simple but generates many duplicate packets. Controlled by TTL or sequence numbers.

6. Network Address Translation (NAT)

NAT allows multiple devices on a private network to share a single public IP address. The NAT router translates private IP addresses to the public IP (and vice versa) using a translation table.

NAT TypeDescription
Static NATOne-to-one mapping between private and public IP addresses
Dynamic NATMaps private IPs to a pool of public IPs on demand
PAT (Port Address Translation / NAPT)Multiple private IPs share one public IP using different port numbers. Most common type.
Private IP Ranges (RFC 1918):
  • 10.0.0.0/8 (Class A): 10.0.0.0 – 10.255.255.255
  • 172.16.0.0/12 (Class B): 172.16.0.0 – 172.31.255.255
  • 192.168.0.0/16 (Class C): 192.168.0.0 – 192.168.255.255

7. Summary and GATE Key Points

Key Points for GATE:
  • Number of usable hosts in a subnet = 2(32−prefix) − 2 (network address and broadcast).
  • Fragment offset is in units of 8 bytes. Data portion of each fragment (except last) must be a multiple of 8.
  • IP header minimum size = 20 bytes; maximum = 60 bytes (with options).
  • TTL is decremented by 1 at each router. When it reaches 0, the packet is dropped and ICMP Time Exceeded is sent.
  • ARP maps IP → MAC within a LAN. ARP request is broadcast; ARP reply is unicast.
  • DHCP uses UDP, not TCP. The DORA process enables automatic IP configuration.
  • Distance vector: count-to-infinity problem. Link state: no such problem but higher overhead.
  • Supernetting (route aggregation) combines multiple networks into one routing entry using a shorter prefix.
  • To find the number of subnets: 2borrowed bits. Hosts per subnet: 2remaining host bits − 2.

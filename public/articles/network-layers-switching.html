2. TCP/IP Model

The TCP/IP model is the practical model used in the Internet. It has four layers (sometimes five when the link layer is split into data link and physical).

2.2 Encapsulation

As data travels down the layers, each layer adds its own header (and sometimes trailer). At the receiving end, each layer strips its header. This is called encapsulation and decapsulation.

LayerEncapsulated UnitHeader Added
ApplicationApplication dataApplication-specific headers
TransportSegment (TCP) / Datagram (UDP)Source port, destination port, sequence numbers
NetworkPacketSource IP, destination IP, TTL
Data LinkFrameSource MAC, destination MAC + CRC trailer
PhysicalBitsPreamble, signal encoding

3. Switching Techniques

Switching determines how data is transferred from source to destination through the network.

3.1 Circuit Switching

A dedicated communication path is established between sender and receiver for the entire duration of the communication. Resources are reserved end-to-end.

  • Advantages: Guaranteed bandwidth, constant delay, no congestion after setup
  • Disadvantages: Wasted bandwidth during silence, setup delay, inflexible
  • Example: Traditional telephone network (PSTN)

3.2 Packet Switching

Data is divided into packets that are independently routed through the network. No dedicated path is established.

TypeDescription
DatagramEach packet is routed independently. Packets may arrive out of order. No connection setup. Example: IP.
Virtual CircuitA logical path is established before data transfer. All packets follow the same path. Packets arrive in order. Example: ATM, MPLS, X.25.

3.3 Message Switching

The entire message is sent to an intermediate node, stored, and then forwarded to the next node (store-and-forward). No real-time communication. Used in early telegraph systems and email.

3.5 Delay Calculations

Total delay for packet switching (store-and-forward):
For a message of size M bits, link bandwidth B bps, packet size P bits, and H hops:
Without pipelining: T = H × (M/B)
With pipelining (packet switching): T = (M/B) + (H − 1) × (P/B)
Packet switching reduces delay by pipelining packet transmissions across links.
Example: M = 10 Mb, P = 1 Kb, B = 1 Mbps, H = 3 hops.
Message switching: T = 3 × (10 Mb / 1 Mbps) = 30 seconds.
Packet switching: T = (10 Mb / 1 Mbps) + (3 − 1) × (1 Kb / 1 Mbps) = 10 + 0.002 = 10.002 seconds.

5. Summary and GATE Key Points

Key Points for GATE:
  • OSI has 7 layers; TCP/IP has 4 layers. The top 3 OSI layers map to TCP/IP Application layer.
  • PDUs: Data (Application) → Segment (Transport) → Packet (Network) → Frame (Data Link) → Bits (Physical).
  • Circuit switching: dedicated path, guaranteed bandwidth, setup delay. Best for constant-rate traffic.
  • Packet switching (datagram): no setup, packets routed independently, efficient for bursty traffic.
  • Virtual circuit: setup required, all packets follow same path, in-order delivery.
  • Switches operate at Layer 2 (MAC); Routers at Layer 3 (IP).
  • Packet switching delay with pipelining: (M/B) + (H-1)(P/B), much less than message switching: H(M/B).
  • Hubs create a single collision domain; switches create separate collision domains per port; routers create separate broadcast domains.

Neural Networks and Deep Learning

Section: Artificial Intelligence

1. The Perceptron

The perceptron is the simplest neural network — a single neuron that computes a weighted sum of inputs and applies a step function.

Output y = step(w1x1 + w2x2 + ... + wnxn + b)
y = step(WTX + b)

step(z) = 1 if z ≥ 0, else 0

The perceptron learning rule updates weights when a misclassification occurs:

wi := wi + α(y - ŷ)xi
where α is the learning rate, y is the true label, ŷ is the predicted label

Perceptron Convergence Theorem: If the training data is linearly separable, the perceptron learning algorithm will converge to a solution in a finite number of steps.

Limitation: The perceptron can only learn linearly separable functions.
It can learn AND, OR, and NOT but cannot learn XOR.
XOR requires at least one hidden layer (Multi-Layer Perceptron).

2. Multi-Layer Perceptron (MLP)

An MLP consists of an input layer, one or more hidden layers, and an output layer. Each layer is fully connected to the next. MLPs can learn non-linear decision boundaries.

  • Input layer: Receives the feature vector (no computation)
  • Hidden layers: Apply weighted sum + activation function. Each hidden layer learns increasingly abstract features.
  • Output layer: Produces the final prediction
For each neuron j in layer l:
zj(l) = ∑ wij(l) ai(l-1) + bj(l) (weighted sum)
aj(l) = g(zj(l)) (activation)
where g is the activation function

Universal Approximation Theorem: A feedforward network with a single hidden layer containing a finite number of neurons can approximate any continuous function on a compact subset of Rn, given a suitable activation function.

3. Activation Functions

FunctionFormulaRangeProperties
Sigmoid σ(z) = 1/(1 + e-z) (0, 1) Smooth, differentiable. Suffers from vanishing gradient for large |z|. Output not zero-centered.
Tanh tanh(z) = (ez - e-z)/(ez + e-z) (-1, 1) Zero-centered. Still suffers from vanishing gradient. tanh(z) = 2σ(2z) - 1
ReLU f(z) = max(0, z) [0, ∞) Efficient computation. No vanishing gradient for z > 0. Can cause "dying ReLU" (neurons stuck at 0).
Leaky ReLU f(z) = max(αz, z), α small (-∞, ∞) Fixes dying ReLU problem. Typically α = 0.01.
Softmax softmax(zi) = ezi / ∑ezj (0, 1), sum = 1 Used for multi-class classification in the output layer. Outputs a probability distribution.
Derivatives:
Sigmoid: σ'(z) = σ(z)(1 - σ(z)), max value = 0.25 at z=0
Tanh: tanh'(z) = 1 - tanh²(z)
ReLU: f'(z) = 1 if z > 0, 0 if z < 0 (undefined at z=0, typically set to 0 or 1)

4. Backpropagation

Backpropagation is the algorithm for computing gradients of the loss function with respect to all weights in the network. It uses the chain rule of calculus to propagate error signals backward from the output layer to earlier layers.

4.1 Steps

  1. Forward pass: Compute activations layer by layer from input to output
  2. Compute loss: Calculate the error between predicted and actual output
  3. Backward pass: Compute gradients starting from the output layer, propagating backward through the network using the chain rule
  4. Update weights: w := w - α · ∂L/∂w
Chain rule application:
∂L/∂wij(l) = ∂L/∂aj(l) · ∂aj(l)/∂zj(l) · ∂zj(l)/∂wij(l)

= δj(l) · ai(l-1)
where δj(l) = ∂L/∂zj(l) is the error signal at neuron j in layer l

5.1 Advanced Optimizers

OptimizerKey Idea
SGD + MomentumAccumulates exponentially decaying moving average of past gradients. Helps accelerate through narrow ravines.
RMSPropAdapts learning rate per parameter using exponential average of squared gradients. Good for non-stationary objectives.
AdamCombines momentum (first moment) and RMSProp (second moment) with bias correction. Most popular default optimizer.
Adam update rules:
mt = β1 mt-1 + (1 - β1) gt (first moment estimate)
vt = β2 vt-1 + (1 - β2) gt² (second moment estimate)
m̂t = mt / (1 - β1t) (bias correction)
v̂t = vt / (1 - β2t) (bias correction)
wt = wt-1 - α m̂t / (√v̂t + ε)
Defaults: β1=0.9, β2=0.999, ε=10-8

6. Convolutional Neural Networks (CNN) Basics

CNNs are designed for processing grid-structured data (images, sequences). They use three key operations:

6.1 Convolution Layer

Applies learnable filters (kernels) that slide over the input. Each filter detects a specific feature (edges, textures, patterns).

Output size = (W - F + 2P) / S + 1
W = input size, F = filter size, P = padding, S = stride

Parameter sharing: The same filter weights are used across the entire input, drastically reducing the number of parameters compared to a fully connected layer.

6.2 Pooling Layer

Reduces spatial dimensions. Max pooling takes the maximum value in each pooling window. Average pooling takes the average. Provides translational invariance and reduces computation.

6.3 Typical CNN Architecture

Input → [Conv → ReLU → Pool] × N → Flatten → Fully Connected → Softmax

Example: Input image 32×32, filter 5×5, stride 1, no padding:
Output size = (32 - 5 + 0) / 1 + 1 = 28×28
With 10 filters: output volume is 28×28×10
Parameters per filter: 5×5×1 + 1 (bias) = 26
Total parameters: 26 × 10 = 260

7. Recurrent Neural Networks (RNN) Basics

RNNs process sequential data by maintaining a hidden state that captures information from previous time steps.

ht = g(Whh ht-1 + Wxh xt + bh) (hidden state update)
yt = Why ht + by (output)

RNNs share weights across time steps. They are trained using Backpropagation Through Time (BPTT), which unrolls the network across time steps and applies standard backpropagation.

Applications: language modeling, machine translation, speech recognition, time series prediction.

8. Vanishing and Exploding Gradient Problem

During backpropagation in deep networks, gradients are multiplied through many layers. This leads to:

ProblemCauseEffectSolutions
Vanishing Gradient Gradients < 1 are multiplied repeatedly, shrinking exponentially Early layers learn extremely slowly; weights barely update ReLU activation, LSTM/GRU (for RNNs), residual connections, batch normalization, careful initialization
Exploding Gradient Gradients > 1 are multiplied repeatedly, growing exponentially Weights oscillate or diverge; training becomes unstable Gradient clipping, weight regularization, LSTM/GRU
For sigmoid: max derivative = 0.25
After L layers: gradient ≤ 0.25L
For L=10: 0.2510 ≈ 9.5 × 10-7 (effectively zero!)

ReLU gradient = 1 for z > 0, avoiding vanishing gradient (but "dying ReLU" can occur).

8.1 LSTM (Long Short-Term Memory)

LSTM cells address the vanishing gradient problem in RNNs by introducing gates that control information flow:

  • Forget gate: Decides what information to discard from the cell state
  • Input gate: Decides what new information to store in the cell state
  • Output gate: Decides what to output based on the cell state

The cell state acts as a "conveyor belt" that allows gradients to flow unchanged through many time steps.

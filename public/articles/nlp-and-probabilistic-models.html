Section: Artificial Intelligence

1. Bayes Classifier

The Bayes classifier is a probabilistic framework for classification based on Bayes' Theorem. It selects the class that maximizes the posterior probability given the observed features.

Bayes' Theorem: P(C|X) = P(X|C) · P(C) / P(X)

Bayes Optimal Classifier: C* = argmaxC P(C|X) = argmaxC P(X|C) · P(C)
(P(X) can be ignored since it is constant across classes)
  • Prior P(C): Probability of each class before observing data (estimated from training set frequencies)
  • Likelihood P(X|C): Probability of observing features X given class C
  • Posterior P(C|X): Probability of class C given the observed features
  • Evidence P(X): Normalization constant, independent of the class

The Bayes optimal classifier has the minimum possible error rate (Bayes error rate). In practice, the Naive Bayes classifier approximates this by assuming feature independence.

Example: Email classification.
Features: word occurrences. Classes: Spam (S), Ham (H).
P(S) = 0.3, P(H) = 0.7
For email with word "offer": P("offer"|S) = 0.6, P("offer"|H) = 0.1
P(S|"offer") ∝ 0.6 × 0.3 = 0.18
P(H|"offer") ∝ 0.1 × 0.7 = 0.07
Normalized: P(S|"offer") = 0.18/0.25 = 0.72 → Classify as Spam.

2. Hidden Markov Models (HMM)

An HMM is a statistical model for sequences where the system is assumed to be a Markov process with hidden (unobservable) states that produce observable outputs.

2.1 Components of an HMM

ComponentSymbolDescription
StatesS = {s1, ..., sN}Hidden states (not directly observable)
ObservationsO = {o1, ..., oM}Observable outputs
Transition probabilitiesA = {aij}aij = P(qt+1=sj | qt=si)
Emission probabilitiesB = {bj(k)}bj(k) = P(ot=vk | qt=sj)
Initial distributionπ = {πi}πi = P(q1=si)

An HMM is fully specified by λ = (A, B, π).

2.2 Three Fundamental HMM Problems

ProblemQuestionAlgorithm
EvaluationWhat is P(O|λ)? (probability of observation sequence)Forward algorithm (or Backward algorithm)
DecodingWhat is the most likely hidden state sequence?Viterbi algorithm
LearningWhat model parameters λ best explain the data?Baum-Welch algorithm (EM)

2.3 Markov Property

First-order Markov assumption:
P(qt+1 | qt, qt-1, ..., q1) = P(qt+1 | qt)
The future state depends only on the current state, not the history.
Example — Weather HMM:
Hidden states: {Sunny, Rainy}
Observations: {Walk, Shop, Clean}
Transition: P(Sunny|Sunny) = 0.7, P(Rainy|Sunny) = 0.3, P(Sunny|Rainy) = 0.4, P(Rainy|Rainy) = 0.6
Emission: P(Walk|Sunny) = 0.6, P(Shop|Sunny) = 0.3, P(Clean|Sunny) = 0.1
Given observation sequence (Walk, Shop, Clean), we can use the Viterbi algorithm to find the most likely weather sequence.

3. Viterbi Algorithm

The Viterbi algorithm finds the most likely sequence of hidden states given an observation sequence. It uses dynamic programming to avoid exponential computation.

3.1 Algorithm Steps

Initialization (t=1):
δ1(i) = πi · bi(o1) for each state i
ψ1(i) = 0

Recursion (t=2,...,T):
δt(j) = maxi [δt-1(i) · aij] · bj(ot)
ψt(j) = argmaxi [δt-1(i) · aij]

Termination:
P* = maxi δT(i)
q*T = argmaxi δT(i)

Backtracking (t=T-1,...,1):
q*t = ψt+1(q*t+1)

Time Complexity: O(N² · T) where N = number of states, T = sequence length.
Space Complexity: O(N · T) for storing δ and ψ tables.

4. Text Preprocessing

Text preprocessing transforms raw text into a form suitable for NLP algorithms. Key steps include:

4.1 Tokenization

Splitting text into individual units (tokens). Tokens can be words, subwords, or characters.

Example:
Input: "Natural language processing is fascinating!"
Word tokens: ["Natural", "language", "processing", "is", "fascinating", "!"]
After lowercasing: ["natural", "language", "processing", "is", "fascinating", "!"]

4.2 Stop Word Removal

Removing common words that carry little semantic meaning: "the", "is", "at", "a", "an", "in", etc.

4.3 Stemming

Reducing words to their root form by removing suffixes. Aggressive and may produce non-words.

Porter Stemmer examples:
"running" → "run", "flies" → "fli", "connection" → "connect"
"studies" → "studi", "caresses" → "caress"

4.4 Lemmatization

Reducing words to their dictionary form (lemma) using vocabulary and morphological analysis. More accurate than stemming but slower.

Lemmatization examples:
"running" → "run", "flies" → "fly", "better" → "good"
"studies" → "study", "geese" → "goose"
FeatureStemmingLemmatization
ApproachRule-based suffix strippingDictionary + morphological analysis
OutputMay not be a real wordAlways a valid word (lemma)
SpeedFastSlower
AccuracyLowerHigher

5. TF-IDF (Term Frequency – Inverse Document Frequency)

TF-IDF is a numerical statistic that reflects the importance of a word to a document within a corpus. Words that are frequent in a document but rare across the corpus get higher scores.

TF(t, d) = count of term t in document d / total terms in d

IDF(t, D) = log(N / df(t))
where N = total documents, df(t) = number of documents containing t

TF-IDF(t, d, D) = TF(t, d) × IDF(t, D)
Example: Corpus with 10,000 documents. Document d has 100 words.
Word "algorithm" appears 5 times in d and in 50 documents total.
TF("algorithm", d) = 5/100 = 0.05
IDF("algorithm") = log(10000/50) = log(200) ≈ 2.301
TF-IDF = 0.05 × 2.301 = 0.115

Word "the" appears 8 times in d and in 9,800 documents.
TF("the", d) = 8/100 = 0.08
IDF("the") = log(10000/9800) ≈ 0.0088
TF-IDF = 0.08 × 0.0088 = 0.0007 (very low, as expected for a stop word)

Key insight: TF-IDF assigns high weight to terms that are frequent in a specific document but rare across the corpus, effectively filtering out common words.

6. Word Embeddings (Word2Vec Basics)

Word embeddings represent words as dense, low-dimensional vectors in a continuous vector space, capturing semantic relationships.

6.1 Word2Vec Architectures

ArchitectureTaskProperties
CBOW (Continuous Bag of Words)Predict the center word from surrounding context wordsFaster training, better for frequent words
Skip-gramPredict surrounding context words from the center wordWorks better for rare words, better with small datasets
CBOW: maximize P(wt | wt-c, ..., wt-1, wt+1, ..., wt+c)
Skip-gram: maximize P(wt-c, ..., wt-1, wt+1, ..., wt+c | wt)
where c is the context window size

Key properties of word embeddings:

  • Semantically similar words have similar vectors (cosine similarity)
  • Analogical relationships are captured: vec("king") - vec("man") + vec("woman") ≈ vec("queen")
  • Typical dimensionality: 100-300 dimensions
  • Trained on large corpora in an unsupervised manner

7. Language Models (N-gram)

A language model assigns probabilities to sequences of words. N-gram models estimate the probability of a word based on the preceding (N-1) words.

Chain rule: P(w1, w2, ..., wn) = ∏ P(wi | w1, ..., wi-1)

N-gram approximation (Markov assumption of order N-1):
P(wi | w1, ..., wi-1) ≈ P(wi | wi-N+1, ..., wi-1)
ModelContextFormulaExample
UnigramNoneP(wi)P("cat") = count("cat")/total words
Bigram1 wordP(wi|wi-1)P("cat"|"the") = count("the cat")/count("the")
Trigram2 wordsP(wi|wi-2,wi-1)P("cat"|"the","fat") = count("the fat cat")/count("the fat")

7.1 Smoothing Techniques

N-gram models assign zero probability to unseen sequences. Smoothing redistributes some probability mass to unseen events.

Laplace (Add-1) smoothing:
P(wi|wi-1) = (count(wi-1, wi) + 1) / (count(wi-1) + V)
where V = vocabulary size

Add-k smoothing (generalization): replace 1 with k (typically 0 < k < 1)

7.2 Perplexity

Perplexity is the standard evaluation metric for language models. Lower perplexity means the model better predicts the test data.

PP(W) = P(w1, w2, ..., wN)-1/N
= exp(-(1/N) ∑ log P(wi | w1, ..., wi-1))

Interpretation: average branching factor — the average number of equally likely words the model considers at each position.

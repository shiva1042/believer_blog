NP-Completeness and Computational Complexity

Based on CLRS — Introduction to Algorithms, Chapter 34

Computational complexity theory is one of the most profound areas of computer science. It asks a fundamental question: How hard is a problem? Not how hard a particular algorithm is, but how hard the problem itself is — what is the inherent computational effort required to solve it? This article covers the theory of NP-completeness, a cornerstone topic for GATE CSE, drawing primarily from CLRS Chapter 34.

1. Decision Problems vs Optimization Problems

Before diving into complexity classes, we must distinguish between two kinds of problems:

Aspect Decision Problem Optimization Problem
Output YES or NO An optimal value (min or max)
Example (TSP) "Is there a tour of cost ≤ k?" "Find the minimum-cost tour."
Example (Clique) "Does G have a clique of size ≥ k?" "Find the largest clique in G."
Complexity theory focus Primary focus Converted to decision version for analysis
Why decision problems? Complexity classes like P and NP are formally defined for decision problems because they have a clean YES/NO output. An optimization problem is at least as hard as its decision version. If the decision version is hard, so is the optimization version.

1.1 Encoding and Languages

Formally, a decision problem is a language L ⊆ {0,1}*. An instance x belongs to L if and only if the answer to the decision problem on input x is YES. An algorithm A accepts x if A(x) = 1, and rejects x if A(x) = 0. A decides L if it correctly accepts every x ∈ L and rejects every x ∉ L.

2. Complexity Classes

2.1 Class P

P is the class of decision problems (languages) that can be decided by a deterministic Turing machine in polynomial time. 

P = { L ⊆ {0,1}* : there exists an algorithm A and constant c
      such that for every input x, A decides whether x ∈ L
      in time O(|x|^c) }

Examples of problems in P:

  • Sorting (decision: "Is A[i] ≤ A[i+1] for all i?")
  • Shortest path (decision: "Is there a path of weight ≤ k?")
  • Maximum bipartite matching
  • Linear programming (Khachiyan's ellipsoid method)
  • Primality testing (AKS algorithm)
  • 2-SAT
  • 2-Coloring of graphs
  • Eulerian path/circuit

2.2 Class NP

NP (Nondeterministic Polynomial time) is the class of decision problems for which a YES answer can be verified in polynomial time given a suitable certificate (also called a witness). 

NP = { L ⊆ {0,1}* : there exists a polynomial-time verifier V
       and a polynomial p such that
       x ∈ L  ⇔  ∃ certificate y with |y| ≤ p(|x|) and V(x,y) = 1 }
Common misconception: NP does NOT stand for "Non-Polynomial." It stands for "Nondeterministic Polynomial." Every problem in P is also in NP (P ⊆ NP), because if you can solve it in polynomial time, you can certainly verify a solution in polynomial time (just solve it and compare).

Verification examples:

Problem Certificate Verification
HAMILTONIAN CYCLE A sequence of vertices Check it visits every vertex exactly once and forms a cycle
SAT A truth assignment Evaluate the formula under the assignment
CLIQUE A set of k vertices Check all pairs are connected by edges
SUBSET SUM A subset of numbers Sum them and check equality with target

2.3 Class co-NP

co-NP is the class of decision problems whose complement is in NP. That is, L ∈ co-NP if and only if the complement language L̅ ∈ NP.

For a co-NP problem, a NO answer can be verified in polynomial time.

Examples:

  • TAUTOLOGY: "Is the Boolean formula true for ALL assignments?" (The NO answer has a certificate: a falsifying assignment.)
  • UNSAT: The complement of SAT.
Key relationships: P ⊆ NP ∩ co-NP. It is conjectured (but not proven) that NP ≠ co-NP. If any NP-complete problem is in co-NP, then NP = co-NP.

2.4 Class NP-hard

A problem H is NP-hard if every problem in NP can be polynomial-time reduced to H. Formally: 

H is NP-hard  ⇔  for all L ∈ NP, L ≤_p H

NP-hard problems are at least as hard as the hardest problems in NP. An NP-hard problem need not be a decision problem and need not be in NP itself.

Examples of NP-hard (but not NP-complete) problems:

  • Halting Problem (undecidable, so not in NP)
  • Optimization version of TSP (not a decision problem)
  • Finding the chromatic number of a graph

2.5 Class NP-complete (NPC)

A problem is NP-complete if it is both in NP and NP-hard: 

NPC = NP ∩ NP-hard

NP-complete problems are the "hardest" problems in NP. If any single NP-complete problem can be solved in polynomial time, then P = NP and every problem in NP can be solved in polynomial time.

3. Polynomial-Time Reductions

A polynomial-time reduction from problem A to problem B (written A ≤p B) is a polynomial-time computable function f such that for every instance x: 

x ∈ A  ⇔  f(x) ∈ B

Intuitively, A ≤p B means "A is no harder than B" (or "B is at least as hard as A").

3.1 Properties of Reductions

  • Transitivity: If A ≤p B and B ≤p C, then A ≤p C.
  • If B ∈ P and A ≤p B, then A ∈ P.
  • If A is NP-hard and A ≤p B, then B is NP-hard.
GATE Tip: The direction of reduction is crucial! To prove B is NP-hard, you reduce a known NP-hard problem A to B (A ≤p B), NOT B to A. Many students get this wrong. Think of it as: "If I could solve B, I could solve A."

3.2 How a Reduction Works (Illustration)

To show problem B is NP-hard:

1. Pick a known NP-complete problem A.
2. Construct a polynomial-time transformation f:
     Given any instance x of A,
     produce an instance f(x) of B
     such that x is a YES-instance of A ⇔ f(x) is a YES-instance of B.
3. Show that f runs in polynomial time.
4. Prove correctness: x ∈ A ⇔ f(x) ∈ B.

Combined with showing B ∈ NP, this proves B is NP-complete.

4. The Cook-Levin Theorem: SAT is NP-Complete

The Cook-Levin Theorem (1971) was the first proof that an NP-complete problem exists. It states:

Theorem (Cook-Levin): The Boolean satisfiability problem (SAT) is NP-complete.

4.1 SAT Problem Definition

Input: A Boolean formula φ in conjunctive normal form (CNF).
Question: Is there a truth assignment to the variables that makes φ true?

Example:

φ = (x1 ∨ ¬x2 ∨ x3) ∧ (¬x1 ∨ x2) ∧ (x2 ∨ ¬x3)

Assignment: x1=T, x2=T, x3=F
  Clause 1: (T ∨ F ∨ F) = T
  Clause 2: (F ∨ T) = T
  Clause 3: (T ∨ T) = T
  Result: T ∧ T ∧ T = TRUE  ⇒  Satisfiable!

4.2 Proof Sketch

  1. SAT ∈ NP: Given a truth assignment (certificate), we can evaluate φ in polynomial time.
  2. SAT is NP-hard: For any problem L ∈ NP, there is a polynomial-time Turing machine M that verifies L. We encode the entire computation of M on input (x, y) as a Boolean formula φ such that φ is satisfiable if and only if there exists a certificate y that M accepts. This encoding uses variables for each cell of the tape, each state, and each time step, producing a formula of polynomial size.
Historical note: Stephen Cook proved this in 1971. Independently, Leonid Levin proved a similar result in the Soviet Union in 1973. This theorem opened the floodgates: once one NP-complete problem was known, others could be proven NP-complete by reduction from SAT.

5. Proving NP-Completeness

To prove a problem Q is NP-complete, you must show two things:

  1. Q ∈ NP: Exhibit a polynomial-time verifier (describe the certificate and how to check it).
  2. Q is NP-hard: Reduce a known NP-complete problem to Q in polynomial time.
TEMPLATE FOR NP-COMPLETENESS PROOF:

Theorem: Problem Q is NP-complete.
Proof:
  (1) Q ∈ NP:
      Certificate: [describe what the witness is]
      Verification: [describe how to check in poly-time]

  (2) Q is NP-hard:
      We reduce [Known NPC Problem] to Q.
      Given an instance I of [Known NPC], construct instance I' of Q as follows:
        [describe the transformation]
      Claim: I is a YES-instance ⇔ I' is a YES-instance.
        (⇒) If I is YES, then ... so I' is YES.
        (⇐) If I' is YES, then ... so I is YES.
      The transformation runs in polynomial time because [reason].
  QED.

6. Important NP-Complete Problems

6.1 SAT (Boolean Satisfiability)

Input: A Boolean formula φ in CNF.
Question: Is φ satisfiable?
NP-complete by: Cook-Levin Theorem (the "first" NPC problem).

6.2 3-SAT

Input: A Boolean formula φ in CNF where each clause has exactly 3 literals.
Question: Is φ satisfiable?
NP-complete by: Reduction from SAT.

Reduction idea (SAT ≤p 3-SAT):

  • A clause with 1 literal (l): Replace with (l ∨ a ∨ b) ∧ (l ∨ a ∨ ¬b) ∧ (l ∨ ¬a ∨ b) ∧ (l ∨ ¬a ∨ ¬b) using fresh variables a, b.
  • A clause with 2 literals (l1 ∨ l2): Replace with (l1 ∨ l2 ∨ a) ∧ (l1 ∨ l2 ∨ ¬a).
  • A clause with 3 literals: Keep as is.
  • A clause with k > 3 literals (l1 ∨ ... ∨ lk): Introduce k−3 fresh variables y1, ..., yk-3 and create: (l1 ∨ l2 ∨ y1) ∧ (¬y1 ∨ l3 ∨ y2) ∧ ... ∧ (¬yk-3 ∨ lk-1 ∨ lk).
Important: 2-SAT is in P! It can be solved in O(n + m) using strongly connected components on the implication graph. The jump from 2 to 3 literals per clause changes the problem from polynomial to NP-complete.

6.3 CLIQUE

Input: An undirected graph G = (V, E) and an integer k.
Question: Does G contain a complete subgraph (clique) of size k?
NP-complete by: Reduction from 3-SAT.

Reduction (3-SAT ≤p CLIQUE):

Given a 3-SAT formula φ with m clauses:

  1. For each literal in each clause, create a vertex. Label it with its clause number and literal.
  2. Add an edge between vertices u and v if:
    • u and v are in different clauses, AND
    • u and v are not complementary (one is not the negation of the other).
  3. Set k = m (the number of clauses).

Correctness: φ is satisfiable ⇔ G has a clique of size m. A satisfying assignment picks one true literal per clause; these form a clique because literals from different clauses that are simultaneously true cannot be contradictory.

Worked Example:

φ = (x1 ∨ ¬x2 ∨ x3) ∧ (¬x1 ∨ x2 ∨ x3) ∧ (x1 ∨ x2 ∨ ¬x3)

Vertices:
  Clause 1: (1,x1), (1,¬x2), (1,x3)
  Clause 2: (2,¬x1), (2,x2), (2,x3)
  Clause 3: (3,x1), (3,x2), (3,¬x3)

Edges exist between all inter-clause pairs EXCEPT complementary ones:
  No edge: (1,x1)-(2,¬x1), (1,¬x2)-(3,x2), (1,x3)-(3,¬x3), (2,x3)-(3,¬x3), (2,x2)-(1,¬x2)

k = 3 (number of clauses)

One clique of size 3: {(1,x1), (2,x2), (3,x2)}
  ⇒ x1=T, x2=T satisfies φ (verify: each clause has a true literal)

6.4 VERTEX COVER

Input: An undirected graph G = (V, E) and an integer k.
Question: Is there a set S ⊆ V with |S| ≤ k such that every edge in E has at least one endpoint in S?
NP-complete by: Reduction from CLIQUE.

Reduction (CLIQUE ≤p VERTEX COVER):

Given (G, k) for CLIQUE, construct (G̅, |V| − k) for VERTEX COVER, where G̅ is the complement graph. S is a clique of size k in G ⇔ V \ S is a vertex cover of size |V| − k in G̅.

6.5 HAMILTONIAN CYCLE

Input: An undirected graph G = (V, E).
Question: Does G have a cycle that visits every vertex exactly once?
NP-complete by: Reduction from VERTEX COVER.

6.6 SUBSET SUM

Input: A set S = {s1, s2, ..., sn} of positive integers and a target integer t.
Question: Is there a subset S' ⊆ S such that the elements of S' sum to exactly t?
NP-complete by: Reduction from 3-SAT.

Example:

S = {2, 3, 7, 8, 10}, t = 15
Subset {2, 3, 10}: 2 + 3 + 10 = 15 ⇒ YES

S = {1, 5, 11, 13}, t = 12
No subset sums to 12 ⇒ NO
GATE Tip: SUBSET SUM has a pseudo-polynomial time DP algorithm O(n · t). This does NOT place it in P because the input size is O(n · log t), and t can be exponential in the number of bits. Problems with pseudo-polynomial algorithms are called weakly NP-complete.

6.7 Graph Coloring (k-Coloring for k ≥ 3)

Input: An undirected graph G = (V, E) and an integer k ≥ 3.
Question: Can the vertices of G be colored with k colors such that no two adjacent vertices share a color?
NP-complete by: Reduction from 3-SAT (for 3-coloring).

Note: 2-coloring is in P (it is equivalent to checking bipartiteness via BFS/DFS).

6.8 TSP (Decision Version)

Input: A complete weighted graph G and an integer k.
Question: Is there a Hamiltonian cycle (tour) of total weight ≤ k?
NP-complete by: Reduction from HAMILTONIAN CYCLE.

Reduction: Given graph G for HAM-CYCLE, create a complete graph where edges in G have weight 0 and all other edges have weight 1. Set k = 0. Then G has a Hamiltonian cycle ⇔ the complete graph has a tour of weight 0.

9. Complementary Relationships

Several NP-complete problems are closely related through graph complement:

Problem Complement Problem Relationship
CLIQUE (size k in G) INDEPENDENT SET (size k in G̅) S is a clique in G ⇔ S is an independent set in G̅
VERTEX COVER (size k) INDEPENDENT SET (size |V|−k) S is a vertex cover ⇔ V\S is an independent set
GATE Tip: These relationships are frequently tested. Know them by heart:
Clique of size k in G ⇔ Independent set of size k in complement of G
Vertex cover of size k ⇔ Independent set of size |V|−k in same graph

10. Approximation Algorithms

Since NP-complete problems are believed to be unsolvable in polynomial time, we often settle for approximation algorithms — polynomial-time algorithms that find near-optimal solutions.

Problem Approximation Ratio Algorithm
VERTEX COVER 2-approximation Pick any edge, add both endpoints, remove covered edges, repeat
TSP (triangle inequality) 3/2 (Christofides) MST + minimum weight perfect matching on odd-degree vertices
SET COVER O(ln n) Greedy: always pick the set covering the most uncovered elements
MAX-SAT 3/4 Randomized rounding of LP relaxation
MAX-CUT 0.878 (GW) Semidefinite programming relaxation

10.1 Vertex Cover 2-Approximation Pseudocode

APPROX-VERTEX-COVER(G):
  C = {}
  E' = E(G)
  while E' is not empty:
      pick any edge (u, v) from E'
      C = C ∪ {u, v}
      remove all edges incident to u or v from E'
  return C

// C is at most 2x the size of the optimal vertex cover.

Why 2-approximation? Each edge picked contributes 2 vertices to C. The optimal cover must include at least one endpoint of each picked edge. Since picked edges share no endpoints, OPT ≥ (number of picked edges). Our solution has size 2 × (number of picked edges) ≤ 2 · OPT.

11. The P vs NP Question

The P vs NP question asks: Is P = NP? It is one of the seven Millennium Prize Problems (Clay Mathematics Institute, $1 million prize).

11.1 What Would P = NP Mean?

  • Every problem whose solution can be quickly verified can also be quickly solved.
  • All NP-complete problems would have polynomial-time algorithms.
  • Modern cryptography (RSA, Diffie-Hellman) would collapse, as factoring and discrete log would be easy.
  • Many optimization problems in logistics, scheduling, and AI would become tractable.

11.2 What Does Most of the Community Believe?

Most computer scientists believe P ≠ NP. Despite decades of effort, no one has found polynomial-time algorithms for any NP-complete problem. However, there is no proof of this.

11.3 Consequences of the Answer

If P = NP If P ≠ NP
All NPC problems solvable efficiently NPC problems inherently require exponential time
Cryptography breaks Cryptography remains secure (assuming other hardness)
Mathematical proofs could be found automatically Creativity in proof construction remains essential
NP-intermediate class would be empty NP-intermediate problems exist (Ladner's Theorem)
Ladner's Theorem (1975): If P ≠ NP, then there exist problems in NP that are neither in P nor NP-complete. These are called NP-intermediate problems. Candidates include Graph Isomorphism and Integer Factoring (though neither is proven to be NP-intermediate).

12. Summary of Key Concepts for GATE

Concept What to Remember
P Problems solvable in polynomial time. P ⊆ NP.
NP Problems verifiable in polynomial time with a certificate. Does NOT mean "not polynomial."
co-NP Complements of NP problems. NO answer verifiable in poly time.
NP-hard At least as hard as all of NP. May not be in NP (e.g., Halting Problem).
NP-complete In NP AND NP-hard. The hardest problems that are still in NP.
Reduction direction To prove B is NPC: reduce known NPC → B (not the other way).
2-SAT vs 3-SAT 2-SAT ∈ P, 3-SAT is NP-complete.
2-Coloring vs 3-Coloring 2-Coloring ∈ P (bipartiteness), 3-Coloring is NP-complete.
Pseudo-polynomial Poly in value, not input size. SUBSET SUM O(nt) is pseudo-poly. Weakly NP-complete.

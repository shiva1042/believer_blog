1.2 Conversion Methods

Binary to Decimal

Multiply each bit by its positional weight and sum: (1011)2 = 1×23 + 0×22 + 1×21 + 1×20 = 8+0+2+1 = 11

Decimal to Binary

Repeated division by 2, reading remainders bottom-to-top: 25 / 2 = 12 R1, 12 / 2 = 6 R0, 6 / 2 = 3 R0, 3 / 2 = 1 R1, 1 / 2 = 0 R1 giving (11001)2

Binary to Octal

Group bits in 3s from right: (110 101 011)2 = (653)8

Binary to Hexadecimal

Group bits in 4s from right: (0011 1010 1111)2 = (3AF)16

2. Signed Number Representations

2.1 Sign-Magnitude

The MSB is the sign bit (0 = positive, 1 = negative). Remaining bits represent magnitude.

  • Range for n bits: -(2n-1 - 1) to +(2n-1 - 1)
  • Two representations of zero: +0 (0000...0) and -0 (1000...0)

2.2 One's Complement

Positive numbers are same as sign-magnitude. Negative numbers are obtained by inverting all bits.

  • Range for n bits: -(2n-1 - 1) to +(2n-1 - 1)
  • Two representations of zero: +0 (0000...0) and -0 (1111...1)
  • Addition requires end-around carry

2.3 Two's Complement

The most widely used representation. Negative numbers are obtained by inverting all bits and adding 1.

  • Range for n bits: -2n-1 to +(2n-1 - 1)
  • Only one representation of zero
  • One extra negative number: -2n-1 has no positive counterpart

3. Fixed Point Representation

In fixed point, the binary point is at a fixed position. For Q(m.n) format: m integer bits and n fractional bits.

Example: Q(4.4) representation of 5.75:

5 = 0101, 0.75 = .1100 so 5.75 = 0101.1100

4. IEEE 754 Floating Point

4.1 General Format

A floating point number is represented as: (-1)S × 1.M × 2(E - Bias)

Where S = sign bit, M = mantissa (fraction), E = biased exponent.

4.5 Example: Converting -6.5 to IEEE 754 Single Precision

  1. Sign: negative, so S = 1
  2. 6.5 in binary = 110.1
  3. Normalize: 1.101 × 22
  4. Exponent: 2 + 127 = 129 = 10000001
  5. Mantissa: 10100000000000000000000 (23 bits)
  6. Result: 1 10000001 10100000000000000000000
  7. Hex: 0xC0D00000

5. Overflow and Underflow

5.1 Integer Overflow

In 2's complement, overflow occurs when adding two numbers of the same sign produces a result of the opposite sign.

Overflow detection rules (2's complement addition):

  • Positive + Positive = Negative: OVERFLOW
  • Negative + Negative = Positive: OVERFLOW
  • Positive + Negative: NEVER overflows

Hardware detection: Overflow = Cn XOR Cn-1 (carry into MSB XOR carry out of MSB)

7. GATE Exam Tips

  • 2's complement is the default representation in most GATE questions unless stated otherwise.
  • Negating in 2's complement: invert all bits and add 1, OR keep all bits from right up to and including the first '1', then invert the rest.
  • IEEE 754 single precision: remember bias = 127, exponent range 1-254 for normalized numbers.
  • Denormalized numbers use exponent = 0 and the implicit leading bit is 0 (not 1).
  • For BCD addition: if sum of two BCD digits > 9, add 6 (0110) to correct.
  • The smallest positive denormalized float (single): 2-149.

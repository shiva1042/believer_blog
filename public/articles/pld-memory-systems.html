PLDs, ROM, RAM, and Memory Types

Digital Logic

Programmable Logic Devices (PLDs) and memory systems form the backbone of modern digital design. This article covers the full spectrum from combinational logic implementation using ROM, PLA, and PAL to the memory hierarchy including SRAM, DRAM, and various ROM types. These topics are drawn from Morris Mano's Digital Design (Chapters 5 and 7) and are high-frequency GATE CSE questions.

1. Programmable Logic Devices (PLDs) — Overview

A PLD is an IC that contains a large number of gates whose interconnections can be programmed by the user. The key idea: any combinational function can be expressed as a sum of products (SOP). A PLD provides an AND-plane (to form product terms) and an OR-plane (to sum selected products).

Product Terms] B --> C[OR Plane

The three classical PLD architectures differ in which planes are programmable:

Device AND Plane OR Plane Notes
ROM Fixed (full decoder) Programmable Generates all 2n minterms; OR plane selects which to sum
PLA Programmable Programmable Most flexible; both planes user-defined
PAL Programmable Fixed Simpler than PLA; each output has a fixed number of product terms
GATE Tip: A ROM with n input lines and m output lines can implement any m functions of n variables without minimisation. It has 2n × m programmable connections in the OR plane.

2. ROM as a Combinational Function Generator

A ROM (Read-Only Memory) is essentially a complete decoder (AND plane) followed by a programmable OR plane. Because the decoder produces every minterm, any Boolean function can be stored directly from its truth table — no minimisation is required.

2.1 ROM Structure

A 2n × m ROM has:

  • n input (address) lines
  • An internal n-to-2n decoder (AND plane)
  • m output lines
  • 2n × m programmable OR-plane connections (fuse or transistor based)

2.2 Worked Example: Implementing Functions with ROM

Problem: Implement the following three functions using a ROM: 

F1(A, B, C) = Sum of minterms(1, 3, 5, 7)
F2(A, B, C) = Sum of minterms(0, 2, 4, 6)
F3(A, B, C) = Sum of minterms(2, 3, 6, 7)

Solution: We need a ROM with 3 inputs and 3 outputs, i.e., 23 × 3 = 8 × 3 ROM.

Observation: F1 = C (LSB), F2 = C' (complement of C), F3 = B. But with a ROM, we do not need to derive these simplifications — we simply program the truth table directly into the OR plane.

Key Insight: ROM size grows exponentially with the number of inputs (2n rows). For functions with many inputs but few product terms, a PLA is more efficient.

3. Programmable Logic Array (PLA)

A PLA has both AND and OR planes programmable. This means we only need to implement the product terms that actually appear in the minimised expressions — no need for a full decoder.

3.1 PLA Structure

  • n inputs, each available in true and complemented form (2n lines into AND plane)
  • k AND gates (product terms) — programmable connections to input lines
  • m OR gates (outputs) — programmable connections to product terms
  • Optional: output inverters (XOR gates) for complemented outputs

3.2 PLA Programming Table

A PLA programming table lists each product term and shows which inputs are connected (1 = true, 0 = complement, dash = not connected) and which outputs include that term.

Example: Implement using a PLA with 3 inputs, 4 product terms, 2 outputs:

F1(A, B, C) = AB' + AC + A'BC
F2(A, B, C) = AC + BC

After minimisation, the shared product terms are: AB', AC, A'BC, BC

Product Term A B C F1 F2
AB'10—10
AC1—111
A'BC01110
BC—1101

The PLA requires 4 product terms. Notice that AC is shared between F1 and F2 — this sharing is a key advantage of PLA. A PAL would not allow this sharing easily.

Feature ROM PLA

Cost for many inputs Exponential Proportional to k
Speed Faster (simpler routing) Slower (two programmable planes)

4. Programmable Array Logic (PAL)

A PAL has a programmable AND plane and a fixed OR plane. Each output OR gate is hard-wired to a specific set of AND gates (typically 4 or 8 product terms per output).

4.2 Key Characteristics of PAL

  • Each output has a fixed number of product terms (e.g., 4 AND gates per OR gate)
  • Product terms cannot be shared between outputs (unlike PLA)
  • Simpler and faster than PLA because the OR plane has fixed connections
  • If a function requires more product terms than available, it cannot be implemented directly
  • Some PALs have registered outputs (flip-flops) for sequential logic

4.3 Worked Example: PAL Implementation

Problem: Implement using a PAL with 3 inputs and max 3 product terms per output:

F1(A, B, C) = AB + AC + BC
F2(A, B, C) = AB + A'C'

Solution: Each function must be expressed with at most 3 product terms.

For output F1, the AND plane is programmed for: AB, AC, BC (3 terms — fits)

For output F2, the AND plane is programmed for: AB, A'C' (2 terms — fits, one AND gate unused)

Note: In a PAL, even though both F1 and F2 use the term AB, it must be generated separately in each output's AND-gate group. In a PLA, the AB term would be shared.

5. FPGA Basics

A Field-Programmable Gate Array (FPGA) is a more advanced PLD that can implement complex digital systems. Unlike ROM/PLA/PAL which use a two-level AND-OR structure, FPGAs use a matrix of configurable logic blocks (CLBs) connected by programmable routing.

5.1 FPGA Architecture

Interconnect"] CLB2 --- RI CLB3["CLB"] --- RI CLB4["CLB"] --- RI IOB3["I/O Block"] --- CLB3 IOB4["I/O Block"] --- CLB4 end

Key components of an FPGA:

Component Function
CLB (Configurable Logic Block) Contains LUTs (Look-Up Tables), flip-flops, and multiplexers. Implements combinational and sequential logic.
LUT (Look-Up Table) A small memory (typically 4-input or 6-input) that can implement any Boolean function of its inputs. Functionally equivalent to a small ROM.
I/O Blocks Interface between internal logic and external pins. Configurable for input, output, or bidirectional.
Programmable Interconnect Routing channels and switch matrices that connect CLBs to each other and to I/O blocks.
Block RAM Dedicated memory blocks for on-chip storage.
DSP Slices Hardened multiply-accumulate units for signal processing.
GATE Note: A k-input LUT is functionally a 2k × 1 ROM. A 4-input LUT stores 24 = 16 bits and can implement any function of 4 variables.

6. RAM Types: SRAM vs DRAM

Random Access Memory (RAM) is volatile memory that allows both read and write operations. The two fundamental types are Static RAM (SRAM) and Dynamic RAM (DRAM).

6.1 SRAM (Static RAM)

SRAM stores each bit in a bistable flip-flop circuit (typically 6 transistors per cell). Data is retained as long as power is supplied — no refresh is needed.

A typical 6T SRAM cell consists of:

  • 2 cross-coupled inverters (4 transistors) forming the latch
  • 2 access transistors controlled by the word line

6.2 DRAM (Dynamic RAM)

DRAM stores each bit as charge on a capacitor (1 transistor + 1 capacitor per cell). The charge leaks over time, so periodic refresh is required (typically every 2-8 ms).

6.3 SRAM vs DRAM Comparison

Parameter SRAM DRAM
Storage element Flip-flop (6 transistors) Capacitor (1T + 1C)

GATE Favourite: DRAM uses address multiplexing — the row address is sent first (latched by RAS), then the column address (latched by CAS). This halves the number of address pins needed but requires two cycles per access.

7. ROM Types

ROM (Read-Only Memory) is non-volatile memory. Data is retained even when power is off. Various ROM types differ in how they are programmed and whether they can be erased.

Type Programming Erasure Notes
Mask ROM Factory (mask) Not possible Cheapest for high volume; programmed during IC fabrication
PROM User (one-time) Not possible Uses fusible links; blown fuse = 0, intact = 1
EPROM User (electrical) UV light (bulk erase) Uses floating-gate transistors; quartz window on chip
EEPROM User (electrical) Electrical (byte-level) Can erase individual bytes; slower write, limited cycles (~105)
Flash Memory User (electrical) Electrical (block-level) Erases in blocks; faster than EEPROM; used in SSDs, USB drives (~105-106 cycles)
Key Difference: EEPROM erases at the byte level while Flash erases in blocks. Flash is cheaper per bit and denser, making it the dominant non-volatile technology today.

8. Memory Expansion

When a single memory chip does not provide enough capacity, multiple chips are combined using two techniques: word expansion and bit expansion.

8.1 Bit Expansion (Increasing Word Size)

Multiple memory chips are connected in parallel to increase the number of bits per word. All chips share the same address and control lines.

Example: Create 1K × 8 memory using 1K × 4 chips.

  • Need 8 / 4 = 2 chips
  • Both chips receive the same 10-bit address (A0-A9)
  • Chip 1 provides bits D0-D3, Chip 2 provides bits D4-D7
  • CS and R/W lines are common

8.2 Word Expansion (Increasing Number of Words)

Memory chips are selected one at a time using a decoder connected to extra address lines. This increases the address space.

Example: Create 4K × 8 memory using 1K × 8 chips.

  • Need 4K / 1K = 4 chips
  • Lower 10 address bits (A0-A9) go to all chips
  • Upper 2 address bits (A10-A11) go to a 2-to-4 decoder
  • Decoder outputs connect to CS (chip select) of each chip
  • Only one chip is enabled at a time

8.3 Combined Expansion

Problem: Design a 4K × 16 memory system using 1K × 4 chips.

Solution:

  • Total bits needed: 4K × 16 = 65,536 bits
  • Bits per chip: 1K × 4 = 4,096 bits
  • Total chips needed: 65,536 / 4,096 = 16 chips
  • Bit expansion: 16 / 4 = 4 chips per row (to get 16-bit word)
  • Word expansion: 4K / 1K = 4 rows of chips
  • Layout: 4 rows × 4 columns = 16 chips
  • Decoder: 2-to-4 (using A10, A11) to select the row
Formula: Number of chips = (Required words / Chip words) × (Required bits per word / Chip bits per word)

9. Memory Address Decoding

Address decoding determines which memory chip or I/O device responds to a given address. There are two main approaches:

9.1 Full Address Decoding

Every address bit is used in the decoding logic. Each memory location has exactly one address. No address aliasing occurs.

Example: A system with 16-bit addresses uses a 2K × 8 RAM chip. The chip needs 11 address lines (A0-A10). The remaining 5 lines (A11-A15) are decoded to generate the chip select. 

Address range for CS when A15-A11 = 10110:
  Start: 1011 0 000 0000 0000 = B000h
  End:   1011 0 111 1111 1111 = B7FFh
  Range: B000h to B7FFh (2K bytes)

9.2 Partial Address Decoding

Not all high-order address bits are used. This is simpler but causes address aliasing — the same physical memory appears at multiple address ranges.

Example: If only A15 is used for chip select (A15 = 0 selects the chip) and the chip is 2K, then A14-A11 are don't cares. The 2K block is mirrored at every 2K boundary in the lower half of the address space (16 aliases).

Trade-off: Partial decoding uses fewer gates but wastes address space due to aliasing. Full decoding is more complex but uses the address space efficiently.

9.3 Address Decoding Example

Problem: A microprocessor has a 16-bit address bus. Design address decoding for:

  • 8K ROM at address 0000h-1FFFh
  • 4K RAM at address 4000h-4FFFh
  • 4K RAM at address 5000h-5FFFh

Solution:

8K ROM (0000-1FFF): A15=0, A14=0, A13=0  => CS_ROM = A15 + A14 + A13 (active low)
  Uses A0-A12 for internal addressing.

4K RAM-1 (4000-4FFF): A15=0, A14=1, A13=0, A12=0 => CS_RAM1 = A15 + A14' + A13 + A12
  Uses A0-A11 for internal addressing.

4K RAM-2 (5000-5FFF): A15=0, A14=1, A13=0, A12=1 => CS_RAM2 = A15 + A14' + A13 + A12'
  Uses A0-A11 for internal addressing.

A 3-to-8 decoder with inputs A13, A14, A15 can generate chip selects, with additional gating on A12 where needed.

10. Worked Examples for GATE

Example 1: ROM Size Calculation

Q: A combinational circuit has 5 inputs and 3 outputs. What is the minimum ROM size needed?

A: ROM size = 25 × 3 = 32 × 3 = 96 bits. The ROM has 32 words of 3 bits each (32 × 3 ROM).

Example 2: PLA Size

Q: A PLA has 12 inputs, 24 product terms, and 6 outputs. How many programmable connections does it have?

A: AND plane: 2 × 12 × 24 = 576 connections (each input in true and complement form). OR plane: 24 × 6 = 144 connections. Total = 576 + 144 = 720 programmable connections.

Example 3: Memory Chip Count

Q: How many 512 × 4 RAM chips are needed to construct a 2K × 8 memory?

A: Word expansion: 2K / 512 = 4. Bit expansion: 8 / 4 = 2. Total chips = 4 × 2 = 8 chips.

Example 4: Decoder for Memory

Q: A 64K memory space is divided into 4 equal blocks using a 2-to-4 decoder. Which address lines are connected to the decoder?

A: 64K = 216 addresses. 4 blocks means each block is 16K = 214. The two highest address lines A14 and A15 go to the decoder. A0-A13 go to the memory chips.

2. Group 2: m(5,7) -- A'BCD + A'BC'D = A'BD (2 cells)

Problem 2: Boolean Algebra Simplification

Problem: Simplify F = AB + A'C + BC using the consensus theorem and Boolean algebra laws.
Approach: Consensus theorem: XY + X'Z + YZ = XY + X'Z (the term YZ is redundant).
Step-by-step Solution:
F = AB + A'C + BC

By the consensus theorem with X=A, Y=B, Z=C:
AB + A'C + BC = AB + A'C

Proof:
BC = BC(A + A') = ABC + A'BC
ABC is covered by AB (when C=1, AB includes ABC).
A'BC is covered by A'C (when B=1, A'C includes A'BC).
Therefore BC is redundant.

F = AB + A'C
Key Takeaway: The consensus theorem is one of the most useful simplification tools. Other key identities: absorption (A + AB = A), De Morgan's, XOR properties (A XOR 0 = A, A XOR 1 = A'). In GATE, identify the consensus pattern to quickly simplify.

Problem 3: JK Flip-Flop State Table

Problem: A sequential circuit has one JK flip-flop (state Q) and one input X. J = XQ', K = X'Q. Find the state table, state diagram, and identify the circuit's function.
Approach: For each combination of current state Q and input X, compute J and K, then find next state Q+ using: Q+ = JQ' + K'Q.
Step-by-step Solution:
QXJ = XQ'K = X'QQ+ = JQ' + K'Q
00000
01101
10010
11001
Simplified: Q+ = X (the next state always equals the input!).

This circuit is a D flip-flop equivalent (since Q+ = X regardless of current state).
The JK flip-flop with J=XQ' and K=X'Q behaves like a D flip-flop with D=X.
GATE Tip: JK flip-flop conversion: D=JQ'+K'Q. T=JQ'+K'Q where T means toggle. To convert D to JK: J=D, K=D'. To convert T to JK: J=K=T. Excitation tables are crucial for sequential circuit design problems.

Problem 4: Number System Conversion

Problem: (a) Convert -43 to 8-bit 2's complement. (b) Perform 25 - 43 using 8-bit 2's complement arithmetic. (c) Detect if overflow occurred.
Approach: 2's complement of N: flip bits of |N| and add 1. Subtraction A-B = A + (-B).
Step-by-step Solution:
(a) -43 in 2's complement:
43 = 00101011
Flip: 11010100
Add 1: 11010101
Verify: 11010101 = -128+64+16+4+1 = -128+85 = -43. Correct.

(b) 25 - 43:
25 = 00011001
-43 = 11010101
  00011001  (25)
+ 11010101  (-43)
----------
  11101110  (-18)
Verify: 11101110 = -128+64+32+8+4+2 = -128+110 = -18. Result: -18

(c) Overflow check:
Positive + Negative: Overflow CANNOT occur (result always fits).
MSBs: 0+1 = 1 (no carry in or out of sign bit issues).
No overflow.
Rule: Overflow occurs only when two numbers of the same sign produce a result with the opposite sign.
GATE Tip: 2's complement range for n bits: -2n-1 to 2n-1-1. For 8 bits: -128 to +127. Overflow detection: carry_in to MSB XOR carry_out from MSB. Alternatively: same-sign operands produce opposite-sign result.

Problem 5: IEEE 754 Representation

Problem: Represent -12.625 in IEEE 754 single-precision (32-bit) format. Also find the decimal value of the bit pattern 0 10000010 10100000000000000000000.
Approach: IEEE 754: sign(1) + exponent(8) + mantissa(23). Bias = 127. Stored exponent = actual + 127. Mantissa stores fractional part (implicit leading 1).
Step-by-step Solution:
Part 1: -12.625
Sign: 1 (negative)
12 in binary: 1100
0.625 in binary: 0.625 × 2 = 1.25 → 1. 0.25 × 2 = 0.5 → 0. 0.5 × 2 = 1.0 → 1. So 0.625 = 0.101
12.625 = 1100.101
Normalized: 1.100101 × 23
Exponent: 3 + 127 = 130 = 10000010
Mantissa: 10010100000000000000000 (drop leading 1, pad with zeros)

IEEE 754: 1 10000010 10010100000000000000000

Part 2: 0 10000010 10100000000000000000000
Sign: 0 (positive)
Exponent: 10000010 = 130. Actual = 130 - 127 = 3.
Mantissa: 1.10100000... = 1 + 0.5 + 0.125 = 1.625
Value = +1.625 × 23 = 1.625 × 8 = 13.0
GATE Tip: Special values: exponent all 0s with mantissa 0 = ±0. All 0s exponent with non-zero mantissa = denormalized. All 1s exponent with mantissa 0 = ±infinity. All 1s with non-zero mantissa = NaN. Range for single precision: ≈ ±3.4 × 1038.

Problem 6: Pipeline Performance

Problem: A 5-stage pipeline has stage delays: IF=1ns, ID=1.5ns, EX=2ns, MEM=1ns, WB=0.5ns. Find: (a) Clock cycle time (pipelined and non-pipelined). (b) Time for 100 instructions. (c) Speedup. (d) CPI with 20% stalls (1 cycle each).
Approach: Pipelined clock = max stage delay. Non-pipelined = sum of stages. Time for n instructions = (k + n - 1) × cycle_time where k = stages.
Step-by-step Solution:
(a) Clock cycle:
Non-pipelined: 1 + 1.5 + 2 + 1 + 0.5 = 6 ns per instruction
Pipelined: max(1, 1.5, 2, 1, 0.5) = 2 ns (bottleneck is EX stage)

(b) Time for 100 instructions:
Non-pipelined: 100 × 6 = 600 ns
Pipelined: (5 + 100 - 1) × 2 = 104 × 2 = 208 ns

(c) Speedup:
S = 600/208 = 2.88
Ideal speedup = 6/2 = 3.0 (limited by imbalanced stages)
For balanced k-stage pipeline, ideal speedup = k.

(d) CPI with stalls:
Ideal CPI = 1 (pipelined). With 20% instructions causing 1-cycle stall:
CPI = 1 + 0.20 × 1 = 1.2
Effective time per instruction = 1.2 × 2 = 2.4 ns
Effective throughput = 1/2.4 ns ≈ 417 MIPS
GATE Tip: Pipeline speedup = Non-pipelined time / Pipelined time. Maximum speedup = number of stages (for balanced pipeline, infinite instructions). With stalls: CPI = 1 + stall_frequency × stall_cycles. Speedup = ideal_CPI / actual_CPI × freq_ratio.

Problem 7: Pipeline Hazards

Problem: Given instructions: (1) ADD R1, R2, R3; (2) SUB R4, R1, R5; (3) AND R6, R1, R7; (4) OR R8, R9, R10. Identify data hazards. How many stalls without forwarding? How many with forwarding?
Approach: RAW hazards: source of later instruction = destination of earlier. Without forwarding, must wait for WB. With forwarding, result available after EX.
Step-by-step Solution:
Dependencies:
- I2 reads R1 (written by I1): RAW hazard, distance = 1
- I3 reads R1 (written by I1): RAW hazard, distance = 2
- I4: no dependency on R1.

Without forwarding (result available after WB, stage 5):
I1 writes R1 at end of cycle 5.
I2 needs R1 at ID (cycle 3). Must wait until cycle 6. 2 stall cycles for I2.
I3 needs R1 at ID (cycle 4, but delayed). With I2 stalled, I3 is also delayed.
After I2's stalls: I3 needs R1 at its ID. I1 has completed WB by then. 0 additional stalls for I3.
Total stalls without forwarding: 2

With forwarding (result available after EX, stage 3):
I1 produces R1 at end of EX (cycle 3).
I2 needs R1 at EX (cycle 4 with forwarding from I1's EX to I2's EX). 0 stalls for I2.
I3 needs R1 at EX (cycle 5). I1's result available from cycle 3. 0 stalls for I3.
Total stalls with forwarding: 0
Key Takeaway: Load-use hazard still requires 1 stall even with forwarding (load result available after MEM, not EX). Other ALU-ALU hazards are fully resolved by forwarding. Control hazards (branches) typically cause 1-2 stalls depending on when branch target is computed.

Problem 9: AMAT Calculation

Problem: L1 cache: hit time = 1 cycle, miss rate = 5%. L2 cache: hit time = 10 cycles, miss rate = 20% (local). Main memory: access time = 100 cycles. Find AMAT.
Approach: AMAT = L1_hit_time + L1_miss_rate × (L2_hit_time + L2_miss_rate × memory_time).
Step-by-step Solution:
L2 miss penalty (going to memory) = 100 cycles
L2 access time = 10 + 0.20 × 100 = 10 + 20 = 30 cycles

AMAT = 1 + 0.05 × 30 = 1 + 1.5 = 2.5 cycles

Alternatively (global miss rate perspective):
Global L2 miss rate = 0.05 × 0.20 = 0.01 (1% of all accesses miss both caches)
AMAT = 1 + 0.05 × 10 + 0.01 × 100 = 1 + 0.5 + 1.0 = 2.5 cycles. Same answer.
Key Takeaway: Local miss rate = misses in this cache / accesses to this cache. Global miss rate = misses in this cache / total CPU accesses. For multi-level caches, either method gives the same AMAT. GATE frequently tests this with 2-level caches.

Problem 11: Memory Interleaving

Problem: A main memory has access time = 80 ns and cycle time = 120 ns. The processor requires a word every 20 ns. How many memory banks are needed for (a) low-order interleaving to avoid stalls? (b) What is the effective bandwidth?
Approach: With m-way interleaving, successive accesses go to different banks. Need cycle_time / processor_rate ≤ m banks.
Step-by-step Solution:
(a) Number of banks:
Each bank needs cycle_time = 120 ns to be ready again.
Processor requests one word every 20 ns.
Banks needed = ⌈cycle_time / processor_period⌉ = ⌈120/20⌉ = 6 banks minimum
(Round up to power of 2 for addressing: 8 banks)

(b) Effective bandwidth:
With 8-way interleaving and 20 ns between requests:
First word available after access_time = 80 ns
Subsequent words available every 20 ns (if no bank conflicts)

For n words: time = access_time + (n-1) × processor_period = 80 + (n-1) × 20 ns
For large n: effective rate ≈ 1 word per 20 ns = 50 million words/sec
Without interleaving: 1 word per 120 ns = 8.33 million words/sec
Speedup = 120/20 = 6x
Key Takeaway: Low-order interleaving distributes consecutive addresses across banks (good for sequential access). High-order interleaving puts consecutive addresses in the same bank (good for independent access by multiple processors). Ideal banks = cycle_time / access_time_per_word.

Problem 12: Microprogrammed Control

Problem: A microprogrammed control unit has: 128 microinstructions in control memory, each 40 bits wide. 4-bit condition select field, 7-bit branch address field. If each microinstruction takes 200 ns, how large is the control memory? If an instruction requires 5 microinstructions on average, and the clock for hardwired control is 50 ns, compare the performance.
Approach: Control memory size = number of microinstructions × width. Performance comparison: microprogrammed instruction time = #microinstructions × micro_cycle.
Step-by-step Solution:
Control memory size:
128 × 40 bits = 5120 bits = 640 bytes
Address bits needed: log2(128) = 7 bits (matches the branch address field).

Performance comparison:
Microprogrammed: 5 microinstructions × 200 ns = 1000 ns per instruction
Hardwired: assume 1 clock cycle per instruction = 50 ns per instruction

Hardwired is 1000/50 = 20x faster.

However, microprogrammed control is:
- Easier to modify (just change microcode)
- Supports complex instruction sets (CISC)
- Less error-prone to design

Hardwired control is used in RISC processors for speed.
GATE Tip: Microprogrammed control: flexible but slower. Hardwired control: fast but inflexible. Horizontal microprogramming: wide microinstructions, more parallelism. Vertical: narrow, encoded fields, slower but smaller control memory. GATE may ask you to calculate control memory dimensions.

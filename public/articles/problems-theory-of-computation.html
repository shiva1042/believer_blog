Problem 1: DFA Construction

Problem: Construct a DFA over Σ = {0, 1} that accepts all strings where the number of 0s is divisible by 3.
Approach: Track (count of 0s) mod 3. Three states: q0 (0 mod 3), q1 (1 mod 3), q2 (2 mod 3). Accept state: q0. Input 1 does not change state.

Problem 2: DFA — Minimum States

Problem: What is the minimum number of states in a DFA that accepts binary strings representing numbers divisible by 5? (Leading zeros allowed, Σ = {0,1}, interpret string as binary number.)
Approach: Track the value mod 5 as bits are read left to right. Reading a new bit: new_state = (2 * old_state + bit) mod 5.
Step-by-step Solution:
States: q0 (mod 0), q1 (mod 1), q2 (mod 2), q3 (mod 3), q4 (mod 4).
Accept: q0. Start: q0 (empty string represents 0, divisible by 5).
State (rem)Input 0 (2r mod 5)Input 1 ((2r+1) mod 5)
q0 (0)q0q1
q1 (1)q2q3
q2 (2)q4q0
q3 (3)q1q2
q4 (4)q3q4
Minimum states = 5.
Test: "1010" (=10, divisible by 5): q0→q1→q2→q0→q0. Accept. Correct.
Test: "110" (=6): q0→q1→q3→q1. Reject. Correct.
Key Takeaway: For "binary number mod n," the minimum DFA has exactly n states (for n that is not a power of 2, all states are distinguishable). This is a classic GATE question. Transition: δ(q_r, b) = q_{(2r+b) mod n}.

Problem 5: Pumping Lemma (Regular Languages)

Problem: Prove that L = {anbn | n ≥ 0} is not regular using the pumping lemma.
Approach: Assume L is regular. Let p be the pumping length. Choose a string s in L with |s| ≥ p. Show that for all decompositions s = xyz, pumping fails.
Step-by-step Solution:
Proof by contradiction:
1. Assume L is regular. Let p be the pumping length.
2. Choose s = apbp. s ∈ L and |s| = 2p ≥ p.
3. By pumping lemma, s = xyz where |xy| ≤ p, |y| ≥ 1, and xyiz ∈ L for all i ≥ 0.
4. Since |xy| ≤ p and s starts with p a's, y consists entirely of a's. Let y = ak where 1 ≤ k ≤ p.
5. Pump up: xy2z = ap+kbp. Since k ≥ 1, p+k > p, so #a's ≠ #b's.
Therefore xy2z ∉ L. Contradiction!

6. Pump down: xy0z = xz = ap-kbp. Again, p-k < p, so #a's ≠ #b's. xz ∉ L.

Therefore L is not regular.
GATE Tip: In pumping lemma proofs: (1) YOU choose the string s, (2) the adversary chooses the decomposition xyz (but constrained by |xy| ≤ p, |y| ≥ 1), (3) YOU choose i to show pumping fails. Choose s cleverly so that y must land in a critical region.

Problem 6: Pumping Lemma (Context-Free Languages)

Problem: Prove that L = {anbncn | n ≥ 0} is not context-free.
Approach: Use the CFL pumping lemma: for s ∈ L with |s| ≥ p, s = uvxyz where |vy| ≥ 1, |vxy| ≤ p, and uvixyiz ∈ L for all i ≥ 0.
Step-by-step Solution:
1. Assume L is context-free. Let p be the pumping length.
2. Choose s = apbpcp. |s| = 3p ≥ p.
3. s = uvxyz with |vxy| ≤ p and |vy| ≥ 1.
4. Since |vxy| ≤ p, vxy can span at most two consecutive types of characters:
  - Case 1: vxy is within a's only. Pumping changes #a's but not #b's or #c's.
  - Case 2: vxy spans a's and b's. Pumping changes #a's and/or #b's but not #c's.
  - Case 3: vxy is within b's only. Pumping changes #b's but not #a's or #c's.
  - Case 4: vxy spans b's and c's. Pumping changes #b's and/or #c's but not #a's.
  - Case 5: vxy is within c's only. Pumping changes #c's but not #a's or #b's.

5. In ALL cases, pumping (say i=2) creates a string where the counts of a, b, c are unequal.
uv2xy2z ∉ L in every case.

Contradiction. L is not context-free.
Key Takeaway: The CFL pumping lemma has two substrings v and y that are pumped together. The constraint |vxy| ≤ p means they are "close together" in the string. For three-way equality like anbncn, pumping can affect at most two of the three counts, breaking equality.

Problem 7: CFG Design

Problem: Design a CFG for L = {w ∈ {a,b}* | w has equal number of a's and b's}.
Approach: Use a single variable S. Each derivation must produce equal a's and b's. Key insight: any string with equal a's and b's can be decomposed.
Step-by-step Solution:
Grammar:
S → aSbS | bSaS | ε

Verification:
- "ab": S ⇒ aSbS ⇒ aS b S ⇒ aεbε = ab. Correct.
- "ba": S ⇒ bSaS ⇒ bεaε = ba. Correct.
- "aabb": S ⇒ aSbS ⇒ aaSbSbS ⇒ aaεbεbε = aabb. Correct.
- "abba": S ⇒ aSbS ⇒ aεbS ⇒ aεbbSaS ⇒ abbεaε = abba. Correct.

Why it works: Every non-empty balanced string must start with either 'a' or 'b'. If it starts with 'a', there exists a matching 'b' that splits the string into parts, each with equal a's and b's. The grammar captures this recursive structure.

Alternative simpler grammar:
S → SS | aSb | bSa | ε
This is also correct but is ambiguous.
GATE Tip: CFG design tips: (1) Equal counts of two symbols: pair them recursively. (2) na > nb: generate extra a's at each step. (3) Palindromes: S → aSa | bSb | a | b | ε. (4) Test your grammar with small strings. GATE frequently asks for CFGs.

Problem 8: PDA Construction

Approach: Push a marker for each 'a'. On seeing 'b', start popping. Accept when stack is empty and input is consumed.

Problem 9: Closure Properties

Problem: Determine whether the following are true or false: (a) Regular languages are closed under complementation. (b) CFLs are closed under intersection. (c) CFLs are closed under intersection with regular languages. (d) Recursive languages are closed under complementation.
Approach: Apply known closure property results for each language class.
Step-by-step Solution:
(a) Regular closed under complement: TRUE.
Proof: For DFA M accepting L, swap accept and non-accept states to get DFA for Lc.

(b) CFLs closed under intersection: FALSE.
Counterexample: L1 = {anbncm} and L2 = {ambncn} are both CFLs.
L1 ∩ L2 = {anbncn} which is NOT a CFL.

(c) CFL ∩ Regular = CFL: TRUE.
Proof: Cross-product construction of PDA and DFA. The stack handles the CFL part, the finite control handles the regular part.

(d) Recursive languages closed under complement: TRUE.
Proof: If TM M decides L (halts on all inputs), modify M to swap accept/reject. The new TM decides Lc.

Problem 10: Decidability

Problem: Classify each problem as decidable, RE (but not decidable), or not RE:
(a) Is DFA M's language empty?
(b) Does TM M accept input w?
(c) Is TM M's language equal to Σ*?
(d) Is CFG G ambiguous?
Approach: Apply known decidability results for DFA, TM, and CFG problems.
Step-by-step Solution:
(a) EDFA = {<M> | M is a DFA and L(M) = ∅}: DECIDABLE.
Algorithm: Check if any accept state is reachable from start state using BFS/DFS. Finite graph search terminates.

(b) ATM = {<M, w> | TM M accepts w}: RE but NOT decidable.
This is the Halting Problem variant. A TM can simulate M on w; if M accepts, accept. But if M loops, we loop forever. So it's RE. Not decidable by Rice's theorem (or direct diagonalization).

(c) ALLTM = {<M> | L(M) = Σ*}: NOT RE (co-RE, and complement is not RE either).
Its complement (does M reject some string?) is RE. Since ALLTM is not RE and its complement is RE, ALLTM is co-RE but not RE.

(d) Ambiguity of CFG: NOT DECIDABLE (and not even RE).
There is no algorithm that, given a CFG G, can determine whether G is ambiguous. This is undecidable.
Key Takeaway: DFA/NFA/regex problems are almost always decidable (finite structures). TM problems are usually undecidable (by Rice's theorem: any non-trivial property of TM languages is undecidable). CFG: emptiness and membership are decidable; ambiguity and universality are undecidable.

Problem 11: Myhill-Nerode Theorem

Problem: Use the Myhill-Nerode theorem to find the minimum number of states for a DFA accepting L = {w ∈ {a,b}* | w ends with "ab"}.
Approach: Find equivalence classes of the Myhill-Nerode relation. Each class corresponds to one state in the minimal DFA.
Step-by-step Solution:
Two strings x and y are equivalent iff for all z, xz ∈ L ⇔ yz ∈ L.

Consider what matters about a prefix x for determining if xz ends in "ab":
Only the last 0, 1, or 2 characters of x matter (specifically, the suffix progress toward "ab").

Equivalence classes:
1. [ε]: strings not ending in 'a' and not ending in "ab" (i.e., ending in 'b' or empty). For these, suffix "ab" is needed.
2. [a]: strings ending in 'a' (but not "ab"). For these, suffix "b" completes "ab".
3. [ab]: strings ending in "ab". For these, suffix ε is already in L.

Distinguish: [ε] needs z = "ab" to be in L. [a] needs z = "b". [ab] needs z = ε.
- [ε] vs [a]: z = "b". εb = "b" ∉ L. "a"b = "ab" ∈ L. Distinguishable.
- [a] vs [ab]: z = ε. a ∉ L. "ab" ∈ L. Distinguishable.
- [ε] vs [ab]: z = ε. ε ∉ L. "ab" ∈ L. Distinguishable.

Three equivalence classes, so minimum DFA has 3 states.
GATE Tip: Myhill-Nerode gives the exact minimum number of states. The number of equivalence classes = number of states in minimum DFA. A language is regular iff the Myhill-Nerode relation has finitely many classes. This is an alternative to DFA minimization.

Problem 12: Rice's Theorem Application

Problem: Using Rice's theorem, determine which of the following are decidable:
(a) {<M> | L(M) is finite}
(b) {<M> | L(M) = L(M)} (always true)
(c) {<M> | L(M) contains "hello"}
(d) {<M> | M has exactly 5 states}
Approach: Rice's theorem: any non-trivial property of the language recognized by a TM is undecidable. "Non-trivial" means some TMs have the property and some don't. Properties of the TM itself (not its language) are outside Rice's scope.
Step-by-step Solution:
(a) {<M> | L(M) is finite}:
This is a property of L(M), the language. It's non-trivial (some TMs have finite languages, some don't).
By Rice's theorem: UNDECIDABLE.

(b) {<M> | L(M) = L(M)}:
L(M) = L(M) is always true for any M. This is a TRIVIAL property (all TMs satisfy it).
Rice's theorem doesn't apply to trivial properties. The set of all TM descriptions is decidable (it's everything).
DECIDABLE (trivially, always accept).

(c) {<M> | L(M) contains "hello"}:
Property of L(M): "contains string hello." Non-trivial (some TMs accept "hello," some don't).
By Rice's theorem: UNDECIDABLE.

(d) {<M> | M has exactly 5 states}:
This is a property of the TM description/structure, NOT of its language. Two different TMs can recognize the same language with different numbers of states.
Rice's theorem doesn't apply. We can simply count states in the TM description.
DECIDABLE.
GATE Tip: Rice's theorem applies ONLY to properties of L(M), not to properties of M itself. A property is trivial if either all RE languages have it or no RE language has it. Structural properties of TMs (number of states, transitions) are decidable. Language properties (finite, regular, empty, contains string) are almost always undecidable.

4. Pointers

4.1 Basics

int x = 10;
int *p = &x;     // p stores address of x
printf("%d", *p); // dereference: prints 10
*p = 20;          // x is now 20

4.2 Pointer Arithmetic

int arr[5] = {10, 20, 30, 40, 50};
int *p = arr;       // p points to arr[0]
printf("%d", *(p+2)); // 30 (advances by 2*sizeof(int) bytes)
printf("%d", p[3]);   // 40 (equivalent to *(p+3))

Key Rule: p + n advances the pointer by n * sizeof(*p) bytes, not n bytes.

4.3 Pointer to Pointer

int x = 5;
int *p = &x;
int **pp = &p;
printf("%d", **pp); // prints 5

// Modifying via double pointer
**pp = 100;        // x is now 100

4.4 Function Pointers

int add(int a, int b) { return a + b; }
int sub(int a, int b) { return a - b; }

int (*operation)(int, int);  // declare function pointer
operation = add;
printf("%d", operation(3, 4)); // prints 7
operation = sub;
printf("%d", operation(3, 4)); // prints -1

4.5 Arrays and Pointers

int arr[3] = {1, 2, 3};
// arr is a constant pointer to arr[0]
// arr == &arr[0], *arr == arr[0]
// arr[i] == *(arr + i) == *(i + arr) == i[arr]

int (*p)[3] = &arr; // pointer to entire array of 3 ints
// *p == arr, (*p)[1] == arr[1] == 2

5. Dynamic Memory Allocation

5.1 Functions

FunctionPrototypeDescription
mallocvoid* malloc(size_t size)Allocates size bytes; uninitialized
callocvoid* calloc(size_t n, size_t size)Allocates n*size bytes; zero-initialized
reallocvoid* realloc(void* ptr, size_t size)Resizes previously allocated block
freevoid free(void* ptr)Deallocates memory

5.2 Examples

// Dynamic array of 10 integers
int *arr = (int *)malloc(10 * sizeof(int));
if (arr == NULL) { /* handle error */ }

// Zero-initialized array
int *arr2 = (int *)calloc(10, sizeof(int));

// Resize to 20 integers
arr = (int *)realloc(arr, 20 * sizeof(int));

// Free memory
free(arr);
arr = NULL;  // avoid dangling pointer

// Dynamic 2D array (array of pointers)
int **mat = (int **)malloc(rows * sizeof(int *));
for (int i = 0; i < rows; i++)
    mat[i] = (int *)malloc(cols * sizeof(int));

Common Mistakes:

  • Memory leak: forgetting to call free()
  • Dangling pointer: using pointer after free()
  • Double free: calling free() on already freed memory
  • Buffer overflow: writing beyond allocated size

6. Structures and Unions

6.1 Structures

struct Point {
    int x;
    int y;
};

struct Point p1 = {10, 20};
struct Point *ptr = &p1;
printf("%d, %d", p1.x, ptr->y); // 10, 20

// sizeof(struct) may include padding for alignment
// e.g., struct { char c; int i; } may be 8 bytes, not 5

6.2 Unions

union Data {
    int i;
    float f;
    char c;
};
// sizeof(union Data) = sizeof(largest member) = sizeof(float) = 4
// All members share the same memory location

union Data d;
d.i = 42;
printf("%d", d.i);  // 42
d.f = 3.14;         // overwrites d.i
printf("%d", d.i);  // garbage (reinterpreted bits)

7. Recursion

7.1 Concept and Structure

// Every recursive function needs:
// 1. Base case (termination condition)
// 2. Recursive case (smaller subproblem)
// 3. Progress toward base case

int factorial(int n) {
    if (n <= 1) return 1;          // base case
    return n * factorial(n - 1);     // recursive case
}

7.2 Stack Frames

Each recursive call creates a new stack frame containing local variables, parameters, and the return address. For factorial(4):

flowchart TD F4["factorial(4): 4 * factorial(3)"] --> F3["factorial(3): 3 * factorial(2)"] F3 --> F2["factorial(2): 2 * factorial(1)"] F2 --> F1["factorial(1): returns 1"] F1 --> R2["returns 2"] R2 --> R3["returns 6"] R3 --> R4["returns 24"]

7.3 Tail Recursion

// NOT tail recursive (multiplication after recursive call)
int factorial(int n) {
    if (n <= 1) return 1;
    return n * factorial(n - 1);
}

// Tail recursive (recursive call is the last operation)
int factorial_tail(int n, int acc) {
    if (n <= 1) return acc;
    return factorial_tail(n - 1, n * acc);
}
// Call: factorial_tail(5, 1)
// Compiler can optimize tail recursion into a loop

8.2 Tricky Output Questions

// Post-increment vs Pre-increment
int a = 5;
printf("%d", a++);  // prints 5, then a becomes 6
printf("%d", ++a);  // a becomes 7, prints 7

// sizeof with expressions
int x = 5;
printf("%zu", sizeof(x++)); // prints 4, x is STILL 5
// sizeof does not evaluate the expression

// String and character
char str[] = "Hello";
printf("%zu", sizeof(str));  // 6 (includes '\0')
printf("%zu", strlen(str));  // 5

9. GATE Exam Tips

  • Pointer arithmetic: p + n moves by n * sizeof(*p) bytes.
  • Array name decays to pointer to first element, except with sizeof and &.
  • sizeof is a compile-time operator; it does not evaluate its argument.
  • In C, arrays are passed to functions as pointers (decay). Changes inside affect original.
  • malloc returns uninitialized memory; calloc returns zero-initialized memory.
  • Stack depth for recursion = maximum number of simultaneous active calls.
  • Undefined behavior: modifying a variable more than once between sequence points.

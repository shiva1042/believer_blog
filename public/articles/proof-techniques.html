Mathematical Induction and Proof Techniques

Proof techniques form the backbone of discrete mathematics and are essential for GATE CSE. This article covers the major proof methods from Rosen's Discrete Mathematics and Its Applications (Chapters 1 and 5), with worked examples, GATE-relevant patterns, and common pitfalls. Mastering these techniques is not only necessary for the Engineering Mathematics section but also strengthens your ability to reason about algorithms, data structures, and complexity theory.

1. Direct Proof

A direct proof of the statement P ⇒ Q begins by assuming P is true and then, through a sequence of logical deductions, arrives at Q.

Structure

  1. Assume the hypothesis P is true.
  2. Use definitions, axioms, and previously proven theorems.
  3. Derive the conclusion Q.

Example 1: Sum of Two Even Numbers

Theorem: The sum of two even integers is even.

Proof:

  1. Let a and b be even integers. By definition, there exist integers k and m such that a = 2k and b = 2m.
  2. Then a + b = 2k + 2m = 2(k + m).
  3. Since k + m is an integer, a + b is even by definition. ▮

Example 2: Product of Odd Numbers

Theorem: If a and b are odd integers, then ab is odd.

Proof:

  1. Since a and b are odd, write a = 2k + 1 and b = 2m + 1 for integers k, m.
  2. ab = (2k+1)(2m+1) = 4km + 2k + 2m + 1 = 2(2km + k + m) + 1.
  3. Since 2km + k + m is an integer, ab has the form 2n + 1 and is therefore odd. ▮

Example 3: Divisibility

Theorem: If a | b and b | c, then a | c.

Proof:

  1. By definition, b = a · s and c = b · t for integers s, t.
  2. Substituting: c = (a · s) · t = a · (st).
  3. Since st is an integer, a | c. ▮
When to use: Direct proof works best when the definitions of the hypothesis naturally lead to the conclusion. If the path from P to Q is not obvious, consider contrapositive or contradiction.

2. Proof by Contrapositive

To prove P ⇒ Q, we instead prove the logically equivalent statement ¬Q ⇒ ¬P. This is called the contrapositive.

Why It Works

The conditional P ⇒ Q and its contrapositive ¬Q ⇒ ¬P have identical truth tables. Proving one automatically proves the other.

PQP ⇒ Q¬Q ⇒ ¬P
TTTT
TFFF
FTTT
FFTT

Example 1: Parity Argument

Theorem: If n² is even, then n is even.

Proof (by contrapositive): We prove: if n is odd, then n² is odd.

  1. Suppose n is odd. Then n = 2k + 1 for some integer k.
  2. n² = (2k+1)² = 4k² + 4k + 1 = 2(2k² + 2k) + 1.
  3. This is odd. Since the contrapositive is true, the original statement holds. ▮

Example 2: Divisibility Contrapositive

Theorem: If 3 ∤ n, then 3 ∤ n². (Equivalently: if 3 | n², then 3 | n.)

Proof (by contrapositive): We prove: if 3 ∤ n, then 3 ∤ n².

  1. If 3 ∤ n, then n ≡ 1 (mod 3) or n ≡ 2 (mod 3).
  2. Case 1: n ≡ 1 (mod 3) ⇒ n² ≡ 1 (mod 3). So 3 ∤ n².
  3. Case 2: n ≡ 2 (mod 3) ⇒ n² ≡ 4 ≡ 1 (mod 3). So 3 ∤ n².
  4. In both cases, 3 ∤ n². ▮
When to use: Contrapositive is ideal when the negation of the conclusion (¬Q) gives you a concrete, easy-to-work-with assumption (e.g., "n is odd" is easier to manipulate than "n² is even").

3. Proof by Contradiction

To prove a statement P, assume ¬P is true and derive a logical contradiction. Since the assumption leads to an impossibility, P must be true.

Structure

  1. Assume ¬P.
  2. Through valid deductions, arrive at a statement that is always false (e.g., R ∧ ¬R).
  3. Conclude that the assumption ¬P is false, so P is true.

Example 1: Irrationality of √2 (Classic GATE Favorite)

Theorem: √2 is irrational.

Proof:

  1. Assume for contradiction that √2 is rational. Then √2 = p/q where p, q are integers with no common factor (i.e., the fraction is in lowest terms) and q ≠ 0.
  2. Squaring: 2 = p²/q², so p² = 2q².
  3. This means p² is even. By our earlier result, p must be even. Write p = 2k.
  4. Substituting: (2k)² = 2q² ⇒ 4k² = 2q² ⇒ q² = 2k².
  5. So q² is even, which means q is even.
  6. Contradiction: Both p and q are even, contradicting our assumption that p/q is in lowest terms.
  7. Therefore, √2 is irrational. ▮

Example 2: Infinitude of Primes

Theorem (Euclid): There are infinitely many prime numbers.

Proof:

  1. Assume for contradiction that there are only finitely many primes: p1, p2, …, pn.
  2. Consider the number N = p1 · p2 · … · pn + 1.
  3. N is not divisible by any pi (since dividing by any pi leaves remainder 1).
  4. But by the Fundamental Theorem of Arithmetic, N > 1 must have a prime factor.
  5. This prime factor is not in our list — contradiction.
  6. Therefore, there are infinitely many primes. ▮

Example 3: No Rational Solution

Theorem: There is no rational number r such that r² = 3.

Proof:

  1. Assume r = p/q in lowest terms with r² = 3. Then p² = 3q².
  2. So 3 | p². Since 3 is prime, 3 | p. Write p = 3k.
  3. Then 9k² = 3q², so q² = 3k², meaning 3 | q.
  4. Contradiction: both p and q are divisible by 3, violating lowest terms. ▮
Contradiction vs. Contrapositive: Use contradiction when the statement is not naturally in P ⇒ Q form (e.g., "√2 is irrational" is a standalone assertion). Use contrapositive when you have a clear conditional to negate.

4. Mathematical Induction

Mathematical induction is used to prove statements of the form ∀n ≥ n0, P(n) where n ranges over integers. It is arguably the most frequently tested proof technique in GATE.

4.1 Principle of Mathematical Induction (PMI)

To prove P(n) for all n ≥ n0:

  1. Base Case: Prove P(n0) is true.
  2. Inductive Step: Prove that for an arbitrary k ≥ n0, if P(k) is true (inductive hypothesis), then P(k+1) is true.

By PMI, P(n) holds for all n ≥ n0.

Intuition: Think of induction as a chain of dominoes. The base case knocks over the first domino. The inductive step ensures each domino knocks over the next one.

4.2 Worked Example 1: Summation Formula

Theorem: For all n ≥ 1, 1 + 2 + 3 + … + n = n(n+1)/2.

Proof by induction:

Base case (n = 1): LHS = 1. RHS = 1(2)/2 = 1. True. ✓

Inductive step: Assume P(k) is true, i.e., 1 + 2 + … + k = k(k+1)/2.

We must show P(k+1): 1 + 2 + … + k + (k+1) = (k+1)(k+2)/2.

LHS = [1 + 2 + ... + k] + (k+1)
    = k(k+1)/2 + (k+1)          [by inductive hypothesis]
    = (k+1)[k/2 + 1]
    = (k+1)(k+2)/2
    = RHS  ✓

By PMI, the formula holds for all n ≥ 1. ▮

4.3 Worked Example 2: Sum of Squares

Theorem: For all n ≥ 1, 1² + 2² + … + n² = n(n+1)(2n+1)/6.

Proof by induction:

Base case (n = 1): LHS = 1. RHS = 1 · 2 · 3 / 6 = 1. ✓

Inductive step: Assume ∑i=1k i² = k(k+1)(2k+1)/6.

Show: ∑i=1k+1 i² = (k+1)(k+2)(2k+3)/6.

LHS = k(k+1)(2k+1)/6 + (k+1)²
    = (k+1)/6 · [k(2k+1) + 6(k+1)]
    = (k+1)/6 · [2k² + k + 6k + 6]
    = (k+1)/6 · [2k² + 7k + 6]
    = (k+1)/6 · (k+2)(2k+3)
    = (k+1)(k+2)(2k+3)/6
    = RHS  ✓

By PMI, the result holds for all n ≥ 1. ▮

4.4 Worked Example 3: Divisibility Proof

Theorem: For all n ≥ 1, 6 | (n³ - n) (i.e., n³ - n is divisible by 6).

Proof by induction:

Base case (n = 1): 1³ - 1 = 0 = 6 · 0. ✓

Inductive step: Assume 6 | (k³ - k). Show 6 | ((k+1)³ - (k+1)).

(k+1)³ - (k+1) = k³ + 3k² + 3k + 1 - k - 1
               = (k³ - k) + 3k² + 3k
               = (k³ - k) + 3k(k + 1)

By the inductive hypothesis, 6 | (k³ - k). Now, k(k+1) is the product of two consecutive integers, so one of them is even. Hence 2 | k(k+1), which means 6 | 3k(k+1).

Since 6 divides both terms, 6 | ((k+1)³ - (k+1)). ▮

4.5 Worked Example 4: Inequality Proof

Theorem: For all n ≥ 5, 2n > n².

Proof by induction:

Base case (n = 5): 25 = 32 > 25 = 5². ✓

Inductive step: Assume 2k > k² for some k ≥ 5. Show 2k+1 > (k+1)².

2^(k+1) = 2 · 2^k
        > 2k²             [by inductive hypothesis]

We need: 2k² ≥ (k+1)² = k² + 2k + 1

This requires: k² - 2k - 1 ≥ 0, i.e., k² ≥ 2k + 1.

For k ≥ 5: k² = 25 ≥ 11 = 2(5) + 1. ✓
In general, k² - 2k - 1 = (k-1)² - 2 ≥ 0 for k ≥ 3.

So 2k+1 > 2k² ≥ (k+1)² for k ≥ 5. By PMI, the inequality holds. ▮

4.6 Worked Example 5: Tree Property

Theorem: A full binary tree with n internal nodes has exactly n + 1 leaves.

Proof by induction on the number of internal nodes:

Base case (n = 0): A full binary tree with 0 internal nodes is a single leaf. Leaves = 0 + 1 = 1. ✓

Inductive step: Assume every full binary tree with k internal nodes has k + 1 leaves. Consider a full binary tree T with k + 1 internal nodes.

  1. Find an internal node v whose both children are leaves (such a node always exists in a full binary tree).
  2. Remove both children of v, making v a leaf. Call the resulting tree T'.
  3. T' is a full binary tree with k internal nodes. By the inductive hypothesis, T' has k + 1 leaves.
  4. Going from T' back to T: we lost one leaf (v became internal) and gained two leaves (the children). Net change: +1 leaf.
  5. So T has (k + 1) + 1 = k + 2 leaves, which equals (k+1) + 1. ✓

By PMI, the result holds. ▮

5. Strong Induction

Strong induction (also called complete induction) is a variant where the inductive hypothesis assumes P(n0), P(n0+1), …, P(k) are all true (not just P(k)).

Principle

  1. Base Case: Prove P(n0).
  2. Inductive Step: Assume P(n0), P(n0+1), …, P(k) are all true. Prove P(k+1).

Strong induction and ordinary induction are logically equivalent, but strong induction can make certain proofs much simpler.

Example: Fundamental Theorem of Arithmetic

Theorem: Every integer n ≥ 2 can be written as a product of primes.

Proof by strong induction:

Base case (n = 2): 2 is itself prime. ✓

Inductive step: Assume every integer from 2 to k can be written as a product of primes. Consider k + 1.

  • Case 1: k + 1 is prime. Then it is trivially a product of primes (just itself).
  • Case 2: k + 1 is composite. Then k + 1 = a · b where 2 ≤ a, b ≤ k. By the strong inductive hypothesis, both a and b can be written as products of primes. Therefore, k + 1 = a · b is also a product of primes.

By strong induction, every integer n ≥ 2 is a product of primes. ▮

Example: Fibonacci Inequality

Theorem: For all n ≥ 1, Fn < 2n, where Fn is the n-th Fibonacci number.

Proof by strong induction:

Base cases: F1 = 1 < 2 = 21. F2 = 1 < 4 = 22. ✓

Inductive step: Assume Fj < 2j for all 1 ≤ j ≤ k where k ≥ 2. Show Fk+1 < 2k+1.

F_(k+1) = F_k + F_(k-1)
        < 2^k + 2^(k-1)     [by strong inductive hypothesis]
        < 2^k + 2^k
        = 2^(k+1)  ✓

By strong induction, Fn < 2n for all n ≥ 1. ▮

6. Structural Induction

Structural induction is used to prove properties of recursively defined structures (trees, strings, formulas, etc.). The proof mirrors the recursive definition:

  1. Base Case: Prove the property for the base elements of the recursive definition.
  2. Recursive Step: Assuming the property holds for the sub-structures, prove it for structures built by the recursive rules.

Example: Well-Formed Propositional Formulas

Recursive definition: (1) Every propositional variable p is a formula. (2) If φ is a formula, so is ¬φ. (3) If φ and ψ are formulas, so are (φ ∧ ψ) and (φ ∨ ψ).

Theorem: Every well-formed formula has an equal number of opening and closing parentheses.

Proof by structural induction:

Base case: A variable p has 0 opening and 0 closing parentheses. 0 = 0. ✓

Recursive step:

  • ¬φ: Same parentheses as φ. By hypothesis, equal. ✓
  • (φ ∧ ψ): Opens = opens(φ) + opens(ψ) + 1. Closes = closes(φ) + closes(ψ) + 1. By hypothesis, opens(φ) = closes(φ) and opens(ψ) = closes(ψ). So opens = closes. ✓
  • The ∨ case is identical. ▮

7. The Well-Ordering Principle

Well-Ordering Principle (WOP): Every non-empty subset of the non-negative integers has a least element.

WOP is logically equivalent to the Principle of Mathematical Induction. It provides an alternative approach to induction-style proofs.

Example: Division Algorithm

Theorem: For any integers a and d > 0, there exist unique integers q and r such that a = dq + r and 0 ≤ r < d.

Proof sketch (existence via WOP):

  1. Consider the set S = {a - dk : k ∈ Z, a - dk ≥ 0}.
  2. S is non-empty (choose a sufficiently small k).
  3. By WOP, S has a least element r = a - dq.
  4. We have r ≥ 0. If r ≥ d, then r - d = a - d(q+1) ≥ 0 is in S and is smaller than r — contradiction.
  5. Therefore 0 ≤ r < d. ▮

8. Common GATE Patterns

Here are the types of problems that appear most frequently in GATE:

Pattern 1: Proving Summation Formulas

Given a formula like ∑ i = n(n+1)/2 or ∑ i² = n(n+1)(2n+1)/6, prove it by induction. GATE may ask you to identify the inductive step or point out errors in a given proof.

Pattern 2: Divisibility

"Prove that n³ - n is divisible by 6" or "Prove 7n - 1 is divisible by 6." The key technique: expand using binomial theorem or factor, then use the inductive hypothesis.

Pattern 3: Graph and Tree Properties

Common statements to prove by induction include:

  • A tree with n vertices has n - 1 edges.
  • A connected planar graph satisfies V - E + F = 2 (Euler's formula).
  • A full binary tree with n internal nodes has n + 1 leaves.
  • The sum of degrees in a graph equals 2|E| (Handshaking Lemma).

Pattern 4: Recurrence Relations

Prove that a given closed-form solution satisfies a recurrence. For example, prove T(n) = 2T(n/2) + n has solution T(n) = O(n log n) by strong induction.

9. Comparison of Proof Techniques

Technique When to Use Structure Key Advantage
Direct Proof Definitions naturally lead to conclusion Assume P, derive Q Most straightforward; easiest to follow
Contrapositive ¬Q provides a useful concrete assumption Assume ¬Q, derive ¬P Often simpler when direct path is unclear
Contradiction Statement is not conditional; need to show non-existence or uniqueness Assume ¬P, derive a contradiction Works for existential / irrationality proofs
Mathematical Induction Statement is ∀n ≥ n0, P(n) Base + P(k) ⇒ P(k+1) Natural for formulas over integers
Strong Induction P(k+1) depends on multiple earlier values Base + P(n0)…P(k) ⇒ P(k+1) Handles recurrences, prime factorization
Structural Induction Property of recursively defined objects Base elements + recursive constructors Natural for trees, strings, formulas
Well-Ordering Existence of a minimal counterexample Assume counterexample set is non-empty; find least Alternative to induction; useful for existence proofs

10. Common Mistakes and Pitfalls

Pitfall 1: Forgetting the Base Case. A "proof" by induction with no base case can "prove" false statements. Example: "All horses are the same color" uses a faulty induction argument that fails at n = 2 because the sets do not overlap.
Pitfall 2: Not Using the Inductive Hypothesis. If your inductive step does not reference P(k), you are likely doing a direct proof, not induction, and the argument may have gaps.
Pitfall 3: Wrong Base Case Index. If the theorem says "for all n ≥ 5," you must use n = 5 as the base case, not n = 1. The inductive step also only needs to work for k ≥ 5.
Pitfall 4: Circular Reasoning in Contradiction Proofs. Make sure the contradiction genuinely arises from the assumption ¬P, not from an algebraic error or an unjustified step.
Pitfall 5: Confusing Contradiction and Contrapositive. In a contrapositive proof, you prove ¬Q ⇒ ¬P. In a contradiction proof, you assume ¬P (or P ∧ ¬Q) and derive any contradiction. They are different techniques.
Pitfall 6: Strong Induction Base Cases. When using strong induction, you may need multiple base cases. For instance, if the recurrence is P(k+1) depends on P(k) and P(k-1), you need base cases for both n0 and n0 + 1.
Pitfall 7: Assuming What You Want to Prove. In the inductive step, you assume P(k) and prove P(k+1). Do NOT assume P(k+1). A common student error is to start with the equation you want to prove and manipulate both sides simultaneously.

11. Additional Important Results

Geometric Series

Theorem: For r ≠ 1 and n ≥ 0, ∑i=0n ri = (rn+1 - 1) / (r - 1).

This is proved by induction on n and is extremely useful for algorithm analysis.

Power Set Cardinality

Theorem: If |S| = n, then |P(S)| = 2n.

Proof by induction on n:

Base case (n = 0): S = ∅, P(S) = {∅}, |P(S)| = 1 = 20. ✓

Inductive step: Let |S| = k + 1. Pick any element a ∈ S. Then S' = S \ {a} has k elements.

Every subset of S either contains a or does not:

  • Subsets not containing a: these are exactly the subsets of S'. By hypothesis, there are 2k.
  • Subsets containing a: each is formed by adding a to a subset of S'. There are 2k of these too.

Total: 2k + 2k = 2k+1. ▮

Handshaking Lemma

Theorem: In any graph G = (V, E), ∑v ∈ V deg(v) = 2|E|.

Proof by induction on |E|:

Base case (|E| = 0): All degrees are 0. Sum = 0 = 2(0). ✓

Inductive step: Assume the result holds for graphs with k edges. Consider a graph with k + 1 edges. Remove one edge {u, v}. By hypothesis, the remaining graph satisfies the formula. Adding {u, v} back increases deg(u) by 1, deg(v) by 1, and |E| by 1. So the sum increases by 2 and 2|E| increases by 2. ▮

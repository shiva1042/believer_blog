Propositional and First Order Logic

Section 1: Engineering Mathematics

1. Propositions and Truth Values

A proposition is a declarative statement that is either true (T) or false (F), but not both. Propositions are the fundamental building blocks of propositional logic.

Examples of propositions:
  • "The earth revolves around the sun." — True
  • "2 + 3 = 6" — False
  • "Every prime number greater than 2 is odd." — True
Non-propositions:
  • "What time is it?" (question)
  • "Close the door." (command)
  • "x + 1 = 5" (contains a free variable)

2. Logical Connectives

Logical connectives combine simple propositions into compound propositions. Let p and q be propositions:

Key insight for Implication (p → q): The only case where an implication is false is when the hypothesis p is true but the conclusion q is false. "A false premise implies anything" — this is called vacuous truth.

4. Logical Equivalences

Two propositions are logically equivalent (denoted ≡) if they have identical truth values in every row of their truth tables.

4.1 Important Equivalences

Distributive (OR over AND)p ∨ (q ∧ r) ≡ (p ∨ q) ∧ (p ∨ r)

4.2 Tautology, Contradiction, Contingency

  • Tautology: A proposition that is always true. Example: p ∨ ¬p
  • Contradiction: A proposition that is always false. Example: p ∧ ¬p
  • Contingency: A proposition that is neither a tautology nor a contradiction.

6. First Order Logic (Predicate Logic)

First order logic extends propositional logic with predicates, variables, and quantifiers, enabling us to reason about objects and their properties.

6.1 Predicates

A predicate is a function that maps objects to truth values. It becomes a proposition when its variables are bound.

Examples:
  • P(x): "x is a prime number" — P(5) is True, P(4) is False
  • G(x, y): "x is greater than y" — G(7, 3) is True
  • E(x): "x is even" — E(2) is True, E(3) is False

6.2 Quantifiers

QuantifierSymbolMeaningExample
Universal∀xFor all x∀x P(x): "Every x satisfies P"
Existential∃xThere exists an x∃x P(x): "Some x satisfies P"

6.3 Negation of Quantifiers

¬(∀x P(x)) ≡ ∃x ¬P(x)
¬(∃x P(x)) ≡ ∀x ¬P(x)

In words: "Not all x satisfy P" means "there exists an x that does not satisfy P," and "there does not exist an x satisfying P" means "for all x, P is false."

6.4 Nested Quantifiers

Multiple quantifiers can be nested. The order of quantifiers matters when mixing ∀ and ∃.

StatementMeaningExample (Domain: Integers)
∀x ∀y P(x,y)For every x and every y, P holds∀x ∀y (x+y = y+x) — True
∃x ∃y P(x,y)There exist some x and y such that P holds∃x ∃y (x+y = 0) — True
∀x ∃y P(x,y)For every x, there exists some y such that P holds∀x ∃y (x+y = 0) — True (y = -x)
∃x ∀y P(x,y)There exists an x such that for all y, P holds∃x ∀y (x+y = 0) — False
GATE-style Example:
Let the domain be real numbers. Consider: ∀x ∃y (y > x).
Translation: For every real number x, there exists a real number y greater than x.
Truth value: True — for any x, we can choose y = x + 1.

Negation: ∃x ∀y (y ≤ x) — "There exists a real number x such that all real numbers y are at most x." This is False (no largest real number exists).

6.5 Prenex Normal Form

A formula is in prenex normal form if it is written as a string of quantifiers (the prefix) followed by a quantifier-free formula (the matrix).

Q1x1 Q2x2 ... Qnxn M(x1, x2, ..., xn)

where each Qi is either ∀ or ∃, and M is the matrix containing no quantifiers.

8. Summary of Key Concepts for GATE

  • Master truth tables for all connectives, especially implication.
  • Know De Morgan's laws and contrapositive — these appear frequently.
  • Understand that ∀x ∃y P(x,y) and ∃y ∀x P(x,y) are different.
  • Practice negating quantified statements.
  • Be comfortable converting between implication and its equivalent forms.
  • Know inference rules, especially Modus Ponens and Resolution.

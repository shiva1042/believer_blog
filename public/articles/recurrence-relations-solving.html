Solving Recurrence Relations

⚡ Algorithms

1. Why Recurrences Matter

Recurrence relations describe the running time of recursive algorithms. Solving them gives us the closed-form time complexity. This is one of the most important topics in GATE Algorithms.

2. Substitution Method

Guess the solution, then prove it correct using mathematical induction.

Example 1: T(n) = 2T(n/2) + n

Guess: T(n) = O(n log n). Prove T(n) <= c * n * log(n).

Assume T(k) <= c*k*log(k) for all k < n.

T(n) = 2T(n/2) + n
     <= 2 * c * (n/2) * log(n/2) + n
     = c * n * (log n - 1) + n
     = c * n * log n - c*n + n
     <= c * n * log n   (when c >= 1)

The guess holds. T(n) = O(n log n).

Example 2: T(n) = T(n-1) + n

Guess: T(n) = O(n^2). Prove T(n) <= c*n^2.

T(n) = T(n-1) + n
     <= c*(n-1)^2 + n
     = c*n^2 - 2cn + c + n
     = c*n^2 - (2c-1)*n + c
     <= c*n^2   (when c >= 1 and n >= 1)

T(n) = O(n^2).
Tip: The hardest part is guessing correctly. Use the recursion tree method to make an informed guess, then verify with substitution.

3. Recursion Tree Method

Draw the call tree. At each level, compute the total work done. Sum across all levels.

Example: T(n) = 2T(n/2) + n

graph TD A["n (cost: n)"] --> B["n/2 (cost: n/2)"] A --> C["n/2 (cost: n/2)"] B --> D["n/4 (cost: n/4)"] B --> E["n/4 (cost: n/4)"] C --> F["n/4 (cost: n/4)"] C --> G["n/4 (cost: n/4)"] 
Level 0: 1 node, cost = n             → total = n
Level 1: 2 nodes, cost = n/2 each     → total = n
Level 2: 4 nodes, cost = n/4 each     → total = n
...
Level k: 2^k nodes, cost = n/2^k each → total = n

Number of levels: log2(n) (tree ends when n/2^k = 1)

Total = n * log2(n) = O(n log n)

Example: T(n) = 3T(n/4) + n^2

graph TD A2["n^2"] --> B2["(n/4)^2"] A2 --> C2["(n/4)^2"] A2 --> D2["(n/4)^2"] 
Level 0: cost = n^2
Level 1: 3 nodes, cost = (n/4)^2 each → 3n^2/16
Level 2: 9 nodes, cost = (n/16)^2 each → 9n^2/256
Level k: 3^k * (n/4^k)^2 = n^2 * (3/16)^k

Geometric series with ratio 3/16 < 1.
Sum = n^2 * 1/(1 - 3/16) = n^2 * 16/13 = O(n^2)

4. Master Theorem

For recurrences of the form: T(n) = aT(n/b) + O(n^c * (log n)^k) where a ≥ 1, b > 1.

Let p = log_b(a) (critical exponent).

Standard Master Theorem (3 Cases)

CaseConditionResult
Case 1c < p (work dominated by leaves)T(n) = O(n^p)
Case 2c = p and k = 0T(n) = O(n^c * log n)
Case 2 (extended)c = p and k ≥ 0T(n) = O(n^c * (log n)^(k+1))
Case 3c > p (work dominated by root)T(n) = O(n^c)

Master Theorem Examples

Example 1: T(n) = 2T(n/2) + n

a=2, b=2, c=1. p = log2(2) = 1.
c = p → Case 2. T(n) = O(n log n).

Example 2: T(n) = 4T(n/2) + n

a=4, b=2, c=1. p = log2(4) = 2.
c < p → Case 1. T(n) = O(n^2).

Example 3: T(n) = 2T(n/2) + n^2

a=2, b=2, c=2. p = log2(2) = 1.
c > p → Case 3. T(n) = O(n^2).

Example 4: T(n) = 3T(n/4) + n log n

a=3, b=4, c=1 (the n part), k=1 (log factor).
p = log4(3) ≈ 0.793.
c=1 > p → Case 3. T(n) = O(n log n).

Example 5: T(n) = 2T(n/4) + sqrt(n)

a=2, b=4, c=0.5. p = log4(2) = 0.5.
c = p → Case 2. T(n) = O(sqrt(n) * log n).

When Master Theorem Does NOT Apply

  • T(n) = T(n-1) + n (subtractive, not divide)
  • T(n) = T(n/2) + T(n/3) + n (unequal subproblems)
  • T(n) = 2^n * T(n/2) + n (a is not constant)

5. Extended Master Theorem

Handles the case where f(n) = n^c (log n)^k with c = log_b(a):

ConditionResult
c = p, k > -1T(n) = O(n^p (log n)^(k+1))
c = p, k = -1T(n) = O(n^p log(log n))
c = p, k < -1T(n) = O(n^p)

6. Akra-Bazzi Method (Brief)

For recurrences of the form: T(n) = Σ a_i * T(n/b_i) + g(n) with possibly unequal subproblems.

  1. Find p such that Σ a_i / b_i^p = 1.
  2. T(n) = Θ(n^p * (1 + integral from 1 to n of g(u)/u^(p+1) du)).

This is rarely tested directly in GATE but useful for non-standard recurrences.

8. Practice Problems

P1: Solve T(n) = 9T(n/3) + n.

a=9, b=3, c=1. p = log3(9) = 2.
c=1 < p=2 → Case 1. T(n) = O(n^2).

P2: Solve T(n) = T(2n/3) + 1.

a=1, b=3/2, c=0. p = log(3/2)(1) = 0.
c=0 = p → Case 2. T(n) = O(log n).

P3: Solve T(n) = 3T(n/9) + sqrt(n).

a=3, b=9, c=0.5. p = log9(3) = 0.5.
c = p → Case 2. T(n) = O(sqrt(n) * log n).

P4: Solve T(n) = T(n-1) + n^2.

Not divide-and-conquer. Expand:
T(n) = n^2 + (n-1)^2 + ... + 1^2 = n(n+1)(2n+1)/6 = O(n^3).

P5: Solve T(n) = 4T(n/2) + n^2.

a=4, b=2, c=2. p = log2(4) = 2.
c = p → Case 2. T(n) = O(n^2 log n).

P6: Solve T(n) = 2T(n/4) + 1.

a=2, b=4, c=0. p = log4(2) = 0.5.
c=0 < p=0.5 → Case 1. T(n) = O(sqrt(n)).

P7: Solve T(n) = T(n/2) + n log n.

a=1, b=2, c=1 (with k=1). p = log2(1) = 0.
c=1 > p=0 → Case 3. T(n) = O(n log n).

P8: Solve T(n) = 8T(n/2) + n^2.

a=8, b=2, c=2. p = log2(8) = 3.
c=2 < p=3 → Case 1. T(n) = O(n^3).

P9: Solve T(n) = 2T(n-1) + 1, T(0) = 1.

Expand: T(n) = 2^n + 2^(n-1) + ... + 2 + 1 = 2^(n+1) - 1 = O(2^n).

P10: Solve T(n) = T(n/3) + T(2n/3) + n.

Recursion tree: every level sums to n.
Depth = log(3/2)(n) (longest path is the 2n/3 branch).
T(n) = O(n log n).
GATE Strategy: Try Master Theorem first. If it does not apply (subtractive recurrence or unequal splits), use the recursion tree method. Save substitution for verification.

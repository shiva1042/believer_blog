Regular Languages and Finite Automata

🔄 Section 6: Theory of Computation

1. Basic Definitions

1.1 Alphabets, Strings, and Languages

  • Alphabet (Sigma): A finite, non-empty set of symbols. Example: Sigma = {0, 1}.
  • String: A finite sequence of symbols from an alphabet. The empty string is denoted by epsilon.
  • Language: A set of strings over an alphabet. Can be finite or infinite.
  • Sigma*: The set of all strings over Sigma, including epsilon.
  • Sigma+: Sigma* minus epsilon (all non-empty strings).

2. Deterministic Finite Automaton (DFA)

2.1 Formal Definition

A DFA is a 5-tuple M = (Q, Sigma, delta, q0, F) where:

  • Q: Finite set of states
  • Sigma: Input alphabet
  • delta: Transition function, delta: Q x Sigma -> Q
  • q0: Start state (q0 in Q)
  • F: Set of accept (final) states (F subset of Q)

2.2 DFA Example: Strings ending in "01"

Language: L = {w in {0,1}* | w ends with 01}

stateDiagram-v2 direction LR [*] --> q0 q0 --> q0: 1 q0 --> q1: 0 q1 --> q1: 0 q1 --> q2: 1 q2 --> q0: 1 q2 --> q1: 0 q2 --> [*]
StateInput 0Input 1Description

2.3 DFA Properties

  • Exactly one transition for each symbol from each state (deterministic).
  • No epsilon transitions.
  • A DFA with n states can have at most 2^n distinct languages it distinguishes.
  • Minimum DFA is unique (up to isomorphism) for any regular language.

3. Non-deterministic Finite Automaton (NFA)

3.1 Formal Definition

An NFA is a 5-tuple M = (Q, Sigma, delta, q0, F) where delta: Q x Sigma -> P(Q) (power set of Q). Multiple transitions from a state on the same input are allowed, including transitions to no state.

3.2 NFA Example: Strings containing "101"

stateDiagram-v2 direction LR [*] --> q0 q0 --> q0: 0,1 q0 --> q1: 1 q1 --> q2: 0 q2 --> q3: 1 q3 --> q3: 0,1 q3 --> [*]

The NFA non-deterministically "guesses" when the substring 101 begins. A string is accepted if any computational path leads to an accept state.

3.3 NFA to DFA Conversion (Subset Construction)

Every NFA can be converted to an equivalent DFA using the subset construction algorithm:

  1. Each DFA state corresponds to a subset of NFA states.
  2. Start state of DFA = epsilon-closure of NFA start state.
  3. For each DFA state S and input symbol a: compute the set of NFA states reachable from any state in S on input a (including epsilon-closure).
  4. A DFA state is accepting if it contains at least one NFA accept state.
Worst case: An NFA with n states can require a DFA with up to 2^n states. This exponential blowup is sometimes unavoidable.

4. Epsilon-NFA

An epsilon-NFA allows transitions on epsilon (empty string) in addition to input symbols. delta: Q x (Sigma union {epsilon}) -> P(Q).

Epsilon-closure: The set of states reachable from a state following only epsilon transitions. Used in NFA-to-DFA conversion.

Equivalence: epsilon-NFA = NFA = DFA in terms of the languages they recognize (all recognize exactly the regular languages).

5.3 Equivalence with Finite Automata

Kleene's Theorem: A language is regular if and only if it can be described by a regular expression. The conversions are:

  • RE to NFA: Thompson's construction
  • NFA to DFA: Subset construction
  • DFA to RE: State elimination method

6. Pumping Lemma for Regular Languages

Used to prove a language is NOT regular.

Statement

If L is regular, then there exists a pumping length p such that for every string s in L with |s| >= p, s can be written as s = xyz where:

  1. |y| > 0 (y is not empty)
  2. |xy| <= p
  3. For all i >= 0, xy^i z is in L

Usage (Proof by Contradiction)

To prove L is not regular:
1. Assume L is regular (for contradiction).
2. Let p be the pumping length (exists by lemma).
3. Choose a string s in L with |s| >= p.
4. For ALL possible decompositions s = xyz satisfying conditions 1 and 2:
   Show there exists some i such that xy^i z is NOT in L.
5. Contradiction. L is not regular.

Example: L = {0^n 1^n | n >= 0} is not regular

Choose s = 0^p 1^p. Since |xy| <= p, y = 0^k for some k > 0. Pumping: xy^2 z = 0^(p+k) 1^p, which has more 0s than 1s, so it is not in L. Contradiction.

GATE Tip: The pumping lemma can only prove a language is NOT regular. It cannot prove a language IS regular. To show regularity, construct a DFA, NFA, or regular expression.

8. Myhill-Nerode Theorem

A language L is regular if and only if it has a finite number of equivalence classes under the relation: x and y are equivalent if for all strings z, xz in L iff yz in L.

The number of equivalence classes equals the number of states in the minimum DFA for L.

9. DFA Minimization

Given a DFA, find the equivalent DFA with the minimum number of states:

  1. Remove unreachable states.
  2. Use table-filling algorithm (or Hopcroft's algorithm) to identify equivalent states.
  3. Merge equivalent states.

Time: O(n2) for table-filling, O(n log n) for Hopcroft's.

10. Key GATE Takeaways

  • DFA, NFA, epsilon-NFA, and regular expressions all define the same class: regular languages.
  • NFA to DFA conversion may cause exponential state blowup (n states to 2^n).
  • Pumping lemma is the primary tool for proving languages are non-regular.
  • Regular languages are closed under union, intersection, complement, concatenation, and Kleene star.
  • Minimum DFA is unique and can be found using state equivalence.
  • Common non-regular languages: {0^n 1^n}, {ww^R}, languages requiring counting.

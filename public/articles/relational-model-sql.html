1. Relational Algebra

Relational algebra is a procedural query language that operates on relations (tables) and produces relations as output. It forms the theoretical foundation of SQL.

2. Tuple Relational Calculus (TRC)

TRC is a non-procedural (declarative) query language. Queries are of the form: { t | P(t) } meaning "the set of all tuples t such that predicate P(t) is true."

Example: Find all students with age > 20:
{ t | Student(t) ∧ t.age > 20 }

Example: Find names of students in the CS department:
{ t.name | Student(t) ∧ t.dept = 'CS' }
Expressive Power: Relational algebra, safe tuple relational calculus, and safe domain relational calculus are all equivalent in expressive power (Codd's theorem).

3. SQL Overview

SQL (Structured Query Language) is based on relational algebra and tuple relational calculus. It is divided into sublanguages:

FROM Student S

4.3 Aggregate Functions and GROUP BY

SELECT dept, COUNT(*) AS num_students, AVG(gpa) AS avg_gpa
FROM Student
GROUP BY dept
HAVING COUNT(*) > 10
ORDER BY avg_gpa DESC;
FunctionDescriptionIgnores NULL?
COUNT(*)Count all rows (including NULLs)No
COUNT(col)Count non-NULL values in columnYes

Execution Order: FROM → WHERE → GROUP BY → HAVING → SELECT → ORDER BY. The WHERE clause filters rows before grouping; HAVING filters groups after aggregation.

4.4 Subqueries

-- Correlated subquery: students with GPA above their department average
SELECT S.Name, S.GPA
FROM Student S
WHERE S.GPA > (
    SELECT AVG(S2.GPA)
    FROM Student S2
    WHERE S2.Dept = S.Dept
);

-- EXISTS subquery: departments that have at least one student
SELECT D.DeptName
FROM Department D
WHERE EXISTS (
    SELECT 1 FROM Student S WHERE S.Dept = D.DeptID
);

-- IN subquery: students enrolled in 'Database Systems'
SELECT Name FROM Student
WHERE StudentID IN (
    SELECT StudentID FROM Enrollment
    WHERE CourseID = (SELECT CourseID FROM Course WHERE Title = 'Database Systems')
);

5. Views and Assertions

5.1 Views

A view is a virtual table defined by a query. It does not store data physically but is computed each time it is referenced.

CREATE VIEW CS_Students AS
SELECT StudentID, Name, GPA
FROM Student
WHERE Dept = 'CS';

-- Query the view like a table
SELECT * FROM CS_Students WHERE GPA > 3.5;
View PropertyDescription
Updatable viewMust be defined on a single table, no aggregates, no GROUP BY, no DISTINCT, includes primary key
Materialized viewPhysically stores the result; must be refreshed when base tables change

5.2 Assertions

An assertion is a declarative constraint on the database that must always be true. The DBMS checks the assertion whenever relevant data is modified.

CREATE ASSERTION salary_check
CHECK (NOT EXISTS (
    SELECT * FROM Employee WHERE Salary < 0
));

6. Summary and GATE Key Points

Key Points for GATE:
  • The five fundamental RA operations: σ, π, ∪, −, ×. All others can be derived.
  • Natural join automatically matches on common attribute names and removes duplicate columns.
  • Division (R ÷ S) finds tuples in R associated with ALL tuples in S.
  • σ and π are unary; all others are binary operations.
  • Project (π) eliminates duplicates; SQL SELECT does not (use DISTINCT).
  • NULL comparisons: any comparison with NULL yields UNKNOWN. Use IS NULL / IS NOT NULL.
  • COUNT(*) counts all rows; COUNT(column) counts non-NULL values.
  • WHERE filters before grouping; HAVING filters after grouping.
  • A correlated subquery is re-evaluated for each row of the outer query.
  • Maximum tuples in R ⨝ S (natural join) = |R| × |S|; minimum = 0.

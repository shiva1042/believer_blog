5. Complexity Classes

  • P: Problems solvable in polynomial time by a deterministic TM.
  • NP: Problems verifiable in polynomial time. P ⊆ NP.
  • NP-Complete: In NP + every NP problem reduces to it. If any NPC problem is in P, then P = NP.
  • NP-Hard: At least as hard as NPC, but not necessarily in NP.
  • Classic NPC problems: SAT, 3-SAT, Vertex Cover, Hamiltonian Cycle, TSP (decision), Graph Coloring, Subset Sum, Clique.

6. Key Algorithm Facts

  • Stable sorts: Bubble, Insertion, Merge, Counting, Radix.
  • In-place sorts: Bubble, Selection, Insertion, Quick, Heap.
  • Quicksort worst case (O(n^2)) when already sorted and pivot = first element.
  • Activity selection and fractional knapsack: greedy works. 0/1 knapsack: needs DP.
  • Longest Common Subsequence: O(mn) DP. Longest Increasing Subsequence: O(n log n).
  • Matrix Chain Multiplication: O(n^3) DP.

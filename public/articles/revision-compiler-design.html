2. FIRST and FOLLOW Computation

FIRST(X) Rules

  1. If X is terminal: FIRST(X) = {X}.
  2. If X → ε: add ε to FIRST(X).
  3. If X → Y1Y2...Yk: add FIRST(Y1) - {ε} to FIRST(X). If ε in FIRST(Y1), add FIRST(Y2) - {ε}, and so on. If all Yi can derive ε, add ε.

FOLLOW(A) Rules

  1. FOLLOW(Start) includes $.
  2. If A → αBβ: add FIRST(β) - {ε} to FOLLOW(B).
  3. If A → αB or A → αBβ where ε in FIRST(β): add FOLLOW(A) to FOLLOW(B).
Remember: ε is NEVER in any FOLLOW set. FIRST can contain ε, FOLLOW cannot.

Power hierarchy: LL(1) ⊂ LR(0) ⊂ SLR(1) ⊂ LALR(1) ⊂ CLR(1)

LL(1) Grammar Conditions

  • No left recursion.
  • Must be left-factored.
  • For A → α | β: FIRST(α) ∩ FIRST(β) = {}.
  • If ε in FIRST(α), then FIRST(β) ∩ FOLLOW(A) = {}.

LR Conflicts

  • Shift-Reduce conflict: Parser cannot decide whether to shift or reduce.
  • Reduce-Reduce conflict: Parser cannot decide which production to reduce by.
  • SLR resolves using FOLLOW. LALR and CLR use more precise lookaheads.

5. Optimization Techniques

Machine-Independent Optimizations

TechniqueDescription
Constant foldingEvaluate constant expressions at compile time: x = 2+3 → x = 5
Constant propagationReplace variable with its constant value
Common subexpression eliminationAvoid recomputing same expression: t1=a+b; t2=a+b → t2=t1
Dead code eliminationRemove code whose results are never used
Copy propagationAfter x=y, replace uses of x with y
Loop invariant code motionMove computations that don't change inside a loop to outside
Strength reductionReplace expensive ops with cheaper ones: x*2 → x<<1
Induction variable eliminationSimplify loop counter arithmetic

Machine-Dependent Optimizations

  • Register allocation (graph coloring)
  • Instruction scheduling
  • Peephole optimization (pattern matching on small instruction sequences)

6. Syntax-Directed Translation

  • Synthesized attributes: computed from children. S-attributed definition uses only synthesized.
  • Inherited attributes: computed from parent/siblings. L-attributed allows both (left-to-right evaluation).
  • S-attributed ⊂ L-attributed. S-attributed can be evaluated in bottom-up parse.
  • L-attributed can be evaluated in a single left-to-right depth-first traversal.

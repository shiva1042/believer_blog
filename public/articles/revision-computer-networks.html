3. TCP vs UDP

FeatureTCPUDP

ReliabilityReliable (ACKs, retransmission)Unreliable (best effort)
OrderingGuaranteed (sequence numbers)No ordering

Header size20 bytes (min)8 bytes
SpeedSlower (overhead)Faster (no overhead)
Use casesHTTP, FTP, SMTP, SSHDNS, DHCP, VoIP, video streaming

TCP 3-Way Handshake

  • SYN → SYN-ACK → ACK
  • Connection termination: FIN → ACK, FIN → ACK (4-way)

TCP Congestion Control

  • Slow Start: cwnd starts at 1 MSS, doubles each RTT (exponential growth).
  • Congestion Avoidance: After threshold, cwnd increases by 1 MSS per RTT (linear/additive increase).
  • On timeout: ssthresh = cwnd/2, cwnd = 1 MSS (Tahoe). Or cwnd = ssthresh (Reno, on 3 dup ACKs).
  • Fast Retransmit: Retransmit on 3 duplicate ACKs without waiting for timeout.

5. Sliding Window & Flow Control

ProtocolWindow SizeSequence Numbers Needed

Efficiency Formulas

  • a = T_prop / T_trans
  • Stop-and-Wait: Efficiency = 1/(1 + 2a)
  • Sliding Window: Efficiency = min(1, W/(1 + 2a))
  • T_trans = L/B (packet length / bandwidth)
  • T_prop = d/v (distance / propagation speed)

7. Routing Protocols

ProtocolTypeAlgorithmMetric
RIPDistance VectorBellman-FordHop count (max 15)
OSPFLink StateDijkstraCost (bandwidth-based)
BGPPath VectorPolicy-basedAS path, policies
  • Distance Vector: Each router shares routing table with neighbors. Count-to-infinity problem.
  • Link State: Each router floods link-state info to all. Uses Dijkstra. No count-to-infinity.
  • RIP: Intra-domain. OSPF: Intra-domain. BGP: Inter-domain (between Autonomous Systems).

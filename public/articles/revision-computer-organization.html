1. Addressing Modes

ModeEffective AddressExample
ImmediateOperand in instruction itselfADD R1, #5
Direct (Absolute)EA = Address fieldLOAD R1, 1000
IndirectEA = M[Address field]LOAD R1, @1000
RegisterEA = RegisterADD R1, R2
Register IndirectEA = M[Register]LOAD R1, (R2)
IndexedEA = Index Register + AddressLOAD R1, 100(R2)
Base RegisterEA = Base Register + OffsetLOAD R1, 50(R3)
PC-RelativeEA = PC + OffsetBEQ label
Auto-incrementEA = (R); R = R + dLOAD R1, (R2)+
Auto-decrementR = R - d; EA = (R)LOAD R1, -(R2)
Remember: Immediate = fastest (no memory access for operand). Indirect = slowest (extra memory access to get the actual address).

2. Pipeline Formulas

FormulaExpression
Execution time (no pipeline)n * k * t (n instr, k stages, t = time per stage)
Execution time (pipeline)(k + n - 1) * t
Speedupn*k / (k + n - 1) → approaches k for large n
Throughputn / [(k + n - 1) * t] → approaches 1/t for large n
With stallsTime = (k + n - 1 + stall_cycles) * t
CPI with stallsIdeal CPI + stall cycles per instruction

Key Tree Facts

  • Binary tree with n nodes has n+1 NULL pointers.
  • Max nodes at level i = 2^i (root = level 0).
  • Max nodes in tree of height h = 2^(h+1) - 1.
  • Min height for n nodes = floor(log2(n)).
  • Number of binary trees with n nodes = Catalan(n) = C(2n,n)/(n+1).
  • Inorder + Preorder (or Postorder) uniquely determine a binary tree.

3. Heap Properties

  • Max Heap: parent ≥ children. Min Heap: parent ≤ children.
  • A heap is a complete binary tree stored as an array.
  • Parent of i: floor((i-1)/2). Left child: 2i+1. Right child: 2i+2 (0-indexed).
  • Build heap: O(n). Insert: O(log n). Extract min/max: O(log n).
  • Heap sort: O(n log n), in-place, not stable.
Remember: A heap is NOT a sorted array. Only the root is guaranteed min (or max). Siblings have no ordering relationship.

5. Hashing

  • Division method: h(k) = k mod m. Choose m as prime not close to power of 2.
  • Multiplication method: h(k) = floor(m * (k*A mod 1)). A is irrational constant.
  • Collision resolution: Chaining (linked lists) or Open Addressing (linear/quadratic probing, double hashing).
  • Load factor α: n/m (items/table size). For chaining: avg search = 1 + α. For open addressing: 1/(1-α).
  • Rehash when α exceeds threshold (typically 0.7-0.75).

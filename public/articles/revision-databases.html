1. Normal Forms Quick Identification

Normal FormConditionQuick Test
1NFAll attributes are atomic (no multi-valued or composite)No repeating groups, no sets in cells
2NF1NF + no partial dependency (non-prime attr depends on part of candidate key)If single-attribute candidate key, already 2NF
3NF2NF + no transitive dependency (non-prime depends on non-prime)For every FD X→A: X is superkey OR A is prime attribute
BCNFFor every FD X→Y, X must be a superkeyStricter than 3NF: no exception for prime attributes

Key Differences

  • 3NF vs BCNF: 3NF allows X→A where A is a prime attribute even if X is not a superkey. BCNF does not allow this.
  • BCNF decomposition: always lossless but may not preserve all FDs.
  • 3NF decomposition: lossless + dependency preserving always possible.
  • Finding candidate keys: Attributes that never appear on the RHS of any FD must be part of every candidate key. Start with these and compute closure.

5. Concurrency Control

  • Conflict serializability: Check using precedence graph. If no cycle, schedule is conflict serializable.
  • Two-Phase Locking (2PL): Growing phase (acquire locks) + Shrinking phase (release locks). Guarantees conflict serializability.
  • Strict 2PL: Hold all exclusive locks until commit. Prevents cascading rollback.
  • Timestamp ordering: Each transaction gets a timestamp. If T_i wants to read/write X and timestamp is wrong order, abort and restart.

7. Functional Dependency Rules (Armstrong's Axioms)

  • Reflexivity: If Y ⊆ X, then X → Y.
  • Augmentation: If X → Y, then XZ → YZ.
  • Transitivity: If X → Y and Y → Z, then X → Z.
  • Union: If X → Y and X → Z, then X → YZ.
  • Decomposition: If X → YZ, then X → Y and X → Z.
  • Pseudo-transitivity: If X → Y and WY → Z, then WX → Z.

8. Lossless Join & Dependency Preservation

  • Lossless join test (2 relations): R1 ∩ R2 → R1-R2 or R1 ∩ R2 → R2-R1 must hold.
  • Dependency preserving: F+ = (F1 ∪ F2 ∪ ... ∪ Fn)+ where Fi are FDs local to each Ri.
Most Tested: Finding candidate keys from FDs, determining normal form, checking conflict serializability via precedence graph, and B+ tree insertion/structure.

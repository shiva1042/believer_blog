Search Algorithms in AI

Section: Artificial Intelligence

1. Problem-Solving as Search

Many AI problems can be formulated as search problems. A search problem is defined by:

  • Initial state: The starting state of the agent
  • Actions: The set of actions available in a given state
  • Transition model: A description of what each action does (Result(s, a))
  • Goal test: Determines whether a given state is a goal state
  • Path cost: A function that assigns a numeric cost to each path

A solution is a sequence of actions from the initial state to a goal state. An optimal solution is a solution with the lowest path cost.

2. Uninformed (Blind) Search Strategies

Uninformed search strategies have no additional information about states beyond the problem definition. They differ in the order in which nodes are expanded.

2.1 Breadth-First Search (BFS)

Expands the shallowest unexpanded node first. Uses a FIFO queue.

  • Complete: Yes (if branching factor b is finite)
  • Optimal: Yes (if all step costs are equal)
  • Time complexity: O(bd)
  • Space complexity: O(bd) — stores all nodes at the current level

2.2 Depth-First Search (DFS)

Expands the deepest unexpanded node first. Uses a LIFO stack (or recursion).

  • Complete: No (may loop in infinite-depth or cyclic spaces; complete in finite spaces with cycle detection)
  • Optimal: No
  • Time complexity: O(bm) where m is maximum depth
  • Space complexity: O(bm) — linear in maximum depth

2.3 Uniform-Cost Search (UCS)

Expands the node with the lowest path cost g(n). Uses a priority queue ordered by g(n). Equivalent to BFS when all step costs are equal. This is essentially Dijkstra's algorithm for search.

  • Complete: Yes (if step costs ≥ ε > 0)
  • Optimal: Yes
  • Time complexity: O(b1 + ⌊C*/ε⌋) where C* is the optimal cost
  • Space complexity: O(b1 + ⌊C*/ε⌋)

2.4 Depth-Limited Search (DLS)

DFS with a predetermined depth limit l. Nodes at depth l are treated as if they have no successors.

  • Complete: No (goal may be beyond depth limit)
  • Optimal: No
  • Time complexity: O(bl)
  • Space complexity: O(bl)

2.5 Iterative Deepening Depth-First Search (IDDFS)

Repeatedly applies DLS with increasing depth limits (0, 1, 2, ...). Combines the benefits of BFS (completeness, optimality with uniform costs) with DFS (linear space).

  • Complete: Yes
  • Optimal: Yes (if all step costs are equal)
  • Time complexity: O(bd)
  • Space complexity: O(bd) — linear!
IDDFS total nodes generated = d·b + (d-1)·b² + ... + 1·bd
The overhead of re-expansion is at most a factor of b/(b-1) compared to BFS.
Example: For b = 10, d = 5:
BFS generates: 10 + 100 + 1,000 + 10,000 + 100,000 = 111,110 nodes
IDDFS generates: 50 + 400 + 3,000 + 20,000 + 100,000 = 123,450 nodes
Only about 11% overhead, but IDDFS uses O(50) space vs. O(100,000) for BFS.

3. Informed (Heuristic) Search Strategies

Informed search uses a heuristic function h(n) that estimates the cost from node n to the nearest goal. A good heuristic can dramatically reduce the search effort.

3.1 Greedy Best-First Search

Expands the node that appears closest to the goal, i.e., the node with the smallest h(n).

  • Complete: No (can get stuck in loops; complete with repeated-state checking in finite spaces)
  • Optimal: No
  • Time complexity: O(bm) in worst case
  • Space complexity: O(bm)

3.2 A* Search

The most widely used informed search algorithm. Evaluates nodes using:

f(n) = g(n) + h(n)
where g(n) = actual cost from start to n, h(n) = estimated cost from n to goal

A* expands the node with the smallest f(n). Uses a priority queue ordered by f(n).

  • Complete: Yes (if step costs ≥ ε > 0 and b is finite)
  • Optimal: Yes (if h(n) is admissible; for graph search, h(n) must also be consistent)
  • Time complexity: Exponential in worst case, but much better with good heuristics
  • Space complexity: O(bd) — keeps all nodes in memory

4. Heuristic Properties

4.1 Admissibility

A heuristic h(n) is admissible if it never overestimates the true cost to reach the goal from n.

h(n) ≤ h*(n) for all n
where h*(n) is the true cost from n to the nearest goal

An admissible heuristic is also called an optimistic heuristic. A* with an admissible heuristic is guaranteed to find the optimal solution (for tree search).

4.2 Consistency (Monotonicity)

A heuristic h(n) is consistent if for every node n and every successor n' generated by action a:

h(n) ≤ c(n, a, n') + h(n')
(triangle inequality)

Every consistent heuristic is admissible, but the converse is not always true. A* with a consistent heuristic is optimal for graph search (with closed list).

4.3 Dominance

If h1(n) ≥ h2(n) for all n (both admissible), then h1 dominates h2. A dominating heuristic is always better or equal — it never expands more nodes than the dominated one.

Example — 8-Puzzle Heuristics:
  • h1 = number of misplaced tiles (admissible)
  • h2 = sum of Manhattan distances (admissible)
  • h2 dominates h1 because h2(n) ≥ h1(n) for all n
  • Therefore, A* with h2 will expand fewer or equal nodes than A* with h1

6. Important Points for GATE

  • IDDFS is the preferred uninformed search method when the search space is large and the solution depth is unknown.
  • A* is optimally efficient — no other optimal algorithm using the same heuristic expands fewer nodes.
  • If h(n) = 0 for all n, A* reduces to UCS.
  • If h(n) = 0 and all step costs are 1, A*/UCS reduces to BFS.
  • Bidirectional search can reduce time complexity from O(bd) to O(bd/2).

Searching, Sorting, and Hashing

⚡ Section 5: Algorithms

1. Searching Algorithms

1.1 Linear Search

Linear search examines each element of the array sequentially until a match is found or the array is exhausted.

LinearSearch(A, n, key):
  for i = 0 to n-1:
    if A[i] == key:
      return i
  return -1
CaseTime Complexity
Best CaseO(1) — element found at first position
Average CaseO(n)
Worst CaseO(n) — element at end or not present

1.2 Binary Search

Binary search works on sorted arrays by repeatedly dividing the search interval in half.

BinarySearch(A, low, high, key):
  if low > high:
    return -1
  mid = (low + high) / 2
  if A[mid] == key:
    return mid
  else if A[mid] > key:
    return BinarySearch(A, low, mid-1, key)
  else:
    return BinarySearch(A, mid+1, high, key)

Time Complexity: O(log n) in all cases except best case O(1).

Recurrence: T(n) = T(n/2) + O(1), which gives T(n) = O(log n) by the Master Theorem.

GATE Tip: Binary search requires the array to be sorted. The number of comparisons for an unsuccessful search in an array of size n is floor(log2 n) + 1.

2. Sorting Algorithms

2.1 Bubble Sort

Repeatedly swaps adjacent elements if they are in the wrong order. After each pass, the largest unsorted element "bubbles" to its correct position.

Best Case: O(n) with optimization (already sorted). Worst Case: O(n2). Stable: Yes.

2.2 Selection Sort

Finds the minimum element in the unsorted portion and swaps it with the first unsorted element.

All Cases: O(n2). Stable: No (default implementation). Minimum number of swaps: O(n).

2.3 Insertion Sort

Builds the sorted array one element at a time by inserting each element into its correct position among previously sorted elements.

Best Case: O(n) (already sorted). Worst Case: O(n2) (reverse sorted). Stable: Yes.

Adaptive: Performs well on nearly sorted data. Number of inversions determines the running time.

2.4 Merge Sort

Divide-and-conquer algorithm that splits the array in half, recursively sorts each half, and merges the sorted halves.

Recurrence: T(n) = 2T(n/2) + O(n) gives T(n) = O(n log n) in all cases.

Space: O(n) auxiliary. Stable: Yes.

2.5 Quick Sort

Selects a pivot element, partitions the array around the pivot (elements less than pivot on left, greater on right), and recursively sorts the partitions.

Best/Average: O(n log n). Worst Case: O(n2) when pivot is always smallest or largest element.

Space: O(log n) average (stack space). Stable: No. In-place: Yes.

2.6 Heap Sort

Builds a max-heap from the array, then repeatedly extracts the maximum element and places it at the end.

All Cases: O(n log n). Space: O(1). Stable: No. In-place: Yes.

2.7 Counting Sort

Non-comparison sort that counts occurrences of each element. Works for integers in a known range [0, k].

Time: O(n + k). Space: O(k). Stable: Yes.

2.8 Radix Sort

Sorts numbers digit by digit using a stable sort (usually counting sort) as a subroutine. Processes from least significant to most significant digit.

Time: O(d(n + k)) where d is the number of digits and k is the range of each digit. Stable: Yes.

4. Hashing

4.1 Hash Functions

A hash function maps a key to an index in a hash table of size m.

  • Division Method: h(k) = k mod m. Choose m as a prime not close to a power of 2.
  • Multiplication Method: h(k) = floor(m * (k * A mod 1)) where 0 < A < 1. Knuth suggests A = (sqrt(5) - 1)/2.
  • Mid-Square Method: Square the key and extract middle digits.
  • Universal Hashing: Randomly select a hash function from a family to minimize worst-case collisions.

4.2 Collision Handling — Chaining (Open Hashing)

Each slot in the hash table contains a linked list of all elements that hash to that slot.

OperationAverage CaseWorst Case
SearchO(1 + alpha)O(n)
InsertO(1)O(1)
DeleteO(1 + alpha)O(n)

Here alpha = n/m is the load factor (ratio of elements to table size).

4.3 Collision Handling — Open Addressing (Closed Hashing)

All elements are stored within the hash table itself. When a collision occurs, probe for the next available slot.

Linear Probing

h(k, i) = (h'(k) + i) mod m for i = 0, 1, 2, ...

Problem: Primary clustering — long runs of occupied slots build up, increasing search time.

Quadratic Probing

h(k, i) = (h'(k) + c1*i + c2*i2) mod m

Problem: Secondary clustering — keys with the same hash follow the same probe sequence.

Double Hashing

h(k, i) = (h1(k) + i * h2(k)) mod m

Best form of open addressing. h2(k) must never evaluate to 0. Common choice: h2(k) = R - (k mod R) where R is a prime smaller than m.

Example: Hash table of size 7, h(k) = k mod 7. Insert keys: 50, 700, 76, 85, 92, 73.
h(50)=1, h(700)=0, h(76)=6, h(85)=1 (collision!), h(92)=1 (collision!), h(73)=3.
With linear probing: 85 goes to slot 2, 92 goes to slot 3, 73 goes to slot 4.

5. Key GATE Takeaways

  • Binary search requires O(log n) time and works only on sorted arrays.
  • Merge sort is the best stable O(n log n) sort; quick sort is fastest in practice but O(n2) worst case.
  • Heap sort is in-place and O(n log n) guaranteed but not stable.
  • For hashing, double hashing is preferred over linear/quadratic probing due to reduced clustering.
  • The expected number of probes in a hash table with load factor alpha = 0.5 using linear probing for unsuccessful search is 2.5.

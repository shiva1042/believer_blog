Sets, Relations, and Functions

Section 1: Engineering Mathematics

1. Sets and Set Operations

A set is a well-defined collection of distinct objects called elements or members.

1.1 Set Notation

  • Roster form: A = {1, 2, 3, 4, 5}
  • Set-builder form: A = {x | x is a positive integer and x ≤ 5}
  • Membership: 3 ∈ A means 3 is an element of A
  • Subset: A ⊆ B means every element of A is in B
  • Proper subset: A ⊂ B means A ⊆ B and A ≠ B

2. Relations

A relation R from set A to set B is a subset of A × B. A relation on set A is a subset of A × A.

2.1 Properties of Relations on a Set A

2.2 Counting Relations

Total relations on a set of n elements: 2n²
Reflexive relations: 2n² − n
Symmetric relations: 2n(n+1)/2
Antisymmetric relations: 2n · 3n(n−1)/2

2.3 Equivalence Relations

A relation R on set A is an equivalence relation if it is reflexive, symmetric, and transitive. It partitions A into disjoint equivalence classes.

Example: Relation "congruence modulo 3" on integers.
a R b if and only if a ≡ b (mod 3).
Equivalence classes: [0] = {..., -3, 0, 3, 6, ...}, [1] = {..., -2, 1, 4, 7, ...}, [2] = {..., -1, 2, 5, 8, ...}
The number of equivalence relations on a set of n elements equals the Bell number B(n).

2.4 Partial Order Relations

A relation R on set A is a partial order (POSET) if it is reflexive, antisymmetric, and transitive. The pair (A, R) is called a partially ordered set.

3. Lattices

A lattice is a partially ordered set in which every pair of elements has a least upper bound (join, ∨) and a greatest lower bound (meet, ∧).

3.1 Properties of Lattices

  • Bounded lattice: Has a greatest element (top, 1) and least element (bottom, 0)
  • Complemented lattice: Every element has a complement
  • Distributive lattice: a ∧ (b ∨ c) = (a ∧ b) ∨ (a ∧ c)
  • Boolean algebra: Complemented distributive lattice

3.2 Hasse Diagram: Divisibility Lattice on {1, 2, 3, 5, 6, 10, 15, 30}

graph BT A["1"] --> B["2"] A --> C["3"] A --> D["5"] B --> E["6"] B --> F["10"] C --> E C --> G["15"] D --> F D --> G E --> H["30"] F --> H G --> H

In the divisibility lattice above, the meet of two elements is their GCD and the join is their LCM. For example: 6 ∧ 10 = GCD(6,10) = 2, and 6 ∨ 10 = LCM(6,10) = 30.

4. Functions

A function f: A → B is a relation from A to B where every element of A is related to exactly one element of B.

4.1 Types of Functions

TypeDefinitionAlso CalledCondition
Injectivef(a) = f(b) ⇒ a = bOne-to-one|A| ≤ |B|
Surjective∀b ∈ B, ∃a ∈ A: f(a) = bOnto|A| ≥ |B|
BijectiveBoth injective and surjectiveOne-to-one correspondence|A| = |B|

4.2 Counting Functions

Total functions from A to B (|A| = m, |B| = n): nm
Injective functions: n! / (n − m)! (when m ≤ n), else 0
Surjective functions: ∑k=0n (-1)k C(n, k)(n − k)m
Bijective functions: n! (when m = n)

4.3 Composition and Inverse

  • Composition: (g ∘ f)(x) = g(f(x)), where f: A → B and g: B → C
  • Inverse: f−1 exists iff f is bijective
  • If f and g are injective, then g ∘ f is injective
  • If f and g are surjective, then g ∘ f is surjective
  • If g ∘ f is injective, then f is injective
  • If g ∘ f is surjective, then g is surjective

5. Closures of Relations

The closure of a relation R with respect to a property P is the smallest relation that contains R and satisfies P.

ClosureMethod
Reflexive closureR ∪ {(a, a) | a ∈ A} = R ∪ IA
Symmetric closureR ∪ R−1
Transitive closureR ∪ R2 ∪ R3 ∪ ... ∪ Rn (Warshall's algorithm)

6. Summary of Key GATE Concepts

  • Know the inclusion-exclusion principle and power set cardinality.
  • Distinguish between equivalence relations (reflexive + symmetric + transitive) and partial orders (reflexive + antisymmetric + transitive).
  • Practice counting: number of relations, equivalence relations, and functions.
  • Understand Hasse diagrams and lattice properties (distributive, complemented, Boolean).
  • For functions: injective requires |A| ≤ |B|, surjective requires |A| ≥ |B|.
  • Composition preserves injectivity and surjectivity.

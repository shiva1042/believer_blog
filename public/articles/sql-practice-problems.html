SQL Practice Problems for GATE

🗄️ Databases

Schema for Problems

The following tables are used across all problems:

Student(sid INT PK, sname VARCHAR, age INT, gpa FLOAT)
Course(cid INT PK, cname VARCHAR, credits INT, dept VARCHAR)
Enrolled(sid INT FK, cid INT FK, grade CHAR(1))
  -- Primary key: (sid, cid)

Employee(eid INT PK, ename VARCHAR, salary INT, dept_id INT FK)
Department(dept_id INT PK, dname VARCHAR, manager_id INT)

Problem 1: Basic SELECT

Q: Find names of all students with GPA greater than 3.5.

SELECT sname
FROM Student
WHERE gpa > 3.5;

Explanation: Simple filtering with WHERE clause.

Problem 2: SELECT with ORDER BY

Q: List all students sorted by GPA in descending order. Show name and GPA.

SELECT sname, gpa
FROM Student
ORDER BY gpa DESC;

Problem 3: INNER JOIN

Q: Find names of students enrolled in course with cid = 101.

SELECT S.sname
FROM Student S INNER JOIN Enrolled E ON S.sid = E.sid
WHERE E.cid = 101;

Explanation: INNER JOIN returns only rows where the join condition matches in both tables.

Problem 4: Multi-table JOIN

Q: Find student names and course names for all enrollments.

SELECT S.sname, C.cname
FROM Student S
JOIN Enrolled E ON S.sid = E.sid
JOIN Course C ON E.cid = C.cid;

Problem 5: LEFT JOIN

Q: Find all students and their enrolled courses. Include students not enrolled in any course.

SELECT S.sname, C.cname
FROM Student S
LEFT JOIN Enrolled E ON S.sid = E.sid
LEFT JOIN Course C ON E.cid = C.cid;

Explanation: LEFT JOIN keeps all rows from the left table (Student). If no match in Enrolled/Course, NULLs appear.

Problem 6: RIGHT JOIN

Q: Find all courses and the students enrolled. Include courses with no students.

SELECT C.cname, S.sname
FROM Student S
RIGHT JOIN Enrolled E ON S.sid = E.sid
RIGHT JOIN Course C ON E.cid = C.cid;

Problem 7: FULL OUTER JOIN

Q: List all students and all courses, showing enrollment where it exists.

SELECT S.sname, C.cname
FROM Student S
FULL OUTER JOIN Enrolled E ON S.sid = E.sid
FULL OUTER JOIN Course C ON E.cid = C.cid;

Problem 8: GROUP BY with COUNT

Q: Find the number of students enrolled in each course.

SELECT E.cid, C.cname, COUNT(E.sid) AS num_students
FROM Enrolled E
JOIN Course C ON E.cid = C.cid
GROUP BY E.cid, C.cname;

Explanation: GROUP BY groups rows by cid. COUNT counts students per group.

Problem 9: GROUP BY with HAVING

Q: Find courses with more than 5 students enrolled.

SELECT E.cid, COUNT(*) AS num_students
FROM Enrolled E
GROUP BY E.cid
HAVING COUNT(*) > 5;

Explanation: HAVING filters groups (like WHERE, but for aggregated results). WHERE filters rows before grouping; HAVING filters after.

Problem 10: Aggregate Functions

Q: Find the average, minimum, and maximum salary in each department.

SELECT D.dname, AVG(E.salary) AS avg_sal,
       MIN(E.salary) AS min_sal, MAX(E.salary) AS max_sal
FROM Employee E
JOIN Department D ON E.dept_id = D.dept_id
GROUP BY D.dname;

Problem 11: Subquery (IN)

Q: Find students who are enrolled in at least one course offered by the 'CS' department.

SELECT sname
FROM Student
WHERE sid IN (
    SELECT E.sid
    FROM Enrolled E
    JOIN Course C ON E.cid = C.cid
    WHERE C.dept = 'CS'
);

Problem 12: Subquery (NOT IN)

Q: Find students who are NOT enrolled in any course.

SELECT sname
FROM Student
WHERE sid NOT IN (
    SELECT sid FROM Enrolled
);
Caution: NOT IN behaves unexpectedly with NULLs. If the subquery returns any NULL, NOT IN returns empty. Use NOT EXISTS for safety.

Problem 13: Correlated Subquery

Q: Find students whose GPA is higher than the average GPA of all students.

SELECT sname, gpa
FROM Student S
WHERE gpa > (SELECT AVG(gpa) FROM Student);

Q (correlated): Find students who have enrolled in more courses than student with sid = 1.

SELECT S.sname
FROM Student S
WHERE (SELECT COUNT(*) FROM Enrolled E WHERE E.sid = S.sid) >
      (SELECT COUNT(*) FROM Enrolled E WHERE E.sid = 1);

Explanation: A correlated subquery references the outer query (S.sid). It executes once for each row of the outer query.

Problem 14: EXISTS

Q: Find students who are enrolled in at least one course.

SELECT S.sname
FROM Student S
WHERE EXISTS (
    SELECT 1 FROM Enrolled E WHERE E.sid = S.sid
);

Explanation: EXISTS returns TRUE if the subquery returns at least one row. Often preferred over IN for performance.

Problem 15: Nested Subquery with ALL

Q: Find the student with the highest GPA.

SELECT sname, gpa
FROM Student
WHERE gpa >= ALL (SELECT gpa FROM Student);

Problem 16: Self-Join

Q: Find pairs of students with the same GPA.

SELECT S1.sname AS student1, S2.sname AS student2, S1.gpa
FROM Student S1
JOIN Student S2 ON S1.gpa = S2.gpa AND S1.sid < S2.sid;

Explanation: The condition S1.sid < S2.sid avoids duplicate pairs and self-pairing.

Problem 17: Division (Students enrolled in ALL CS courses)

Q: Find students enrolled in every course offered by the CS department.

SELECT S.sname
FROM Student S
WHERE NOT EXISTS (
    SELECT C.cid
    FROM Course C
    WHERE C.dept = 'CS'
    AND NOT EXISTS (
        SELECT 1 FROM Enrolled E
        WHERE E.sid = S.sid AND E.cid = C.cid
    )
);

Explanation: This is relational division. Read as: "Find students S such that there is no CS course that S is not enrolled in."

Problem 18: SUM and GROUP BY

Q: Find total credits each student is enrolled for.

SELECT S.sname, SUM(C.credits) AS total_credits
FROM Student S
JOIN Enrolled E ON S.sid = E.sid
JOIN Course C ON E.cid = C.cid
GROUP BY S.sid, S.sname;

Common GATE SQL Patterns

PatternSQL Construct
Find tuples satisfying a conditionSELECT ... WHERE ...
Find tuples in one table but not anotherNOT IN, NOT EXISTS, EXCEPT
Find tuples satisfying ALL conditionsDouble NOT EXISTS (division)
Count/sum per groupGROUP BY + aggregate
Filter groupsHAVING
Find max/min tupleWHERE val >= ALL (...) or ORDER BY ... LIMIT 1
Compare with averageSubquery: WHERE col > (SELECT AVG(...))
GATE Tip: GATE frequently tests: (1) difference between WHERE and HAVING, (2) correlated vs non-correlated subqueries, (3) division queries, (4) behavior of NULL in aggregates and NOT IN.

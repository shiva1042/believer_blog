1. IP Addressing Basics Review

1.1 IP Address Structure

An IPv4 address is 32 bits, written as four octets in dotted decimal: a.b.c.d

ClassFirst Octet RangeDefault MaskNetwork/Host Bits
A1 - 126255.0.0.0 (/8)8 / 24
B128 - 191255.255.0.0 (/16)16 / 16

1.2 Special Addresses

  • 127.0.0.0/8 — Loopback
  • 0.0.0.0 — Default route / "this network"
  • 255.255.255.255 — Limited broadcast
  • Network address: host bits all 0. Broadcast address: host bits all 1.

1.3 Key Formulas

FormulaExpression
Number of subnets2^s (s = borrowed bits from host)
Hosts per subnet2^h - 2 (h = remaining host bits)
Network addressIP AND subnet mask (bitwise AND)
Broadcast addressNetwork address OR (NOT mask)
First usable hostNetwork address + 1
Last usable hostBroadcast address - 1
Block size256 - mask octet value (for the interesting octet)

2. Subnetting Step-by-Step Method

  1. Write the IP and mask in binary.
  2. Perform bitwise AND to find the network address.
  3. Identify network bits (n) and host bits (h). h = 32 - n.
  4. Number of usable hosts = 2^h - 2.
  5. Broadcast = set all host bits to 1.
  6. First host = network + 1. Last host = broadcast - 1.

4. VLSM (Variable Length Subnet Masking)

VLSM allows different subnets within the same network to have different masks. This avoids wasting IP addresses.

Method:

  1. Sort subnets by required hosts (largest first).
  2. Allocate the largest subnet first.
  3. Use the next available block for the next subnet.
  4. Each subnet gets the smallest block that fits its host requirement.

Example: Given 192.168.1.0/24, create subnets for: 60 hosts, 30 hosts, 10 hosts, 2 hosts.

Sort by size: 60, 30, 10, 2

Subnet 1: 60 hosts → need 2^h - 2 >= 60 → h=6, /26 (62 hosts)
  192.168.1.0/26  (hosts: .1 to .62, broadcast: .63)

Subnet 2: 30 hosts → h=5, /27 (30 hosts)
  192.168.1.64/27  (hosts: .65 to .94, broadcast: .95)

Subnet 3: 10 hosts → h=4, /28 (14 hosts)
  192.168.1.96/28  (hosts: .97 to .110, broadcast: .111)

Subnet 4: 2 hosts → h=2, /30 (2 hosts)
  192.168.1.112/30  (hosts: .113 to .114, broadcast: .115)

5. Practice Problems

Problem 1

Q: Given IP 192.168.10.130/26, find the network address, broadcast address, and usable host range.

/26 → mask = 255.255.255.192
Host bits = 32 - 26 = 6
Block size = 256 - 192 = 64

130 / 64 = 2.03 → network starts at 2 * 64 = 128

Network address: 192.168.10.128
Broadcast: 192.168.10.191 (128 + 63)
First host: 192.168.10.129
Last host: 192.168.10.190
Hosts per subnet: 2^6 - 2 = 62

Problem 2

Q: How many subnets and hosts per subnet does 172.16.0.0/20 provide compared to default Class B?

Default Class B mask: /16
Given: /20 → borrowed 20 - 16 = 4 bits from host
Subnets = 2^4 = 16
Host bits = 32 - 20 = 12
Hosts per subnet = 2^12 - 2 = 4094

Problem 3

Q: An organization needs 500 hosts. What is the minimum prefix length?

Need 2^h - 2 >= 500
2^9 - 2 = 510 >= 500 ✓ (h=9)
2^8 - 2 = 254 < 500 ✗

Prefix length = 32 - 9 = /23
Subnet mask: 255.255.254.0

Problem 4

Q: Given 10.1.1.0/24, can it be subnetted to provide at least 6 subnets each with at least 20 hosts?

Need at least 6 subnets: 2^s >= 6 → s=3 (8 subnets)
With /24 and 3 borrowed bits: /27
Host bits = 32 - 27 = 5
Hosts per subnet = 2^5 - 2 = 30 >= 20 ✓

Answer: Yes. Use /27 (8 subnets, 30 hosts each).

Problem 5

Q: What is the network address of 172.25.150.67/21?

/21 → mask = 255.255.248.0
Third octet: 150 AND 248
150 in binary: 10010110
248 in binary: 11111000
AND result:    10010000 = 144

Network: 172.25.144.0

Problem 6

Q: Two hosts: 192.168.1.100/28 and 192.168.1.110/28. Are they on the same subnet?

/28 → block size = 256 - 240 = 16
100 / 16 = 6.25 → network at 6*16 = 96 → 192.168.1.96
110 / 16 = 6.87 → network at 6*16 = 96 → 192.168.1.96

Both in subnet 192.168.1.96/28. YES, same subnet.

Problem 7

Q: 192.168.1.120/28 and 192.168.1.140/28. Same subnet?

120 / 16 = 7.5 → network at 7*16 = 112
140 / 16 = 8.75 → network at 8*16 = 128

Different subnets (192.168.1.112 vs 192.168.1.128). NO.

Problem 8: Supernetting

Q: Combine 192.168.0.0/24, 192.168.1.0/24, 192.168.2.0/24, 192.168.3.0/24 into one supernet.

Four /24 networks → need 2 bits less specificity.
/24 - 2 = /22

192.168.0.0 = 11000000.10101000.000000|00.00000000
192.168.3.0 = 11000000.10101000.000000|11.00000000
First 22 bits are the same.

Supernet: 192.168.0.0/22
Mask: 255.255.252.0

Problem 9

Q: An ISP has the block 200.10.0.0/21. It wants to divide it equally among 4 organizations. What block does each get?

/21 has 2^(32-21) = 2048 addresses.
Divide by 4: each gets 512 addresses = 2^9 → /23

Org 1: 200.10.0.0/23
Org 2: 200.10.2.0/23
Org 3: 200.10.4.0/23
Org 4: 200.10.6.0/23

Problem 10

Q: How many /28 subnets can be created from a /24 network?

A /24 has 2^8 = 256 addresses.
A /28 has 2^4 = 16 addresses.
Number of /28 subnets = 256 / 16 = 16

Problem 11

Q: Given 10.0.0.0/8, find the subnet mask needed for 1000 subnets.

Default: /8. Need 2^s >= 1000 → s = 10 (2^10 = 1024)
New prefix: /8 + 10 = /18
Mask: 255.255.192.0
Hosts per subnet: 2^14 - 2 = 16382

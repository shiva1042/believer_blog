Supervised Learning Algorithms

Section: Artificial Intelligence

1. Linear Regression

Linear regression models the relationship between a dependent variable y and one or more independent variables x using a linear function.

Simple Linear Regression: y = w·x + b
Multiple Linear Regression: y = w1x1 + w2x2 + ... + wnxn + b
In vector form: y = WTX + b

The model is trained by minimizing the Mean Squared Error (MSE) loss function:

MSE = (1/n) ∑ (yi - ŷi)² for i = 1 to n

The optimal weights can be found using the Normal Equation (closed-form solution):

W = (XTX)-1 XTy

Alternatively, Gradient Descent iteratively updates weights:

w := w - α · ∂MSE/∂w
where α is the learning rate

2. Logistic Regression

Despite its name, logistic regression is a classification algorithm. It models the probability that an instance belongs to the positive class using the sigmoid function.

σ(z) = 1 / (1 + e-z) where z = WTX + b
P(y=1|x) = σ(WTX + b)
Decision: y = 1 if P(y=1|x) ≥ 0.5, else y = 0

The loss function is Binary Cross-Entropy (Log Loss):

L = -(1/n) ∑ [yi log(ŷi) + (1 - yi) log(1 - ŷi)]

Properties of the sigmoid function: output is always in (0, 1), monotonically increasing, σ(0) = 0.5, σ'(z) = σ(z)(1 - σ(z)).

3. Decision Trees

Decision trees classify instances by sorting them from the root to a leaf node, which holds the classification. Each internal node tests an attribute, each branch corresponds to an attribute value, and each leaf assigns a class.

3.1 Information Gain and Entropy (ID3 Algorithm)

Entropy: H(S) = -∑ pi log2(pi) for each class i

Information Gain: IG(S, A) = H(S) - ∑ (|Sv|/|S|) · H(Sv) for each value v of attribute A

ID3 selects the attribute with the highest information gain at each node. Entropy = 0 when all examples belong to one class (pure). Entropy = 1 (for binary classification) when examples are evenly split.

Example: Dataset S with 9 positive and 5 negative examples.
H(S) = -(9/14)log2(9/14) - (5/14)log2(5/14) = 0.940 bits
If attribute A splits S into S1 (3+, 2-) and S2 (6+, 3-):
H(S1) = -(3/5)log2(3/5) - (2/5)log2(2/5) = 0.971
H(S2) = -(6/9)log2(6/9) - (3/9)log2(3/9) = 0.918
IG(S, A) = 0.940 - (5/14)(0.971) - (9/14)(0.918) = 0.940 - 0.347 - 0.590 = 0.003

3.2 C4.5 Algorithm

Improvement over ID3. Uses Gain Ratio to avoid bias toward multi-valued attributes:

SplitInfo(S, A) = -∑ (|Sv|/|S|) log2(|Sv|/|S|)
GainRatio(S, A) = IG(S, A) / SplitInfo(S, A)

C4.5 also handles continuous attributes, missing values, and performs post-pruning.

3.3 CART (Classification and Regression Trees)

Uses the Gini Index for classification:

Gini(S) = 1 - ∑ pi² for each class i

Gini = 0 when all examples are of one class (pure)
Gini = 0.5 for binary classification with equal split

CART always creates binary splits. For regression, CART uses variance reduction as the splitting criterion.

4. Support Vector Machines (SVM)

SVM finds the optimal hyperplane that separates classes with the maximum margin.

4.1 Linear SVM

Decision boundary: WTX + b = 0
Margin = 2 / ||W||
Objective: Minimize ||W||² / 2 subject to yi(WTxi + b) ≥ 1 for all i

Support vectors are the data points closest to the decision boundary. They lie on the margin and determine the hyperplane. Only support vectors matter for the decision boundary; removing other points does not change the classifier.

4.2 Soft Margin SVM

Allows some misclassification using slack variables ξi for non-linearly separable data:

Minimize: ||W||²/2 + C ∑ ξi
subject to: yi(WTxi + b) ≥ 1 - ξi, ξi ≥ 0

The parameter C controls the tradeoff: large C = less misclassification (harder margin), small C = wider margin (more misclassification allowed).

4.3 Kernel Trick

Maps data to a higher-dimensional space where it becomes linearly separable, without explicitly computing the mapping. Common kernels:

KernelFormula K(x, x')Use Case
LinearxTx'Linearly separable data
Polynomial(xTx' + c)dNon-linear boundaries, moderate complexity
RBF (Gaussian)exp(-γ||x - x'||²)Complex non-linear boundaries, most popular
Sigmoidtanh(αxTx' + c)Neural network-like behavior

5. K-Nearest Neighbors (KNN)

KNN is a lazy learning (instance-based) algorithm that stores all training data and classifies new instances based on the majority class of their K nearest neighbors.

Euclidean distance: d(x, y) = √(∑(xi - yi)²)
Manhattan distance: d(x, y) = ∑|xi - yi|
Minkowski distance: d(x, y) = (∑|xi - yi|p)1/p
  • K is odd for binary classification to avoid ties
  • Small K: complex boundary, high variance, low bias (overfitting)
  • Large K: smoother boundary, low variance, high bias (underfitting)
  • Sensitive to feature scaling — normalization is essential
  • Computationally expensive at prediction time: O(nd) per query (n samples, d dimensions)

6. Naive Bayes Classifier

Based on Bayes' Theorem with the "naive" assumption that all features are conditionally independent given the class label.

Bayes' Theorem: P(C|X) = P(X|C) · P(C) / P(X)

Naive assumption: P(x1, x2, ..., xn|C) = ∏ P(xi|C)

Classification: C* = argmaxC P(C) ∏ P(xi|C)

Variants: Gaussian NB (continuous features, assumed normal), Multinomial NB (text classification, word counts), Bernoulli NB (binary features).

Example: Spam classification with features: "free" and "money".
P(Spam) = 0.4, P(Not Spam) = 0.6
P("free"|Spam) = 0.8, P("free"|Not Spam) = 0.1
P("money"|Spam) = 0.7, P("money"|Not Spam) = 0.05
For email with both "free" and "money":
P(Spam|X) ∝ 0.4 × 0.8 × 0.7 = 0.224
P(Not Spam|X) ∝ 0.6 × 0.1 × 0.05 = 0.003
Since 0.224 >> 0.003, classify as Spam.

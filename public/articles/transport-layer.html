1. User Datagram Protocol (UDP)

UDP is a connectionless, unreliable transport protocol that provides minimal overhead. It sends datagrams without establishing a connection and offers no guarantees on delivery, ordering, or duplicate protection.

1.1 UDP Header (8 bytes)

FieldSizeDescription
Source Port16 bitsSender's port number
Destination Port16 bitsReceiver's port number
Length16 bitsTotal length of UDP datagram (header + data), minimum 8 bytes
Checksum16 bitsError detection (optional in IPv4, mandatory in IPv6)

1.2 UDP Characteristics and Use Cases

2. Transmission Control Protocol (TCP)

TCP is a connection-oriented, reliable, byte-stream protocol. It provides ordered delivery, error detection, flow control, and congestion control.

2.1 TCP Header (20-60 bytes)

Source Port16 bitsSender's port
Destination Port16 bitsReceiver's port

Acknowledgment Number32 bitsNext byte expected from the other side
Data Offset (Header Length)4 bitsHeader length in 32-bit words
Flags6 bitsURG, ACK, PSH, RST, SYN, FIN
Window Size16 bitsReceiver's advertised window (flow control)
Checksum16 bitsError detection (covers header + data + pseudo-header)
Urgent Pointer16 bitsPoints to end of urgent data (when URG flag set)
Options0-320 bitsMSS, Window Scaling, Timestamps, SACK

3. TCP Connection Management

3.1 Three-Way Handshake (Connection Establishment)

sequenceDiagram participant C as Client participant S as Server Note over S: LISTEN state C->>S: SYN (seq=x) Note over C: SYN_SENT S->>C: SYN+ACK (seq=y, ack=x+1) Note over S: SYN_RCVD C->>S: ACK (seq=x+1, ack=y+1) Note over C,S: ESTABLISHED
SYN consumes a sequence number. After the handshake, the client's next sequence number is x+1 and the server's is y+1. The ACK in step 3 can carry data (piggybacking).

3.2 Four-Way Termination (Connection Teardown)

sequenceDiagram participant C as Client participant S as Server C->>S: FIN (seq=u) Note over C: FIN_WAIT_1 S->>C: ACK (ack=u+1) Note over C: FIN_WAIT_2 Note over S: CLOSE_WAIT Note right of S: Server may still send data S->>C: FIN (seq=v) Note over S: LAST_ACK C->>S: ACK (ack=v+1) Note over C: TIME_WAIT (2MSL) Note over S: CLOSED Note over C: CLOSED (after timeout)

TIME_WAIT: The client waits for 2 × MSL (Maximum Segment Lifetime, typically 2 minutes) before closing to ensure the final ACK is received and to prevent old segments from being confused with new connections.

4. Flow Control: Sliding Window

TCP uses a sliding window mechanism for flow control. The receiver advertises its available buffer space (rwnd = receive window). The sender cannot have more than min(cwnd, rwnd) unacknowledged bytes outstanding.

ProtocolDescriptionEfficiency
Stop-and-WaitSend one frame, wait for ACKη = ttrans / (ttrans + 2 × tprop) = 1/(1+2a)
Go-Back-NWindow size N; retransmit all frames from the errored oneη = N/(1+2a) if N < 1+2a; else 1
Selective RepeatWindow size N; retransmit only the errored frameSame formula as GBN but only one frame retransmitted
Window Size Constraints:
  • Go-Back-N: Sender window ≤ 2n − 1 (where n = bits for sequence number)
  • Selective Repeat: Sender window ≤ 2n−1 and Receiver window ≤ 2n−1
  • Reason: To avoid ambiguity between new and retransmitted frames

5. Congestion Control

TCP uses congestion control to prevent overloading the network. The sender maintains a congestion window (cwnd) that limits the amount of data in transit.

5.1 Congestion Control Phases

sequenceDiagram participant S as Sender Note over S: cwnd = 1 MSS Note over S: === Slow Start === Note over S: cwnd doubles each RTT Note over S: cwnd: 1, 2, 4, 8, 16... Note over S: Until cwnd >= ssthresh Note over S: === Congestion Avoidance === Note over S: cwnd increases by 1 MSS per RTT Note over S: cwnd: 16, 17, 18, 19... Note over S: Until packet loss detected Note over S: === Loss Event === Note over S: Timeout: ssthresh = cwnd/2, cwnd = 1 Note over S: 3 dup ACKs: ssthresh = cwnd/2, cwnd = ssthresh + 3 (Fast Recovery)

6. Sockets

A socket is an endpoint for communication, identified by an IP address and a port number. It serves as the interface between the application layer and the transport layer.

7. Summary and GATE Key Points

Key Points for GATE:
  • UDP header = 8 bytes; TCP header = 20-60 bytes.
  • TCP 3-way handshake: SYN, SYN+ACK, ACK. Both SYN and FIN consume one sequence number.
  • TIME_WAIT = 2 × MSL ensures old segments do not interfere with new connections.
  • Go-Back-N window size ≤ 2n − 1; Selective Repeat ≤ 2n−1.
  • Slow Start: cwnd doubles each RTT (exponential). Congestion Avoidance: cwnd += 1 each RTT (linear).
  • TCP Tahoe: always goes to cwnd=1 on loss. TCP Reno: goes to cwnd = ssthresh on 3 dup ACKs (fast recovery).
  • Effective throughput = cwnd / RTT (approximately).
  • Sender's effective window = min(cwnd, rwnd).
  • TCP provides: reliability, ordering, flow control, congestion control. UDP provides: none of these.
  • Stop-and-Wait efficiency = 1/(1+2a) where a = propagation delay / transmission delay.

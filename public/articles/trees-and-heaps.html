1. Binary Tree Basics

A binary tree is a hierarchical data structure where each node has at most two children: left and right.

1.1 Key Properties

PropertyFormula
Max nodes at level l2l (root at level 0)
Max nodes in tree of height h2h+1 - 1
Min height for n nodesfloor(log2 n)
Number of leaf nodesn0 = n2 + 1 (where n2 = nodes with 2 children)
Number of NULL pointersn + 1 (for n nodes)

1.2 Types of Binary Trees

TypeDefinition
Full (Strict)Every node has 0 or 2 children
CompleteAll levels filled except possibly the last, which is filled left to right
PerfectAll internal nodes have 2 children and all leaves are at the same level
BalancedHeight difference between left and right subtrees is at most 1 for every node
Degenerate (Skewed)Every node has at most 1 child (essentially a linked list)

2.3 Traversal Implementations

// Inorder traversal (recursive)
void inorder(struct Node *root) {
    if (root == NULL) return;
    inorder(root->left);
    printf("%d ", root->data);
    inorder(root->right);
}

// Preorder traversal (recursive)
void preorder(struct Node *root) {
    if (root == NULL) return;
    printf("%d ", root->data);
    preorder(root->left);
    preorder(root->right);
}

// Level-order traversal (iterative, using queue)
void levelOrder(struct Node *root) {
    Queue q;
    enqueue(&q, root);
    while (!isEmpty(&q)) {
        struct Node *curr = dequeue(&q);
        printf("%d ", curr->data);
        if (curr->left) enqueue(&q, curr->left);
        if (curr->right) enqueue(&q, curr->right);
    }
}

Reconstruction: A unique binary tree can be constructed from Inorder + Preorder, or Inorder + Postorder. Preorder + Postorder alone is NOT sufficient (unless the tree is full).

3. Binary Search Tree (BST)

A BST is a binary tree where for every node: all keys in the left subtree are less, and all keys in the right subtree are greater.

3.1 Operations

OperationAverageWorst (skewed)
SearchO(log n)O(n)
InsertO(log n)O(n)
DeleteO(log n)O(n)
Find Min/MaxO(log n)O(n)
Inorder traversalO(n) — gives sorted order

3.2 BST Deletion

Three cases when deleting a node:

  1. Leaf node: Simply remove it.
  2. One child: Replace node with its child.
  3. Two children: Replace with inorder successor (smallest in right subtree) or inorder predecessor (largest in left subtree), then delete that replacement node.

3.3 Number of Distinct BSTs

The number of structurally unique BSTs with n keys = Catalan number Cn = C(2n,n)/(n+1).

4. AVL Trees

An AVL tree is a self-balancing BST where the balance factor (height of left subtree - height of right subtree) of every node is -1, 0, or 1.

4.1 Rotations

ImbalanceRotationTrigger
Left-Left (LL)Single Right RotationInsert in left subtree of left child
Right-Right (RR)Single Left RotationInsert in right subtree of right child
Left-Right (LR)Left then Right RotationInsert in right subtree of left child
Right-Left (RL)Right then Left RotationInsert in left subtree of right child

4.2 AVL Tree Properties

PropertyValue
Height of AVL tree with n nodesO(log n), specifically ~1.44 log2(n+2)
Min nodes for height hN(h) = N(h-1) + N(h-2) + 1 (Fibonacci-like)
All operationsO(log n) guaranteed

5. Red-Black Trees

A Red-Black tree is a self-balancing BST with colored nodes (red or black) that satisfy these properties:

  1. Every node is either red or black.
  2. The root is black.
  3. Every leaf (NIL) is black.
  4. A red node cannot have a red child (no two consecutive reds).
  5. Every path from a node to its descendant NIL nodes has the same number of black nodes (black-height).
PropertyAVL TreeRed-Black Tree

6. Binary Heaps

A binary heap is a complete binary tree satisfying the heap property. It is efficiently stored as an array.

6.1 Heap Properties

TypeProperty
Min-HeapParent ≤ Children (root is minimum)
Max-HeapParent ≥ Children (root is maximum)

6.2 Array Representation (0-indexed)

RelationshipFormula
Parent of node i(i - 1) / 2
Left child of node i2i + 1
Right child of node i2i + 2

6.4 Heap Operations

6.5 Heapify (Sift Down)

void maxHeapify(int arr[], int n, int i) {
    int largest = i;
    int left = 2 * i + 1;
    int right = 2 * i + 2;
    if (left < n && arr[left] > arr[largest])
        largest = left;
    if (right < n && arr[right] > arr[largest])
        largest = right;
    if (largest != i) {
        swap(&arr[i], &arr[largest]);
        maxHeapify(arr, n, largest);
    }
}

// Build heap in O(n) - start from last non-leaf
void buildMaxHeap(int arr[], int n) {
    for (int i = n / 2 - 1; i >= 0; i--)
        maxHeapify(arr, n, i);
}

Why Build Heap is O(n): Most nodes are near the leaves and need very few sift-down steps. The sum converges to O(n), not O(n log n). This is a common GATE question.

6.6 Heap Sort

void heapSort(int arr[], int n) {
    buildMaxHeap(arr, n);
    for (int i = n - 1; i > 0; i--) {
        swap(&arr[0], &arr[i]);  // move max to end
        maxHeapify(arr, i, 0);    // restore heap on reduced array
    }
}
// Time: O(n log n), Space: O(1), Not stable

8. GATE Exam Tips

  • Inorder traversal of BST gives sorted output.
  • Number of structurally distinct BSTs/binary trees with n nodes = Catalan number.
  • Build heap is O(n), NOT O(n log n). This is frequently tested.
  • Heap sort: O(n log n) always, in-place, but NOT stable.
  • AVL: max 2 rotations for insert, O(log n) rotations for delete.
  • Red-Black: at most 2 rotations for insert, at most 3 for delete.
  • Min nodes in AVL tree of height h: N(h) = N(h-1) + N(h-2) + 1, with N(0)=1, N(1)=2.
  • A heap is NOT a BST. Left child can be greater than right child in a max-heap.

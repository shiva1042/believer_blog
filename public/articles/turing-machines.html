Turing Machines and Undecidability

🔄 Section 6: Theory of Computation

1. Turing Machine Definition

1.1 Formal Definition

A Turing Machine (TM) is a 7-tuple M = (Q, Sigma, Gamma, delta, q0, B, F) where:

  • Q: Finite set of states
  • Sigma: Input alphabet (does not include blank symbol B)
  • Gamma: Tape alphabet (Sigma is a subset of Gamma, B is in Gamma)
  • delta: Transition function, delta: Q x Gamma -> Q x Gamma x {L, R}
  • q0: Start state
  • B: Blank symbol (in Gamma but not in Sigma)
  • F: Set of accept (final) states

1.2 TM Configuration

A configuration (instantaneous description) is a triple (state, tape contents, head position) that captures the complete state of the TM at any moment. Written as alpha q beta where q is the current state, and the head points to the first symbol of beta.

2. TM as Language Acceptor

A TM M accepts a string w if M enters an accept state when started with w on its tape.

A TM M rejects a string w if M halts in a non-accept state.

A TM M may also loop forever on input w (neither accepting nor rejecting).

Language ClassTM BehaviorDefinition
Recursively Enumerable (RE)Accept or loopTM halts and accepts for w in L; may loop for w not in L
Recursive (Decidable)Always haltsTM halts and accepts for w in L; halts and rejects for w not in L
Key Insight: Every recursive language is RE, but not every RE language is recursive. The complement of a recursive language is recursive. If both L and its complement are RE, then L is recursive.

3. TM as Function Computer

A TM can compute functions f: Sigma* -> Sigma*. Given input w on the tape, if the TM halts, the output f(w) is what remains on the tape. If f is total (defined for all inputs), it is a total computable function.

4. Variants of Turing Machines

4.1 Multi-tape TM

Has k tapes, each with its own head. Transition function: delta: Q x Gamma^k -> Q x Gamma^k x {L,R,S}^k.

Equivalence: Every multi-tape TM can be simulated by a single-tape TM (with polynomial slowdown: if multi-tape runs in T(n), single-tape simulates in O(T(n)^2)).

4.2 Non-deterministic TM (NTM)

delta: Q x Gamma -> P(Q x Gamma x {L, R}). Multiple possible transitions from each configuration.

Equivalence: Every NTM can be simulated by a deterministic TM (with exponential slowdown). NTM does NOT add computational power beyond DTM — both recognize the same class of languages (RE).

5. Church-Turing Thesis

The Church-Turing thesis states that any function that can be computed by an effective procedure (algorithm) can be computed by a Turing machine.

This is a thesis, not a theorem — it cannot be formally proved but is universally accepted. It means Turing machines capture our intuitive notion of "computability."

8. The Halting Problem

Problem: Given a TM M and input w, determine whether M halts on w.

Proof of Undecidability (by Diagonalization)

  1. Assume for contradiction that a TM H exists that decides the halting problem: H(M, w) = "accept" if M halts on w, "reject" otherwise.
  2. Construct a new TM D that takes a TM M as input:
    • D(M) = run H(M, encoding of M)
    • If H accepts (M halts on itself), then D loops forever.
    • If H rejects (M doesn't halt on itself), then D halts and accepts.
  3. Now run D on itself — D(D):
    • If D halts on D, then by construction, D should loop. Contradiction.
    • If D loops on D, then by construction, D should halt. Contradiction.
  4. Therefore, H cannot exist. The halting problem is undecidable.

9. Rice's Theorem

Statement: Every non-trivial property of the language of a Turing machine is undecidable.

A property is non-trivial if it is satisfied by some but not all RE languages.

Critical: Rice's theorem applies only to properties of the LANGUAGE recognized by the TM, not to structural properties of the TM itself (like number of states or transitions).

10. Reducibility

A reduction from problem A to problem B is a computable function f such that: w is in A if and only if f(w) is in B.

Key Results

  • If A reduces to B and B is decidable, then A is decidable.
  • If A reduces to B and A is undecidable, then B is undecidable.
  • Reduction is used to prove new problems undecidable by reducing known undecidable problems to them.

11. Key GATE Takeaways

  • A TM is the most powerful computational model. All variants (multi-tape, NTM) are equivalent.
  • Recursive languages (decidable): TM always halts. RE languages: TM may loop on non-members.
  • L is recursive iff both L and complement(L) are RE.
  • The halting problem is RE but not recursive (undecidable).
  • Rice's theorem: any non-trivial property of L(M) is undecidable.
  • Church-Turing thesis equates effective computability with Turing computability.
  • Reduction is the primary tool for proving undecidability of new problems.

Unsupervised Learning and Clustering

Section: Artificial Intelligence

1. Introduction to Unsupervised Learning

Unsupervised learning works with unlabeled data to discover hidden patterns, groupings, or representations. Unlike supervised learning, there are no target labels to guide the learning process. The main tasks are clustering, dimensionality reduction, and association rule mining.

2. K-Means Clustering

K-Means partitions n data points into K clusters, where each point belongs to the cluster with the nearest centroid. It minimizes the Within-Cluster Sum of Squares (WCSS).

Objective: Minimize J = ∑k=1K ∑x∈Ck ||x - μk||²
where μk is the centroid of cluster Ck

2.1 K-Means Algorithm

  1. Initialize: Randomly select K data points as initial centroids
  2. Assign: Assign each data point to the nearest centroid
  3. Update: Recompute centroids as the mean of all points in each cluster
  4. Repeat: Steps 2–3 until centroids converge (no change) or maximum iterations reached
Example: Given points {2, 3, 6, 8, 10, 12} and K=2.
Iteration 1: Initial centroids: μ1=3, μ2=10
Assign: C1={2, 3, 6}, C2={8, 10, 12}
Update: μ1=(2+3+6)/3=3.67, μ2=(8+10+12)/3=10
Iteration 2: Assign: C1={2, 3, 6}, C2={8, 10, 12} (no change)
Converged! Final clusters: {2, 3, 6} and {8, 10, 12}.

2.2 K-Means Properties

PropertyDetails
Time ComplexityO(n · K · d · I) where n=points, K=clusters, d=dimensions, I=iterations
ConvergenceAlways converges (J is monotonically non-increasing), but may reach local minimum
Cluster ShapeAssumes spherical (convex) clusters of similar size
SensitivitySensitive to initial centroid selection and outliers
K SelectionElbow method, silhouette score, gap statistic

2.3 K-Means++ Initialization

Improved initialization that spreads initial centroids apart: (1) Choose first centroid randomly, (2) Choose next centroid with probability proportional to D(x)² (squared distance to nearest existing centroid), (3) Repeat until K centroids are chosen. This provides O(log K) competitive approximation.

3. Hierarchical Clustering

Builds a hierarchy of clusters represented as a dendrogram (tree). No need to pre-specify K.

3.1 Agglomerative (Bottom-Up)

  1. Start with each data point as its own cluster (n clusters)
  2. Merge the two closest clusters
  3. Repeat until only one cluster remains

3.2 Divisive (Top-Down)

  1. Start with all data points in one cluster
  2. Split the cluster that has the most dissimilarity
  3. Repeat until each point is its own cluster

3.3 Linkage Criteria

LinkageDistance Between Clusters A and BProperties
Single Linkagemin d(a, b) for a∈A, b∈BCan handle non-convex shapes; chaining effect
Complete Linkagemax d(a, b) for a∈A, b∈BProduces compact clusters; sensitive to outliers
Average Linkageavg d(a, b) for a∈A, b∈BCompromise between single and complete
Ward's MethodIncrease in total WCSS after mergingMinimizes variance; tends to create equal-sized clusters

Time Complexity: O(n³) in general, O(n² log n) with efficient implementations. Space Complexity: O(n²) for the distance matrix.

4. DBSCAN (Density-Based Spatial Clustering of Applications with Noise)

A density-based clustering algorithm that can discover clusters of arbitrary shape and identify noise points (outliers).

4.1 Key Parameters

  • ε (eps): The radius of the neighborhood around a point
  • MinPts: Minimum number of points required to form a dense region

4.2 Point Classification

TypeDefinition
Core PointHas ≥ MinPts points within its ε-neighborhood (including itself)
Border PointWithin ε of a core point but has < MinPts in its neighborhood
Noise PointNeither core nor border; not within ε of any core point

4.3 DBSCAN Algorithm

  1. For each unvisited point, compute its ε-neighborhood
  2. If it has ≥ MinPts neighbors, create a new cluster and expand it by recursively adding density-reachable points
  3. If not a core point, mark as noise (may later become a border point)
DBSCAN Advantages over K-Means:
  • Does not require specifying K in advance
  • Can find clusters of arbitrary shape (not just spherical)
  • Robust to outliers (classifies them as noise)
  • Deterministic (given fixed ε and MinPts)
Disadvantages:
  • Difficulty with varying density clusters
  • Sensitive to ε and MinPts parameter selection
  • Does not work well in very high dimensions

5. Principal Component Analysis (PCA)

PCA is a dimensionality reduction technique that transforms data to a new coordinate system where the axes (principal components) are ordered by the amount of variance they capture.

5.1 PCA Algorithm

  1. Standardize the data (zero mean, unit variance)
  2. Compute the covariance matrix: C = (1/n) XTX
  3. Compute eigenvalues and eigenvectors of C
  4. Sort eigenvectors by decreasing eigenvalues
  5. Select the top k eigenvectors as principal components
  6. Project data onto the k-dimensional subspace
Variance explained by component i = λi / ∑λj
Total variance retained with k components = ∑i=1k λi / ∑j=1d λj
Example: If eigenvalues are λ1=4.5, λ2=2.3, λ3=0.15, λ4=0.05:
Total variance = 7.0
PC1 explains 4.5/7.0 = 64.3%
PC1 + PC2 explain (4.5+2.3)/7.0 = 97.1%
Retaining 2 components captures 97.1% of variance while reducing from 4D to 2D.

Key properties: Principal components are orthogonal. PCA is a linear transformation. PCA maximizes variance (equivalently, minimizes reconstruction error). It is sensitive to feature scaling.

6. Association Rule Mining (Apriori Algorithm)

Finds interesting relationships (associations) between items in transactional datasets.

6.1 Key Metrics

Support(A) = count(A) / N [frequency of itemset A]

Support(A → B) = count(A ∪ B) / N

Confidence(A → B) = Support(A ∪ B) / Support(A) = P(B|A)

Lift(A → B) = Confidence(A → B) / Support(B) = P(B|A) / P(B)

Lift > 1: positive association. Lift = 1: independent. Lift < 1: negative association.

6.2 Apriori Algorithm

Uses the Apriori property (anti-monotone): if an itemset is infrequent, all its supersets are also infrequent. This prunes the search space.

  1. Find all frequent 1-itemsets (support ≥ min_support)
  2. Generate candidate (k+1)-itemsets from frequent k-itemsets
  3. Prune candidates containing infrequent subsets
  4. Scan database to count support of remaining candidates
  5. Repeat until no more frequent itemsets are found
  6. Generate rules from frequent itemsets that meet min_confidence
Example: Market basket analysis.
Rule: {bread, butter} → {milk}, Support=0.3, Confidence=0.8, Lift=1.6
Interpretation: 30% of transactions contain all three items. Of those who buy bread and butter, 80% also buy milk. This is 1.6 times more likely than buying milk randomly.

7. Silhouette Score

Measures how similar a point is to its own cluster compared to other clusters. Useful for evaluating clustering quality and selecting the optimal K.

s(i) = (b(i) - a(i)) / max(a(i), b(i))

a(i) = average distance of point i to all other points in the same cluster
b(i) = minimum average distance of point i to points in any other cluster

s(i) ranges from -1 to +1
+1: well-clustered, 0: on boundary, -1: likely misclassified

The overall silhouette score is the average s(i) across all points. Higher values indicate better-defined clusters.
